# `<AlertDialogAction>`

## Overview
`AlertDialogAction` renders the **primary / confirm button** inside an `AlertDialog`.
The component wires your click handler to the dialog’s “OK” action and automatically closes the dialog after the handler runs.

## Styling Guidelines
* All examples use Tailwind classes.
* No additional Contember-specific theme classes are required.

## Core Concepts
* Place `AlertDialogAction` inside `AlertDialogFooter`.
* Use the `asChild` prop when you need a custom button component.
* The component closes the dialog after the click handler completes.

## Quick Start
```tsx
import {
	AlertDialog,
	AlertDialogTrigger,
	AlertDialogContent,
	AlertDialogFooter,
	AlertDialogCancel,
	AlertDialogAction,
} from '~/lib/ui/alert-dialog'
import { Button } from '~/lib/ui/button'

export default function Demo() {
	return (
		<AlertDialog>
			<AlertDialogTrigger asChild>
				<Button className="bg-blue-600 text-white px-3 py-1">Delete post</Button>
			</AlertDialogTrigger>

			<AlertDialogContent className="max-w-sm">
				<p>Are you sure you want to delete this post?</p>

				<AlertDialogFooter className="flex gap-2">
					<AlertDialogCancel className="border px-3 py-1">Cancel</AlertDialogCancel>
					<AlertDialogAction className="bg-red-600 text-white px-3 py-1">
						Confirm
					</AlertDialogAction>
				</AlertDialogFooter>
			</AlertDialogContent>
		</AlertDialog>
	)
}
```

## Props Reference
| Name    | Type    | Required | Default | Description                                           |
|---------|---------|----------|---------|-------------------------------------------------------|
| asChild | boolean | No       | `false` | Pass `true` to render your **own button component**.  |
| children | ReactNode | No | – | Content displayed inside the action button. |
| ...button props | *Inherited* | No | – | All props accepted by the underlying HTML `<button>`. |

## Examples

### Confirm deletion with a custom destructive button
```tsx
<AlertDialog>
	<AlertDialogTrigger asChild>
		<Button size="icon" variant="ghost">
			<Trash2Icon size={16} />
		</Button>
	</AlertDialogTrigger>

	<AlertDialogContent>
		<AlertDialogFooter>
			<AlertDialogCancel>Cancel</AlertDialogCancel>

			<AlertDialogAction asChild>
				<Button variant="destructive" onClick={handleDelete}>
					Delete
				</Button>
			</AlertDialogAction>
		</AlertDialogFooter>
	</AlertDialogContent>
</AlertDialog>
```
`asChild` lets you keep the styling and behavior of the `Button` component while still closing the dialog automatically.

### Persist entities from a repeater
```tsx
<AlertDialogFooter>
	<AlertDialogCancel>Cancel</AlertDialogCancel>

	<AlertDialogAction asChild>
		<SaveButton onPersist={onPersist} />
	</AlertDialogAction>
</AlertDialogFooter>
```
`AlertDialogAction` wraps `SaveButton`, so the dialog closes immediately after the save succeeds.

## Best Practices & Gotchas
* Always keep **one** `AlertDialogAction` per dialog to avoid user confusion.
* Use `asChild` when you need advanced styling, icons, or custom behavior.
* Do not attach `type="submit"` to the wrapped button; the component already handles click events.
* Combine with `AlertDialogCancel` for an accessible two-button pattern.
