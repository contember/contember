# AlertDialogCancel

## Overview
AlertDialogCancel renders the “Cancel” button inside an `AlertDialog`.
It closes the dialog and rolls back the pending action.

## Styling Guidelines
* All examples use Tailwind CSS classes.
* The component does not require any Contember-specific theme classes.

## Core Concepts
* Alert dialogs need a safe exit action.
* AlertDialogCancel must sit inside `AlertDialogFooter`.
* Clicking the button simply closes the dialog.
* The button usually carries the `Cancel` label, but you can pass any children.

## Quick Start
```tsx
import {
	AlertDialog,
	AlertDialogTrigger,
	AlertDialogContent,
	AlertDialogFooter,
	AlertDialogCancel,
	AlertDialogAction,
} from '~/lib/ui/alert-dialog'

export const SimpleAlert = () => (
	<AlertDialog>
		<AlertDialogTrigger className="px-4 py-2 bg-blue-600 text-white rounded">
			Delete
		</AlertDialogTrigger>

		<AlertDialogContent>
			<p className="text-sm">This action is permanent. Continue?</p>

			<AlertDialogFooter className="flex gap-2">
				<AlertDialogCancel className="px-3 py-1 border rounded">
					Cancel
				</AlertDialogCancel>
				<AlertDialogAction className="px-3 py-1 bg-red-600 text-white rounded">
					Delete
				</AlertDialogAction>
			</AlertDialogFooter>
		</AlertDialogContent>
	</AlertDialog>
)
```

## Examples

### Confirm deletion inside entity list
```tsx
<AlertDialog>
	<AlertDialogTrigger asChild>
		<Button size="icon" variant="ghost">
			<Trash2Icon size={16} />
		</Button>
	</AlertDialogTrigger>

	<AlertDialogContent>
		<AlertDialogHeader>
			<AlertDialogTitle>Delete item</AlertDialogTitle>
			<AlertDialogDescription>
				This action cannot be undone.
			</AlertDialogDescription>
		</AlertDialogHeader>

		<AlertDialogFooter>
			<AlertDialogCancel>Cancel</AlertDialogCancel>
			<AlertDialogAction asChild>
				<Button variant="destructive" onClick={handleDelete}>
					Delete
				</Button>
			</AlertDialogAction>
		</AlertDialogFooter>
	</AlertDialogContent>
</AlertDialog>
```
The `AlertDialogCancel` button lets users back out safely while `AlertDialogAction` performs the destructive operation.

## Best Practices & Gotchas
* Always include `AlertDialogCancel`; never force users into irreversible actions.
* Keep the cancel button visually less prominent than the primary action.
* Place it first in reading order for accessibility.
* Avoid extra side effects in `onClick`; the dialog handles closing automatically.
