## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* `AlertDialogContent` works with any Contember theme; no additional theme classes are required.

## Core Concepts
* `AlertDialogContent` renders the body of an `AlertDialog`.
* Place interactive elements (forms, buttons) inside `AlertDialogContent`.
* Combine it with `AlertDialogHeader`, `AlertDialogFooter`, and other dialog primitives for a complete UX.
* Wrap the whole dialog with `AlertDialog` and use `AlertDialogTrigger` to open it.

## Quick Start
```tsx
import { AlertDialog, AlertDialogTrigger, AlertDialogContent, AlertDialogAction, AlertDialogCancel, AlertDialogFooter } from '~/lib/ui/alert-dialog'

export const DeleteButton = () => (
	<AlertDialog>
		<AlertDialogTrigger asChild>
			<button className="px-3 py-2 text-sm bg-red-600 text-white rounded">Delete</button>
		</AlertDialogTrigger>

		<AlertDialogContent className="max-w-sm">
			<p className="text-sm">Are you sure you want to delete this item?</p>

			<AlertDialogFooter>
				<AlertDialogCancel className="px-3 py-2 border rounded">Cancel</AlertDialogCancel>
				<AlertDialogAction className="px-3 py-2 bg-red-600 text-white rounded">Confirm</AlertDialogAction>
			</AlertDialogFooter>
		</AlertDialogContent>
	</AlertDialog>
)
```

## Sub-Components / Building Blocks
### AlertDialogHeader
High-level wrapper for titles and descriptions.

### AlertDialogFooter
Container for action buttons placed at the dialog bottom.

### AlertDialogTrigger
Element that toggles dialog visibility.

### AlertDialogAction
Primary action button—often “Confirm”.

### AlertDialogCancel
Secondary action button—often “Cancel”.

## Examples

### Validate locales before save
```tsx
const [open, setOpen] = useState(false)

<EntityBeforePersist listener={entity => {
	const isValid = checkLocalesConsistency()
	if (!isValid) {
		setOpen(true)
		return () => entity().addError({ type: 'validation', code: 'inconsistent' })
	}
}}>
	<AlertDialog open={open} onOpenChange={setOpen}>
		<AlertDialogContent>
			<AlertDialogTitle>Translations are inconsistent</AlertDialogTitle>
			<p className="text-sm">Fix all translations or force save.</p>

			<div className="flex gap-2 justify-end">
				<Button onClick={() => forceSave()} className="bg-green-600 text-white">Save anyway</Button>
				<Button onClick={() => setOpen(false)} variant="outline">Back</Button>
			</div>
		</AlertDialogContent>
	</AlertDialog>
</EntityBeforePersist>
```
`AlertDialogContent` holds both text and action buttons, proving it can contain complex layouts.

## Best Practices & Gotchas
* Always place `AlertDialogContent` inside an `AlertDialog`.
* Use `className` to constrain width or height; the component forwards the prop to its root.
* Pair `AlertDialogContent` with `AlertDialogFooter` for consistent button alignment.
* Remember to add `overflow-auto` when the content can get long, e.g., large forms.
* Wrap destructive actions in an extra confirmation layer to avoid accidental data loss.
