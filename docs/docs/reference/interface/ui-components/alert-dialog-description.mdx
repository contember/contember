# AlertDialogDescription

## Overview
`AlertDialogDescription` renders auxiliary text inside an `AlertDialog`.
Use it to explain the consequences of the action your dialog confirms.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No special Contember theme classes are required for `AlertDialogDescription`.

## Core Concepts
* Belongs inside `AlertDialogContent` â†’ `AlertDialogHeader`.
* Accepts React children; it does not manage state or styling.
* Renders as a paragraph element, so typography utilities apply.

## Quick Start
```tsx
import {
	AlertDialog,
	AlertDialogTrigger,
	AlertDialogContent,
	AlertDialogHeader,
	AlertDialogTitle,
	AlertDialogDescription,
	AlertDialogFooter,
	AlertDialogCancel,
	AlertDialogAction,
} from '~/lib/ui/alert-dialog'
import { Button } from '~/lib/ui/button'

export const DeleteButton = () => (
	<AlertDialog>
		<AlertDialogTrigger asChild>
			<Button className="bg-red-600 text-white hover:bg-red-700">Delete</Button>
		</AlertDialogTrigger>

		<AlertDialogContent className="max-w-sm">
			<AlertDialogHeader>
				<AlertDialogTitle>Delete item?</AlertDialogTitle>
				<AlertDialogDescription className="text-sm text-gray-500">
					This action is irreversible. Your data will be permanently removed.
				</AlertDialogDescription>
			</AlertDialogHeader>

			<AlertDialogFooter>
				<AlertDialogCancel className="px-3 py-2">Cancel</AlertDialogCancel>
				<AlertDialogAction asChild>
					<Button className="bg-red-600 text-white hover:bg-red-700">Confirm</Button>
				</AlertDialogAction>
			</AlertDialogFooter>
		</AlertDialogContent>
	</AlertDialog>
)
```

## Examples

### Display API error inside a dialog
```tsx
<AlertDialogDescription className="text-sm text-red-600">
	{errorMessage}
</AlertDialogDescription>
```
Use Tailwind color utilities to highlight critical information.

### Bulk delete confirmation
```tsx
<AlertDialogDescription className="text-sm text-gray-500">
	You are about to remove all selected attendees. This action cannot be undone.
</AlertDialogDescription>
```
Pair the description with a destructive button to make intent clear.

## Best Practices & Gotchas
* Keep text concise; one or two sentences suffice.
* Apply `text-sm` and muted colors to keep attention on the title and actions.
* Do not place interactive elements inside `AlertDialogDescription`.
* Always localize the description in multilingual apps.
