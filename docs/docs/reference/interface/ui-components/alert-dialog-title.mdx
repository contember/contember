# AlertDialogTitle

## Styling Guidelines
* All examples use Tailwind utility classes.
* No additional Contember theme classes are required for `AlertDialogTitle`.

## Core Concepts
* `AlertDialogTitle` provides an accessible heading for the dialog.
* Place it inside `AlertDialogContent` or `AlertDialogHeader`.
* Keep the title short; longer text belongs in `AlertDialogDescription`.
* Tailwind classes can override typography or spacing.

## Quick Start
```tsx
import {
	AlertDialog,
	AlertDialogTrigger,
	AlertDialogContent,
	AlertDialogTitle,
	AlertDialogAction,
	AlertDialogCancel,
	AlertDialogFooter,
} from '~/lib/ui/alert-dialog'
import { Button } from '~/lib/ui/button'

export default function QuickStart() {
	return (
		<AlertDialog>
			<AlertDialogTrigger asChild>
				<Button className="bg-red-600 text-white hover:bg-red-700">Delete file</Button>
			</AlertDialogTrigger>

			<AlertDialogContent>
				<AlertDialogTitle className="text-lg font-semibold">
					Are you sure?
				</AlertDialogTitle>

				<AlertDialogFooter className="flex gap-2 justify-end mt-4">
					<AlertDialogCancel asChild>
						<Button variant="outline">Cancel</Button>
					</AlertDialogCancel>
					<AlertDialogAction asChild>
						<Button variant="destructive">Delete</Button>
					</AlertDialogAction>
				</AlertDialogFooter>
			</AlertDialogContent>
		</AlertDialog>
	)
}
```

## Examples

### Confirm deletion dialog
```tsx
import {
	AlertDialog,
	AlertDialogTrigger,
	AlertDialogContent,
	AlertDialogHeader,
	AlertDialogFooter,
	AlertDialogTitle,
	AlertDialogDescription,
	AlertDialogCancel,
	AlertDialogAction,
} from '~/lib/ui/alert-dialog'
import { Button } from '~/lib/ui/button'

export function DeleteEntityDialog({ onConfirm }: { onConfirm: () => void }) {
	return (
		<AlertDialog>
			<AlertDialogTrigger asChild>
				<Button size="icon" variant="ghost">
					<Trash2Icon size={16} />
				</Button>
			</AlertDialogTrigger>

			<AlertDialogContent>
				<AlertDialogHeader>
					<AlertDialogTitle>Delete entity</AlertDialogTitle>
					<AlertDialogDescription>
						This action cannot be undone. This will permanently delete the record from the database.
					</AlertDialogDescription>
				</AlertDialogHeader>

				<AlertDialogFooter>
					<AlertDialogCancel asChild>
						<Button variant="outline">Cancel</Button>
					</AlertDialogCancel>
					<AlertDialogAction asChild>
						<Button variant="destructive" onClick={onConfirm}>
							Delete
						</Button>
					</AlertDialogAction>
				</AlertDialogFooter>
			</AlertDialogContent>
		</AlertDialog>
	)
}
```
The example shows `AlertDialogTitle` as the primary heading, followed by a descriptive paragraph and actionable buttons.

## Best Practices & Gotchas
* Use only one `AlertDialogTitle` per dialog for accessibility.
* Keep the title concise (sentence case, no punctuation).
* Pair the title with `AlertDialogDescription` for additional context.
* Avoid heavy styling; override with Tailwind classes when necessary.
* Do not place interactive elements (links, buttons) inside the title.
