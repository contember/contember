## Core Concepts
* AnchorRenderer renders a Slate `AnchorElement`.
* You must pass the `element` prop obtained from the Slate editor.
* The component is presentation-only; editing logic lives elsewhere.

## Quick Start
```tsx
import { AnchorRenderer } from './elements/AnchorRenderer'

/**
 * `anchorElement` comes from your Slate editor's element tree.
 */
const Example = ({ anchorElement }: { anchorElement: AnchorElement }) => (
	<AnchorRenderer element={anchorElement} />
)
```

## Examples
### Rendering inside a Slate element loop
```tsx
import { AnchorRenderer } from './elements/AnchorRenderer'

const RenderElement = ({ element, attributes, children }: RenderElementProps) => {
	switch (element.type) {
		case 'anchor':
			return (
				<AnchorRenderer element={element} {...attributes}>
					{children}
				</AnchorRenderer>
			)
		default:
			return <p {...attributes}>{children}</p>
	}
}
```
This example shows how you can delegate rendering of anchor elements while keeping other elements generic.

## Best Practices & Gotchas
* Always pass the exact `AnchorElement` instance you receive from Slate.
* Do not wrap AnchorRenderer with additional DOM nodes; it already outputs an anchor.
