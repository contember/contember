## Overview
`AudioRepeaterField` lets you upload, preview, and order multiple audio files inside an Entity context.  
It supports common audio formats, validates size, and renders an inline audio player for each file.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* No Contember-specific theme classes are required by default.

## Core Concepts
* You work inside `<EntitySubTree>` or `<EntityListSubTree>`.
* The component manages an ordered relation of audio files.
* Each audio item can expose additional metadata fields (duration, fileSize, etc.).
* Sorting happens through `orderBy` or `sortableBy`.

## Quick Start
```tsx
<EntitySubTree entity="podcast">
	<AudioRepeaterField
		className="w-full"
		label="Podcast Episodes"
		field="episodes"
		urlField="audio.url"
		orderBy="createdAt"
	/>
</EntitySubTree>
```

## Examples

### Drag-and-Drop Sorting with Metadata
```tsx
<EntityListSubTree entities="episodes">
	<AudioRepeaterField
		className="w-full border rounded p-4"
		field="episodes"
		baseField="audio"
		urlField="url"
		sortableBy="order"
		durationField="duration"
		fileNameField="fileName"
		fileSizeField="fileSize"
		fileTypeField="fileType"
		lastModifiedField="lastModified"
		label="Audio files"
	/>
</EntityListSubTree>
```
The list becomes draggable thanks to `sortableBy="order"`.  
Contember stores duration, size, and other metadata into dedicated columns.

## Best Practices & Gotchas
* Always wrap the component in an Entity context; otherwise it throws an error.
* Keep `fileSizeField` indexed if you plan to filter by size.
* Use `orderBy` only when you do not need manual reordering.
* Validate accepted MIME types server-side too; the client check is not foolproof.
