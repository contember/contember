## Overview
BlockEditorHandle renders a small drag handle inside a block-editor layout.  
You use it to reorder blocks via drag-and-drop in Contember’s sortable block editor.

## Styling Guidelines
* All snippets use Tailwind classes.  
* The component itself has no required Contember theme classes, but you should add cursor and spacing utilities for better UX.

## Core Concepts
* Each block inside BlockEditor should contain exactly one `<BlockEditorHandle />`.
* Contember’s drag-and-drop context recognizes the handle automatically.
* The component renders a `div`—style it as you would any other element.

## Quick Start
```tsx
import { BlockEditorHandle } from '@contember/react-blocks'

export const MyBlock = () => (
	<div className="flex items-start gap-2">
		<BlockEditorHandle className="cursor-move p-2 text-gray-500 hover:text-gray-700" />
		<div className="flex-1">
			{/* Block content */}
		</div>
	</div>
)
```

## Detailed Guide
1. Place `<BlockEditorHandle />` as the first child of your block container.  
2. Apply `cursor-move` so users know they can drag.  
3. Use padding to enlarge the hit-area for accessibility.

```tsx
<BlockEditorHandle className="cursor-move p-2 rounded hover:bg-gray-100" />
```

You can combine with icons:

```tsx
<BlockEditorHandle className="cursor-grab p-2">
	<svg className="w-4 h-4 text-gray-400" viewBox="0 0 20 20">
		<circle cx="6" cy="6" r="2" />
		<circle cx="14" cy="6" r="2" />
		<circle cx="6" cy="14" r="2" />
		<circle cx="14" cy="14" r="2" />
	</svg>
</BlockEditorHandle>
```

### Reordering within a `SortableBlocksField`
```tsx
import { SortableBlocksField } from '@contember/react-blocks'

const PageForm = () => (
	<SortableBlocksField field="content">
		{({ blocks }) => blocks.map(block => (
			<div key={block.id} className="flex gap-4 items-start border p-4 rounded mb-2">
				<BlockEditorHandle className="cursor-grab p-2 text-gray-500" />
				{/* other block fields */}
			</div>
		))}
	</SortableBlocksField>
)
```
The handle enables drag-and-drop ordering without extra configuration.

### Advanced drag overlay with `@dnd-kit`
```tsx
({ children, element }: { children: ReactNode; element: Element }) => {
	const sortable = useSortable({ id: element.key as string })
	const isOver = sortable.isOver
	const activeSortable = sortable.active?.data.current?.sortable
	const isAfter = (sortable.data?.sortable.index ?? 0) > activeSortable?.index
	const isActive = sortable.active?.id === element.key
	const contentRef = React.useRef<HTMLDivElement>(null)

	return (
		<>
			<div className="relative">
				<div className="relative">
					{isOver && !isAfter ? <DropIndicator placement="top" /> : null}
				</div>

				<div className={'p-4 ' + (isActive ? 'opacity-60' : '')} ref={sortable.setNodeRef}>
					<BlockEditorHandle
						ref={sortable.setActivatorNodeRef}
						{...sortable.listeners}
						className="cursor-grab p-2 text-gray-500"
					/>
					<div ref={contentRef}>{children}</div>
				</div>

				<div className="relative">
					{isOver && isAfter ? <DropIndicator placement="bottom" /> : null}
				</div>
			</div>

			{isActive && contentRef.current && (
				<Portal>
					<DragOverlay>
						<div className="opacity-80 flex">
							<div
								className="p-4 bg-white bg-opacity-80 backdrop-blur-sm"
								dangerouslySetInnerHTML={{ __html: contentRef.current.innerHTML }}
							/>
						</div>
					</DragOverlay>
				</Portal>
			)}
		</>
	)
}
```
This pattern mirrors production use.  
You connect the handle to `useSortable`, render drop indicators, and show an overlay during drag.

## Best Practices & Gotchas
* Always wrap the handle in a `flex` container so it keeps its size.  
* Add `cursor-grab` or `cursor-move` to signal interactivity.  
* Increase padding for touch devices.  
* Do not nest multiple handles inside one block.  
* When integrating with `@dnd-kit`, forward `sortable.listeners` directly to `<BlockEditorHandle />`.
