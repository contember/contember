# blockEditorPlugins

## Overview
`blockEditorPlugins` bundles advanced Slate plugins that bring structural editing to Contember’s `BlockEditor`. You gain block-level formatting, lists, tables and drag‑and‑drop sorting without extra wiring.

Use the preset when you need rich, section‑based documents rather than plain text fields.

## Styling Guidelines
* All snippets apply Tailwind CSS directly in JSX.
* No Contember‑specific theme classes are required, but keep your editor container flexible with `prose` or `flex` utilities.

## Core Concepts
* Plugin preset — a ready‑made array passed to `BlockEditor`.
* Structural blocks — headings, paragraphs, lists, rules and tables.
* Drag‑and‑drop — powered by `withSortable` and rendered by `SortableBlock`.
* Declarative sections — each sortable block receives a unique `name` to map data.

## Quick Start
```tsx
import { BlockEditor, SortableBlock } from '@contember/react-admin';
import { blockEditorPlugins } from './blockEditorPlugins';

export const ArticleEditor = () => (
  <div className="prose max-w-none">
    <BlockEditor plugins={blockEditorPlugins}>
      <SortableBlock name="section" className="border-l-4 pl-4" />
    </BlockEditor>
  </div>
);
```

## Sub-Components / Building Blocks

### SortableBlock
Renders the draggable handle and wraps each block so users can reorder them.

### withSortable
Higher‑order plugin that injects sortability into editor nodes. It is already included inside `blockEditorPlugins`.

## Examples

### Page Builder with Sections and Tables
```tsx
<BlockEditor plugins={blockEditorPlugins} className="bg-white shadow-sm p-6">
  <SortableBlock name="section" className="mb-8" />
</BlockEditor>
```
This setup lets authors insert headings, lists or tables inside each section and reorder sections by dragging the handle.

### Horizontal Rule Divider
```tsx
<BlockEditor plugins={blockEditorPlugins}>
  <SortableBlock name="section" />
  {/* Users can insert a horizontal rule via the toolbar */}
</BlockEditor>
```
A horizontal rule appears as a draggable block, separating content logically.

## Best Practices & Gotchas
* Keep each `SortableBlock`’s `name` unique within the parent editor.
* Wrap the editor in a scrollable container to improve drag‑and‑drop UX.
* Disable Tailwind’s `prose` default margins if your layout already has spacing.
* Do not mix `richTextFieldPlugins` with `blockEditorPlugins` in the same editor.
