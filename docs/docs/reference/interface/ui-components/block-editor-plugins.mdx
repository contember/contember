# blockEditorPlugins

## Overview
`blockEditorPlugins` bundles advanced Slate plugins that add structural editing to Contember’s `BlockEditor`.  
You gain headings, lists, tables, scroll targets, and drag-and-drop sorting in one preset.

Use the bundle when you need rich, section-based content rather than plain text.

## Styling Guidelines
* All examples use Tailwind CSS classes directly in JSX.
* No Contember-specific theme classes are required, but keep the editor container flexible with `prose` or `flex` utilities.

## Core Concepts
* Plugin preset – a ready-made array passed to `BlockEditor`.
* Structural blocks – headings, paragraphs, lists, rules, tables, and scroll targets.
* Drag-and-drop – powered by `withSortable` and rendered by `SortableBlock`.
* Declarative sections – each sortable block receives a unique `name` to map data.

## Quick Start
```tsx
import { BlockEditor, SortableBlock } from '@contember/react-admin'
import { blockEditorPlugins } from './plugins'

export const ArticleEditor = () => (
  <div className="prose max-w-none">
    <BlockEditor plugins={blockEditorPlugins}>
      <SortableBlock name="section" className="border-l-4 pl-4" />
    </BlockEditor>
  </div>
)
```

## Sub-Components / Building Blocks

### SortableBlock
Renders the draggable handle and wraps each block so you can reorder them.

### withSortable
Higher-order plugin that injects sortability into editor nodes. It is already included inside `blockEditorPlugins`.

## Examples

### Minimal Setup
```tsx
<BlockEditor plugins={blockEditorPlugins}>
  <SortableBlock name="section" />
</BlockEditor>
```
Use this when you need the lightest possible configuration.  
The preset still adds headings, lists, tables, and scroll targets.

## Best Practices & Gotchas
* Keep each `SortableBlock`’s `name` unique within the parent editor.
* Wrap the editor in a scrollable container to improve drag-and-drop UX.
* Disable Tailwind’s `prose` default margins if your layout already has spacing.
* Do not mix `richTextFieldPlugins` with `blockEditorPlugins` in the same editor.
