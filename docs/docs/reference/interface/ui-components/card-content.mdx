## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* CardContent adds no extra Contember-specific theme classes.

## Core Concepts
* CardContent is a structural child of `Card`.
* It simply wraps arbitrary JSX to create consistent internal card spacing.
* Compose it with `CardHeader`, `CardFooter`, and other card primitives.

## Quick Start
```tsx
import { Card, CardContent } from '~/lib/ui/card'

export const SimpleCard = () => (
	<Card className="w-80">
		<CardContent className="grid gap-4">
			<p>Hello world</p>
		</CardContent>
	</Card>
)
```

## Examples

### Change-Password Card
```tsx
<Card className="w-[32rem]">
	<CardHeader>
		<CardTitle className="text-2xl">Change Password</CardTitle>
	</CardHeader>
	<CardContent>
		<ChangeMyPasswordForm>
			<form className="grid gap-4">
				<ChangeMyPasswordFormFields />
			</form>
		</ChangeMyPasswordForm>
	</CardContent>
</Card>
```
CardContent wraps the entire form so padding and spacing stay consistent inside the card body.

### Stats Dashboard Widget
```tsx
<Card className="w-full">
	<CardHeader>
		<CardTitle>Stats</CardTitle>
	</CardHeader>
	<CardContent>
		<StatsCardContent total={activeTotal} categories={activeCategories} />
	</CardContent>
</Card>
```
You can render any component—charts, tables, or plain text—inside CardContent.

## Best Practices & Gotchas
* Use one CardContent per card; avoid nesting CardContent inside another CardContent.
* Combine CardContent with Tailwind’s `grid`, `flex`, or `space-*` utilities to control internal layout.
* Keep heavy logic outside; CardContent should stay a presentational wrapper.
