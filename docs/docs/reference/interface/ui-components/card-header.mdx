# CardHeader

## Overview
CardHeader wraps the top section of a Card.
Use it to group CardTitle and optional CardDescription or action buttons inside a consistent padding block.

## Styling Guidelines
* All snippets use Tailwind classes.
* CardHeader relies on the Contember card theme classes applied by the parent Card.

## Core Concepts
* CardHeader is a visual container, not a layout primitive.
* It inherits HTML `<div>` props, so you can forward `className`, `id`, and events.
* Place only header-related elements (titles, descriptions, actions) inside CardHeader.

## Quick Start
```tsx
import { Card, CardHeader, CardTitle } from '~/lib/ui/card'

<Card className="w-96">
  <CardHeader className="pb-0">
    <CardTitle className="text-2xl">Profile</CardTitle>
  </CardHeader>
</Card>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| *Inherited props* | `React.HTMLAttributes<HTMLDivElement>` | No | – | All standard div attributes are forwarded. |

## Examples

### Header with description
```tsx
import { Card, CardHeader, CardTitle, CardDescription } from '~/lib/ui/card'

<Card className="w-96">
  <CardHeader>
    <CardTitle className="text-2xl">Login</CardTitle>
    <CardDescription>Enter your credentials</CardDescription>
  </CardHeader>
</Card>
```
Place CardDescription directly after CardTitle for a compact header.

### Header with actions
```tsx
import { Card, CardHeader, CardTitle } from '~/lib/ui/card'
import { Button } from '~/lib/ui/button'

<Card className="w-96">
  <CardHeader className="flex items-center justify-between">
    <CardTitle>Members</CardTitle>
    <Button size="sm">Invite</Button>
  </CardHeader>
</Card>
```
Add utility classes to align actions next to the title.

## Best Practices & Gotchas
* Keep header content concise; move form fields to CardContent.
* Use flex utilities when adding action buttons.
* Avoid heavy padding in CardHeader—Card already provides spacing.
* Do not place interactive fields inside CardHeader unless they are simple toggles.
