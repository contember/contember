## Card

### Overview
The `Card` component groups related content into a visually separated container.
Use it to build dashboards, forms, and lists with a consistent look and feel.

### Styling Guidelines
* All examples use Tailwind CSS.
* `Card` and its sub-components rely on Contember’s default theme classes shipped with `@contember/ui`. No extra configuration is required.

### Core Concepts
* `Card` is a plain `div` with rounded corners, shadow, and background.
* Compose the UI with `CardHeader`, `CardTitle`, `CardDescription`, `CardContent`, and `CardFooter`.
* Pass `className` to any sub-component to extend or override styling.
* Nest Contember form or data components inside `CardContent` for clean layouts.

### Quick Start
```tsx
import { Card, CardHeader, CardTitle, CardContent } from '@app/lib/ui/card'

export const SimpleCard = () => (
  <Card className="w-96">
    <CardHeader>
      <CardTitle className="text-2xl">Hello</CardTitle>
    </CardHeader>
    <CardContent>
      <p className="text-gray-700">World</p>
    </CardContent>
  </Card>
)
```

### Sub-Components / Building Blocks
#### CardHeader
Groups header elements. Usually contains `CardTitle` and `CardDescription`.

#### CardTitle
Displays the main heading. Accepts typography classes like `text-2xl`.

#### CardDescription
Shows an optional subtitle or helper text.

#### CardContent
Holds the main body. Use grids, forms, or any custom layout inside.

#### CardFooter
Provides space for actions such as buttons and links.

### Examples

#### Login Form
```tsx
<Card className="w-96 relative">
  <CardHeader>
    <CardTitle className="text-2xl">Sign in</CardTitle>
    <CardDescription>Please enter your credentials</CardDescription>
  </CardHeader>
  <CardContent>
    <LoginForm>
      <form className="grid gap-4">
        <LoginFormFields />
      </form>
    </LoginForm>
  </CardContent>
</Card>
```
The `Card` wraps the whole form, giving a clear boundary and consistent spacing.

#### Infinite Scroll Grid Item
```tsx
<DataViewInfiniteLoadEachRow>
  <Card className="h-72">
    <CardHeader>
      <CardTitle>
        <Field field="title" />
      </CardTitle>
    </CardHeader>
  </Card>
</DataViewInfiniteLoadEachRow>
```
Use `Card` inside an infinite load grid to keep each record visually distinct.

### Best Practices & Gotchas
* Keep titles short; wrap long text inside `CardDescription` instead.
* Avoid placing heavy layout logic in `CardHeader`; keep it lean.
* Pass responsive width classes (`w-full`, `md:w-96`) directly to `Card`.
* Remember that `Card` spreads all native `div` props—ARIA attributes work out of the box.

### Further Reading
* Button – commonly used inside `CardFooter`.
* DataView – often renders rows as `Card`.
* Form components like `LoginForm`, `ChangeMyPasswordForm`, `PasswordResetForm`.
