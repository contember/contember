## Styling Guidelines
All examples use Tailwind CSS utility classes.
The component relies on the default Contember theme; no extra theme classes are required.

## Core Concepts
- CheckboxInput is a thin wrapper around the native `<input type="checkbox">`.
- You control its state through the `checked` and `onChange` props, just like a regular checkbox.
- All standard HTML input props pass through to the underlying element.
- The component is visually unstyled; you add Tailwind classes on the wrapping label or container.

## Quick Start
```tsx
import { useState } from 'react'
import { CheckboxInput } from '~/lib/ui/input'

export default function Example() {
	const [accepted, setAccepted] = useState(false)

	return (
		<label className="flex gap-2 items-center">
			<CheckboxInput
				checked={accepted}
				onChange={e => setAccepted(e.target.checked)}
			/>
			<span className="text-sm">I accept the terms</span>
		</label>
	)
}
```

## Examples

### Select Multiple Items
The pattern below shows how to add or remove an item’s `id` from an array stored in state.

```tsx
function ItemCheckbox({ id, value, onChange }: { id: string, value: string[], onChange: (v: string[]) => void }) {
	return (
		<label className="flex gap-2 items-center font-normal">
			<CheckboxInput
				checked={value.includes(id)}
				onChange={e =>
					onChange(e.target.checked ? [...value, id] : value.filter(v => v !== id))
				}
			/>
			<span>{id}</span>
		</label>
	)
}
```

### Toggle Boolean Flag
Use a single checkbox to flip a boolean flag in state.

```tsx
function ReplaceExistingToggle() {
	const [replaceExisting, setReplaceExisting] = useState(true)

	return (
		<div className="flex items-center gap-2">
			<CheckboxInput
				id="replace-existing"
				checked={replaceExisting}
				onChange={() => setReplaceExisting(prev => !prev)}
			/>
			<label htmlFor="replace-existing" className="text-sm">
				Replace existing items
			</label>
		</div>
	)
}
```

## Best Practices & Gotchas
- Keep the component controlled; always pair `checked` with `onChange`.
- Wrap the checkbox in a `<label>` to enlarge the click target.
- Store arrays of selected IDs in state, then compute `checked` per item.
- Forward other HTML props (`id`, `name`, `disabled`) as needed.
