## Overview
CollapsibleContent displays collapsible sections inside a Collapsible container.
You place it under a `CollapsibleTrigger` to reveal or hide extra information without navigating away.

## Styling Guidelines
* All examples use Tailwind classes.
* No special Contember theme classes are required; standard Tailwind utilities work out of the box.

## Core Concepts
* Collapsible state lives in the parent `Collapsible`.
* `CollapsibleContent` only shows when the parent is open.
* Combine with `CollapsibleTrigger` to build accessible disclosure patterns.
* You may nest multiple `CollapsibleContent` blocks inside one `Collapsible`.

## Quick Start
```tsx
import { Collapsible, CollapsibleTrigger, CollapsibleContent } from '~/lib/ui/collapsible'

export const SimpleCollapsible = () => (
	<Collapsible className="w-full max-w-md border rounded-md">
		<CollapsibleTrigger className="w-full px-4 py-2 text-left font-medium">
			Show details
		</CollapsibleTrigger>

		<CollapsibleContent className="px-4 py-2 bg-gray-50">
			Here is the hidden content.
		</CollapsibleContent>
	</Collapsible>
)
```

## Sub-Components / Building Blocks
### Collapsible
Parent component that controls open / close state.

### CollapsibleTrigger
Button-like element that toggles the `Collapsible` state.

### CollapsibleContent
The component described in this page; wraps the hidden area.

## Examples
### Sidebar Menu Item
```tsx
<Collapsible open={isOpen} onOpenChange={setIsOpen} className="group/collapsible">
	<SidebarMenuItem>
		<CollapsibleTrigger asChild>
			<SidebarMenuButton>
				{icon}
				<span>{label}</span>
				<span className="ml-auto">
					{isOpen ? <ChevronDown className="h-4 w-4" /> : <ChevronRight className="h-4 w-4" />}
				</span>
			</SidebarMenuButton>
		</CollapsibleTrigger>

		<CollapsibleContent>
			<SidebarMenuSub>{children}</SidebarMenuSub>
		</CollapsibleContent>
	</SidebarMenuItem>
</Collapsible>
```
This pattern lets you build multi-level sidebar navigation. Each submenu collapses independently while keeping active paths visible.

## Best Practices & Gotchas
* Place `CollapsibleContent` immediately after its matching `CollapsibleTrigger`.
* Manage the open state in a parent component when you need to sync multiple collapsibles.
* Avoid heavy layouts inside `CollapsibleContent`; large height changes can feel jarring.
* Remember to add transition classes if you want smooth height animations.
