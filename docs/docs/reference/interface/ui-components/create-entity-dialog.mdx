# CreateEntityDialog

## Overview
`CreateEntityDialog` is a modal that lets you create a new entity without leaving the current screen.  
You pass a trigger element (for example, a button) and any form or content you want inside the dialog.

## Styling Guidelines
* All code samples use Tailwind CSS.
* No Contember‑specific theme classes are required out of the box.

## Core Concepts
* A **trigger** element opens the dialog.
* The component **renders children** as the dialog body.
* All state (open / close) is encapsulated; you handle only the form logic.

## Quick Start
```tsx
import { CreateEntityDialog } from '@contember/react'
import { Button } from './ui'

<CreateEntityDialog
  trigger={<Button className="bg-primary text-white px-4 py-2 rounded">Create</Button>}
>
  <EntityForm className="p-6 space-y-4" />
</CreateEntityDialog>
```

## Examples

### 1. Dialog with Confirmation Toast
```tsx
import { CreateEntityDialog } from '@contember/react'
import { Button } from './ui'
import { toast } from 'react-hot-toast'

<CreateEntityDialog
  trigger={<Button className="bg-green-600 text-white px-4 py-2 rounded">Add Company</Button>}
>
  <CompanyForm
    onSuccess={() => toast.success('Company created')}
    className="p-6 grid gap-4"
  />
</CreateEntityDialog>
```
Use this when you want user feedback after creation.

### 2. Inline List Refresh
```tsx
const CompaniesPage = () => {
  const [reloadKey, setReloadKey] = useState(0)

  return (
    <div className="space-y-6">
      <CreateEntityDialog
        trigger={<button className="btn-primary">New Company</button>}
      >
        <CompanyForm onSuccess={() => setReloadKey(Date.now())} />
      </CreateEntityDialog>

      <CompanyList key={reloadKey} />
    </div>
  )
}
```
The list reloads once the form inside the dialog succeeds.

## Best Practices & Gotchas
* Keep the dialog form concise; long forms feel cramped in modals.
* Pass a visually distinct trigger button; otherwise users miss the action.
* Handle success inside the child form, not in `CreateEntityDialog`.
* Avoid nesting multiple dialogs—they confuse focus and accessibility.
