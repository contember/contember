# CreateEntityDialog

## Overview
`CreateEntityDialog` is a modal component that lets you create a new entity without leaving the current screen.
You pass a trigger element (for example, a button) and any form or content you want inside the dialog.

## Styling Guidelines
* All code samples use Tailwind CSS.
* No Contember-specific theme classes are required out of the box.

## Core Concepts
* A **trigger** element opens the dialog.
* The component **renders children** as the dialog body.
* All state (open / close) is encapsulated; you handle only the form logic.

## Quick Start
```tsx
import { CreateEntityDialog } from '@contember/react'
import { Button } from './ui'

<CreateEntityDialog
  trigger={<Button className="bg-primary text-white px-4 py-2 rounded">Create</Button>}
>
  <EntityForm className="p-6 space-y-4" />
</CreateEntityDialog>
```

## Examples

### 1. Basic Dialog
```tsx
import { CreateEntityDialog } from '@contember/react'
import { Button } from './ui'

<CreateEntityDialog
  trigger={<Button className="bg-blue-600 text-white px-4 py-2 rounded">Create New Entity</Button>}
>
  <EntityForm />
</CreateEntityDialog>
```
Use this as the minimal setup when you only need a modal wrapper around an existing form.

### 2. Dialog with Confirmation Toast
```tsx
import { CreateEntityDialog } from '@contember/react'
import { Button } from './ui'
import { toast } from 'react-hot-toast'

<CreateEntityDialog
  trigger={<Button className="bg-green-600 text-white px-4 py-2 rounded">Add Company</Button>}
>
  <CompanyForm
    onSuccess={() => toast.success('Company created')}
    className="p-6 grid gap-4"
  />
</CreateEntityDialog>
```
Show a toast from the nested form’s `onSuccess` handler to confirm the action.

### 3. Inline List Refresh
```tsx
const CompaniesPage = () => {
  const [reloadKey, setReloadKey] = useState(0)

  return (
    <div className="space-y-6">
      <CreateEntityDialog
        trigger={<button className="btn-primary">New Company</button>}
      >
        <CompanyForm onSuccess={() => setReloadKey(Date.now())} />
      </CreateEntityDialog>

      <CompanyList key={reloadKey} />
    </div>
  )
}
```
Update the surrounding list by bumping a `key` once the dialog form succeeds.

### 4. “Create New” Inside Select
```tsx

  SelectInputUI,
  SelectPlaceholder,

  <SelectInputUI className="w-full">

      <SelectDefaultPlaceholderUI />
    </SelectPlaceholder>
  </SelectInputUI>

  <SelectPopoverContent>
    {/* …existing list of options… */}
  </SelectPopoverContent>
</Select>

<CreateEntityDialog
  trigger={<SelectCreateNewTrigger />}
>
  <CategoryForm />
</CreateEntityDialog>
```
Embed `CreateEntityDialog` next to a `<Select>` so users can add a missing option without leaving the flow.

Pairing `MultiSelect` with `CreateEntityDialog` lets users add new tags on the fly while keeping the interface compact.

## Best Practices & Gotchas
* Keep the dialog form concise; long forms feel cramped in modals.
* Pass a visually distinct trigger button; otherwise users miss the action.
* Handle success inside the child form, not in `CreateEntityDialog`.
* Avoid nesting multiple dialogs—they confuse focus and accessibility.
* When integrating with `<Select>` or `<MultiSelect>`, place the trigger **next to** the input to preserve keyboard navigation.
