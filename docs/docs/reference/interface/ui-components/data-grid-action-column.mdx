# DataGridActionColumn

## Overview
`DataGridActionColumn` renders a column filled with action buttons inside a `DataGridTable`.
You use it to add edit, delete, view, or any custom interactions to each row of the grid.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No Contember-specific theme classes are required; feel free to extend the Tailwind classes as needed.

## Core Concepts
* The column slots **any React nodes**; typically buttons, dropdowns, or dialogs.
* It lives **inside** `DataGridTable` or `DefaultDataGrid`.
* You can place **multiple** action columns in one grid (e.g., leading and trailing).
* Children render **per row**, so keep components lightweight.

## Quick Start
```tsx
import { DataGridTable, DataGridActionColumn } from '~/lib/datagrid'
import { Button } from '~/components/ui/button'

export default function Example() {
  return (
    <DataGridTable className="w-full">
      <DataGridActionColumn>
        <Button size="sm" className="px-2 py-1">Edit</Button>
      </DataGridActionColumn>
    </DataGridTable>
  )
}
```

## Detailed Guide
### Adding multiple actions
```tsx
<DataGridActionColumn>
  <div className="flex gap-2">
    <Button size="sm" variant="secondary">View</Button>
    <Button size="sm" variant="destructive">Delete</Button>
  </div>
</DataGridActionColumn>
```

### Combining with dialogs
```tsx
<DataGridActionColumn>
  <EntityEditDialog entityName="Article" />
</DataGridActionColumn>
```
The dialog opens in the context of the current row, giving users an inline editing flow without page navigation.

## Examples
### Action dropdown with destructive option
```tsx
<DataGridActionColumn>
  <GridDropdown />
</DataGridActionColumn>
```
`GridDropdown` groups edit and delete actions; the destructive option shows in red for clarity.

### Inline preview link
```tsx
<DataGridActionColumn>
  <Link to="article/detail(id: $entity.id)">
    <AnchorButton size="sm" className="flex items-center gap-1">
      <EyeIcon size={16} /> View
    </AnchorButton>
  </Link>
</DataGridActionColumn>
```
Each row gets a “View” button that navigates to a detail screen.

## Best Practices & Gotchas
* Keep children lightweight; they render once per row.
* Wrap long lists of buttons in a dropdown to save horizontal space.
* Use `size="sm"` on buttons to prevent row height jumps.
* Place action columns at the **start** or **end** of the grid for easy scanning.
* Remember to memoize expensive children if they perform heavy work.
