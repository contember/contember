# DataGridActiveFilterUI

## Overview
DataGridActiveFilterUI renders a compact button inside a filter list.
Clicking the button removes the currently active filter from a Contember DataGrid.

## Styling Guidelines
* All snippets use Tailwind for utility-first styling.
* The component adds no Contember-specific theme classes by default; style it with `className` like any other React element.

## Core Concepts
* Filter lists can show multiple active filters.
* DataGridActiveFilterUI should sit inside `DataGridSingleFilterUI` or a similar wrapper.
* Clicking the button must trigger your filter-clearing logic (e.g., `setOrderBy(..., 'clear')`).
* The component accepts `children`—usually the filter’s label.
* You can forward standard HTML `<button>` props such as `className`, `onClick`, or `disabled`.

## Quick Start
```tsx
import { DataGridActiveFilterUI, DataGridSingleFilterUI } from '~/lib/datagrid'

export const QuickStart = () => (
  <DataGridSingleFilterUI>
    {/* Clicking clears the filter set elsewhere in your code */}
    <DataGridActiveFilterUI className="text-xs px-2 py-1 bg-gray-100 rounded">
      Remove
    </DataGridActiveFilterUI>
  </DataGridSingleFilterUI>
)
```

## Examples

### Sorting Dropdown With Clear Action
This pattern adds a “sort by” dropdown and shows DataGridActiveFilterUI only when a sort is active.

```tsx
const SortingDropdown = ({ options }: DataGridSortingDropdownProps) => {
  const sortingState = useDataViewSortingState()
  const sortingMethods = useDataViewSortingMethods()

  const firstDirection = Object.entries(sortingState.directions)[0] as
    | [string, 'asc' | 'desc']
    | undefined

  const activeOption = firstDirection
    ? options.find(opt => opt.field === firstDirection[0] && opt.direction === firstDirection[1])
    : undefined

  return (
    <DataGridSingleFilterUI>
      {/* Trigger controlling the orderBy state */}
      <DropdownMenu>
        {/* …menu omitted for brevity… */}
      </DropdownMenu>

      {/* Show the “clear” button when a sort is active */}
      {activeOption && (
        <DataGridActiveFilterUI
          onClick={() => sortingMethods.setOrderBy(activeOption.field, 'clear', false)}
          className="inline-flex items-center gap-1 text-xs underline"
        >
          {activeOption.label}
        </DataGridActiveFilterUI>
      )}
    </DataGridSingleFilterUI>
  )
}
```

Explanation
* The component gets rendered only if `activeOption` exists.
* Clicking the button calls `setOrderBy` with `'clear'`, removing the sort.
* Tailwind classes keep the button small and consistent with other filters.

## Best Practices & Gotchas
* Keep the button’s footprint small; filters often live in dense toolbars.
* Always wrap the component in a parent that supplies the clearing callback.
* Pass `disabled` when asynchronous work is in progress to avoid double clicks.
* Keep labels short (“Date”, “Status”) to prevent line wrapping.
