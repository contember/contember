## Overview
`DataGridAutoExport` renders a button that exports the current data-grid content to a CSV file.
When you omit explicit field configuration, the component exports **all** data-grid fields automatically.

## Styling Guidelines
* All code samples use Tailwind CSS classes.
* No Contember theme classes are required.

## Core Concepts
* `DataGridAutoExport` must be rendered inside a Contember `DataGrid` context.
* Clicking the button triggers a CSV file download.
* Omitting a field list exports every visible data-grid column.

## Quick Start
```tsx
import { DataGridAutoExport } from '@contember/react'

<DataGridAutoExport className="px-3 py-2 bg-blue-600 text-white rounded" />
```

## Examples
### Basic export button inside a data grid
```tsx
import { DataGrid, DataGridAutoExport } from '@contember/react'

<DataGrid entities="Article">
	<DataGrid.Column field="title" header="Title" />
	<DataGrid.Column field="createdAt" header="Created" />
	<div className="mt-2">
		<DataGridAutoExport className="px-2 py-1 border rounded text-sm" />
	</div>
</DataGrid>
```
The button exports both `title` and `createdAt` columns because no fields are specified.

## Best Practices & Gotchas
* Place the button near the grid for clear user context.
* Keep the label short, e.g., “Export CSV”.
* Avoid hiding essential columns; hidden columns will still export unless you filter them out beforehand.
* Large datasets may take noticeable time—use a loading indicator if possible.
