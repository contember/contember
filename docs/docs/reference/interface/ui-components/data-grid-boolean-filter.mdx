# DataGridBooleanFilter

## Overview
`DataGridBooleanFilter` adds a ready-made boolean filter to a `DataGrid`. It renders a small UI that lets users choose between *Yes* or *No* values for the specified entity field. You supply the field and an optional label; the component handles query generation and state management for you.

Use the filter alongside other `DataGrid*Filter` components to build powerful, composable search experiences in Contember projects.

## Styling Guidelines
* All code samples use Tailwind CSS classes.
* The default Contember theme already styles filter controls—no extra classes are required.

## Core Concepts
* The filter targets a single boolean column in your entity (`field` prop).
* It integrates with `DataGrid`’s internal filtering pipeline.
* You may place multiple filters anywhere in your layout; their state is automatically combined.
* The `label` prop overrides the auto-generated label.

## Quick Start
```tsx
import { DefaultDataGrid, DataGridBooleanFilter } from '~/lib/datagrid'

export const ProjectsTable = () => (
  <DefaultDataGrid entities="Project">
    <DataGridBooleanFilter field="locked" label="Locked" />
  </DefaultDataGrid>
)
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| label | `React.ReactNode` | Yes | — | Text shown next to the filter control. |
| **Inherited** | | | | |
| field | `string` | Yes | — | Entity field to filter. Provided by the base `DataGridFilter` props. |

## Examples

### Popover with multiple filters
```tsx
import { Button, Popover, PopoverTrigger, PopoverContent } from '~/components/ui'
import {
  DataGridQueryFilter,
  DataGridBooleanFilter,
  DataGridDateFilter,
  createDataGridDateRange,
} from '~/lib/datagrid'

export const ArticlesTableFilters = () => (
  <>
    <DataGridQueryFilter />
    <Popover>
      <PopoverTrigger asChild>
        <Button variant="outline" className="h-auto">More filters</Button>
      </PopoverTrigger>
      <PopoverContent>
        <div className="flex flex-col gap-2">
          <DataGridBooleanFilter field="locked" label="Locked" />
          <DataGridDateFilter
            field="publishedAt"
            label="Published at"
            ranges={[
              createDataGridDateRange('Today', 0, 0),
              createDataGridDateRange('Last 7 days', -7, 0),
            ]}
          />
        </div>
      </PopoverContent>
    </Popover>
  </>
)
```
Placing the boolean filter in a popover keeps the toolbar clean while still offering granular control.

## Best Practices & Gotchas
* Always pass `field`; omit it only when inheriting from typed wrappers.
* Combine boolean filters with a `DataGridQueryFilter` to let users search and refine results simultaneously.
* Place filters outside the `DataGridTable` to avoid re-mounting on pagination changes.
* Keep labels short; long text can push the filter dropdown past viewport edges.
