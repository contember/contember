# DataGridColumnHeader

## Overview
`DataGridColumnHeader` renders a single column header inside a data grid.
It bundles filtering, sorting, and column-hiding controls into one reusable component.

## Styling Guidelines
* All examples use Tailwind classes.
* No Contember theme classes are required.

## Core Concepts
* Each header can sort rows by one `sortingField`.
* `filter` renders any React node that mutates grid filters.
* `hidingName` toggles column visibility in user settings.
* Children define the visible title and can include Tailwind styling.

## Quick Start
```tsx
<DataGridColumnHeader sortingField="name">
  <span className="font-semibold">Name</span>
</DataGridColumnHeader>
```

## Props Reference
| Name         | Type                | Required | Default | Description                                                                 |
|--------------|---------------------|----------|---------|-----------------------------------------------------------------------------|
| sortingField | string              | No       | –       | Field name used when the header toggles sorting.                            |
| hidingName   | string              | No       | –       | Unique key that identifies the column in user-controlled hiding settings.   |
| filterName   | string              | No       | –       | Key that groups filters belonging to this column.                           |
| filter       | React.ReactNode     | No       | –       | React element rendered inside a filter pop-over or inline filter area.      |
| children     | React.ReactNode     | Yes      | –       | Visible header content (text, icons, etc.).                                 |

## Examples

### 1. Custom Filter Input
Add a simple text filter that updates the grid when the user types.

```tsx
<DataGridColumnHeader
  filterName="name"
  filter={<input className="border p-1 w-full" placeholder="Search name…" />}
>
  <span className="flex gap-1 items-center">
    <svg className="w-4 h-4" /* icon markup */ />
    Name
  </span>
</DataGridColumnHeader>
```
The `filter` prop receives a plain `<input>`. The grid implementation should read `filterName` and apply the entered value.

### 2. Hide-able Date Column With Sorting
```tsx
<DataGridColumnHeader
  sortingField="createdAt"
  hidingName="createdAt"
>
  <span className="flex gap-1 items-center">
    <svg className="w-4 h-4" /* calendar icon */ />
    Created
  </span>
</DataGridColumnHeader>
```
Users can click the header to sort by `createdAt` and use UI controls (provided elsewhere) to hide the column.

## Best Practices & Gotchas
* Always pass `children`; the component renders nothing without it.
* Use stable strings for `hidingName` and `filterName` to avoid losing user preferences.
* Do not pass Tailwind classes directly to `DataGridColumnHeader`; style inside `children`.
* Keep `sortingField` values in sync with server-side query names.
* Provide accessible labels inside `children`; assistive tech does not read icons alone.
