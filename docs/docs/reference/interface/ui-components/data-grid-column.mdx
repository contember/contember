# DataGridColumn

## Overview
`DataGridColumn` is a low-level building block for custom cells and headers in a Contember data grid.
Use it when the higher-level typed columns (Text, Enum, Date…) do not fit your needs.

## Styling Guidelines
* All examples below use Tailwind CSS utility classes.
* No special Contember theme classes are required; add your own with `headerClassName` or `cellClassName`.

## Core Concepts
* Wraps custom markup displayed in every cell.
* Accepts an optional `header` for the column title.
* Connect sorting via `sortingField`.
* Provide a `filter` component to enable column filtering.
* Control visibility toggles with `hidingName`.
* Style header or body separately via `headerClassName` and `cellClassName`.

## Quick Start
```tsx
<DataGrid entities="Article">
  <DataGridTable>
    <DataGridColumn
      header="Title"
      headerClassName="font-semibold"
      cellClassName="px-2"
    >
      <Field field="title" />
    </DataGridColumn>
  </DataGridTable>
  <DataGridPagination />
</DataGrid>
```

## Detailed Guide
### Sorting
```tsx
<DataGridColumn
  header="Created"
  sortingField="createdAt"
>
  <FormattedField field="createdAt" />
</DataGridColumn>
```

### Custom Filter
```tsx
<DataGridColumn
  header="Tags"
  filterName="tags"
  filter={
    <DataGridHasManyFilter field="tags" name="tags" />
  }
>
  <DecoratedHasMany field="tags" />
</DataGridColumn>
```

## Props Reference

| Name | Type | Required | Default | Description |
| ---- | ---- | -------- | ------- | ----------- |
| children | React.ReactNode | Yes | – | Content rendered for every row. |
| header | React.ReactNode | No | – | Column header. |
| name | string | No | – | Unique column identifier. |
| hidingName | string | No | – | Label used in the column-visibility menu. |
| sortingField | string | No | – | Entity field used for sorting. |
| cellClassName | string | No | – | Tailwind classes for `<td>`. |
| headerClassName | string | No | – | Tailwind classes for `<th>`. |
| filter | React.ReactNode | No | – | Custom filter UI. |
| filterName | string | No | – | URL/query-string key for the filter value. |

## Examples
### Linked actions column
```tsx
<DataGridColumn header="Actions" headerClassName="w-0">
  <UseEntity render={it => (
    <Button size="sm" onClick={() => openDetail(it)}>Open</Button>
  )} />
</DataGridColumn>
```
Use `headerClassName="w-0"` to keep the header narrow and action buttons compact.

### Tooltip list
```tsx
<DataGridColumn header="Authors">
  <div className="flex flex-wrap gap-2">
    <HasMany field="comments">
      <HasOne field="author">
        <DataGridHasManyTooltip field="comments.author">
          <span className="cursor-pointer border-b border-dashed">
            <Field field="name" />
          </span>
        </DataGridHasManyTooltip>
      </HasOne>
    </HasMany>
  </div>
</DataGridColumn>
```
Display nested relations and keep rows tidy by wrapping authors in a tooltip.

## Best Practices & Gotchas
* Prefer a dedicated typed column when possible; fall back to `DataGridColumn` for complex layouts.
* Always set `sortingField` to avoid ambiguous server sorting requests.
* Add `name` when you use multiple dynamic columns; it helps with React keys.
* Keep cell content lightweight to prevent slow renders on large datasets.
* Use Tailwind’s `w-0` on headers that should not consume width.
