# DataGridDateFilter

## Overview
`DataGridDateFilter` adds a date-range filter to a `DataGrid`.  
It renders a ready-made UI that lets users narrow results by absolute or predefined ranges.

## Styling Guidelines
* All examples use Tailwind classes.  
* The component itself relies only on Contember default theme variables—no extra classes are required.

## Core Concepts
* A filter binds to one entity field through the `field` prop (from `DataGridFilter`, inherited).  
* You must supply a visible `label`.  
* Optional `ranges` let you offer quick-select shortcuts (e.g., “Last 7 days”).  
* The filter works inside a `<DataGrid>` along with other filter components.  
* You can create ranges programmatically with `createDataGridDateRange`.

## Quick Start
```tsx
import { DataGrid, DataGridDateFilter } from '~/lib/datagrid'

export default () => (
	<DataGrid entities="Post">
		<DataGridDateFilter
			field="createdAt"
			label="Created at"
			className="w-full sm:w-64"
		/>
	</DataGrid>
)
```

## Detailed Guide
### Adding predefined ranges
```tsx
import { DataGrid, DataGridDateFilter, createDataGridDateRange } from '~/lib/datagrid'

const ranges = [
	createDataGridDateRange('Today', 0, 0),
	createDataGridDateRange('Last 7 days', -7, 0),
	createDataGridDateRange('Last 30 days', -30, 0),
]

export default () => (
	<DataGrid entities="Article">
		<DataGridDateFilter
			field="publishedAt"
			label="Published at"
			ranges={ranges}
			className="w-full sm:w-72"
		/>
	</DataGrid>
)
```

## Sub-Components / Building Blocks
### createDataGridDateRange
Utility that returns a `DataGridPredefinedDateRange` object from a label and offset values. Use it to avoid manual `Date` math.

## Examples
### Filtering blog posts by publish date
```tsx
import {
	DataGrid,
	DataGridDateFilter,
	DataGridTextColumn,
	DataGridDateColumn,
	createDataGridDateRange,
} from '~/lib/datagrid'

const dateRanges = [
	createDataGridDateRange('This week', -7, 0),
	createDataGridDateRange('Last month', -30, 0),
]

export default () => (
	<DataGrid entities="Post" className="space-y-4">
		<DataGridDateFilter
			field="publishedAt"
			label="Published at"
			ranges={dateRanges}
			className="w-full md:w-60"
		/>
		<DataGridDateColumn field="publishedAt" header="Published" />
		<DataGridTextColumn field="title" header="Title" />
	</DataGrid>
)
```
The filter limits the dataset before columns render, so pagination and totals reflect the selection.

## Best Practices & Gotchas
* Always pass `label`; without it the filter shows no caption.  
* Keep predefined ranges short and relevant; too many overwhelm users.  
* Place filters in a logical order—field filters first, then boolean or enum filters.  
* Remember to memoize `ranges` arrays in large grids to avoid re-renders.  
* The filter uses local time. Handle timezone issues on the API level if necessary.
