# DataGridEnumFilterControls

## Overview
`DataGridEnumFilterControls` is an internal Contember component. It renders filter controls for enum-typed columns inside a DataGrid.
Because it is marked **@internal**, the public API can change without notice.

## Styling Guidelines
* All snippets below use Tailwind utility classes.
* No special Contember theme classes are required.

## Core Concepts
* Internal components can break on major or minor releases.
* Enum filters let users limit DataGrid rows to specific enum values.
* You should place filter controls inside a layout container with `flex` or `grid` Tailwind classes.

## Quick Start
```tsx
import { DataGridEnumFilterControls } from '@contember/react-datagrid'

export default function Filters() {
  return (
    <div className="p-4 bg-gray-50 rounded">
      <DataGridEnumFilterControls />
    </div>
  )
}
```

## Examples

### Sidebar filter panel
A common pattern is a sidebar that hosts all filters while the grid remains visible.

```tsx
import { DataGridEnumFilterControls } from '@contember/react-datagrid'

export default function Page() {
  return (
    <aside className="w-64 border-r p-6 space-y-4">
      <h2 className="text-lg font-semibold">Filters</h2>
      <DataGridEnumFilterControls />
    </aside>
  )
}
```

The `aside` element uses Tailwind classes to create a fixed-width sidebar with padding and vertical spacing.

## Best Practices & Gotchas
* Use this component only when you target a Contember version you control.
* Wrap it in semantic containers (`aside`, `section`) for better accessibility.
* Keep enum value counts reasonable; large enums create unwieldy filter UIs.
* Test upgrades carefully because internal APIs can change silently.
