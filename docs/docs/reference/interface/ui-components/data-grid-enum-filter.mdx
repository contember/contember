## Overview
`DataGridEnumFilter` renders a dropdown filter for enum fields inside a `DataGrid`.
You can let Contember infer enum values or pass custom `options` for finer control.

## Styling Guidelines
* All examples use Tailwind for layout and spacing.
* No Contember-specific theme classes are required; the component uses Contember defaults.

## Core Concepts
* Enum values map to filter predicates on a single field.
* The filter works only inside a `DataGrid` context.
* Provide `options` when the backend does not expose enum metadata.
* Combine multiple filters (enum, date, boolean) to build advanced search UIs.

## Quick Start
```tsx
import { DataGridEnumFilter, DataGrid } from '~/lib/datagrid'

export default () => (
  <DataGrid>
    <DataGridEnumFilter field="status" className="mb-4" />
    {/* columns … */}
  </DataGrid>
)
```

## Detailed Guide
```tsx
// 1️⃣ Show three explicit states
<DataGridEnumFilter
  field="state"
  options={[
    { value: 'draft',   label: 'Draft' },
    { value: 'review',  label: 'In review' },
    { value: 'published', label: 'Published' },
  ]}
  className="w-40"
/>

// 2️⃣ Combine with other filters
<>
  <DataGridEnumFilter field="state" className="mr-2" />
  <DataGridDateFilter field="publishedAt" />
</>

// 3️⃣ Place filters in a Popover to save space
<Popover>
  <PopoverTrigger asChild>
    <Button className="h-auto" variant="outline">More filters</Button>
  </PopoverTrigger>
  <PopoverContent className="flex flex-col gap-2">
    <DataGridEnumFilter field="category" label="Category" />
    <DataGridEnumFilter field="priority" label="Priority" />
  </PopoverContent>
</Popover>
```

## Examples
### Custom label and options
```tsx
<DataGridEnumFilter
  field="visibility"
  label="Visibility"
  options={{
    public: 'Public',
    private: 'Private',
    archived: 'Archived',
  }}
  className="w-48"
/>
```
The filter shows “Visibility” and lets users pick one of three states.

### Nested field filtering
```tsx
<DataGridEnumFilter field="details.status" />
```
You can target nested fields by dot-notation. Contember translates the path automatically.

### Real-world mix
```tsx
<>
  <DataGridQueryFilter />
  <DataGridEnumFilter field="details.eduStatus" options={educationStatusOptions()} />
  <DataGridBooleanFilter field="active" />
</>
```
The enum filter sits beside a query and boolean filter to provide a comprehensive search bar.

## Best Practices & Gotchas
* Keep enum option keys in sync with backend values.
* Use short labels; long texts make select boxes unwieldy.
* Wrap multiple filters in a responsive container using `flex flex-wrap gap-2`.
* Do not render the filter outside a `DataGrid`; it relies on context.
