## Overview
`DataGridEnumListFilter` lets you add a multi-select enum filter to any Contember `DataGrid`. You configure it with predefined enum values or let it read them from the entity schema.
The component wraps `DataViewEnumListFilter` and shows the UI through `DataGridSingleFilterUI`, so you get a consistent look without extra code.

## Styling Guidelines
All examples below use Tailwind CSS.
The filter relies on Contember’s default theme classes shipped with `@contember/ui`, so you normally add no extra classes.

## Core Concepts
- Multi-select filtering on enum columns.
- Works only inside a `DataGrid` context.
- Accepts optional `options` for custom value–label pairs.
- Falls back to the field’s schema label when `label` is undefined.
- Uses the same filter layout as other `DataGrid*Filter` components.

## Quick Start
```tsx
<div className="p-4">
  <DataGridEnumListFilter field="status" />
</div>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| options | Record&lt;string, React.ReactNode&gt; | No | – | Predefined value–label pairs shown in the dropdown. |
| label | React.ReactNode | No | Field label | Human-readable label shown above the filter. |
| *(inherited)* | See `DataViewEnumListFilter` | No | – | All props supported by the underlying filter component. |

## Sub-Components / Building Blocks
### DataViewEnumListFilter
Wraps the low-level filtering logic and exposes `options`, `label`, and `field`.

### DataGridSingleFilterUI
Renders the filter inside a standardized `DataGrid` panel.

## Examples
### Basic enum filter
```tsx
<DataGridEnumListFilter field="status" />
```
This adds a filter with all enum values defined in the `status` field schema.

### Custom enum options
```tsx
<DataGridEnumListFilter
  field="status"
  options={{
    active: 'Active',
    inactive: 'Inactive',
    pending: 'Pending',
  }}
/>
```
You override the default schema values and provide custom labels.

## Best Practices & Gotchas
- Keep the number of enum options small for better UX.
- Use `options` only when you need to rename values or hide some of them.
- Place all filters at the top of your `DataGrid` for consistency.
- Do not pass uncontrolled props; stick to the documented API.
