# DataGridFilterMobileHiding

## Overview
`DataGridFilterMobileHiding` conditionally hides or shows its filter children.
Filters stay visible when they are active or when `DataGridShowFiltersContext` equals `true`.

## Styling Guidelines
* All code samples use Tailwind CSS.
* No special Contember theme classes are required.

## Core Concepts
* Filters auto-hide on mobile until activated.
* The component checks each child’s active state.
* Visibility also depends on `DataGridShowFiltersContext`.
* You can target a single filter via the optional `name` prop.

## Quick Start
```tsx
import { DataGridFilterMobileHiding, DataGridTextFilter } from '@contember/admin';

<DataGridFilterMobileHiding>
  <DataGridTextFilter field="name" />
</DataGridFilterMobileHiding>
```

## Props Reference
| Name     | Type        | Required | Default | Description                                                   |
|----------|-------------|----------|---------|---------------------------------------------------------------|
| name     | string      | No       | —       | Filter name to watch. Only the matching filter is evaluated.  |
| children | ReactNode   | Yes      | —       | One or more filter components. *(inherited from React)*       |

## Examples

### Show a specific enum filter
```tsx
<DataGridFilterMobileHiding name="status">
  <DataGridEnumFilter field="status" />
</DataGridFilterMobileHiding>
```
This setup hides the `status` enum filter on small screens until the user activates it or toggles filters globally.

### Multiple filters, automatic visibility
```tsx
<DataGridFilterMobileHiding>
  <DataGridTextFilter field="name" />
  <DataGridEnumFilter field="status" />
</DataGridFilterMobileHiding>
```
Each filter controls its own visibility. Active filters remain visible even after page refresh, improving user experience on mobile.

## Best Practices & Gotchas
* Wrap only filter components—do not nest unrelated elements.
* Avoid heavy layouts inside the wrapper to keep mobile rendering fast.
* Use consistent `name` values across filters and their labels.
* Combine with a global “Show filters” button that writes to `DataGridShowFiltersContext`.
