# DataGridFilterSelectTriggerUI

## Overview
DataGridFilterSelectTriggerUI renders the clickable element that opens a select-style filter popover for a Contember DataGrid.
You place any React nodes inside to render a custom label or icon.

## Styling Guidelines
* All examples use Tailwind CSS.
* The component relies only on your Tailwind classes; no special Contember theme classes are required.

## Core Concepts
* A trigger must live inside a `PopoverTrigger` wrapper.
* The trigger opens a `SelectPopoverContent` with filter controls.
* You can render text, icons, or both inside the trigger.
* Combine it with `DataViewQueryFilter` to filter entities in a `DataGrid`.
* The trigger itself holds no state; the surrounding popover manages visibility.

## Quick Start
```tsx
import {
	DataGridFilterSelectTriggerUI,
	DataGridFilterSelectItemUI,
} from '@app/lib/datagrid/ui'
import { Popover, PopoverTrigger } from '@contember/ui'

export const StatusFilter = () => (
	<Popover>
		<PopoverTrigger asChild>
			<DataGridFilterSelectTriggerUI className="inline-flex items-center gap-1 px-3 py-1.5 rounded bg-slate-100 hover:bg-slate-200">
				Status
			</DataGridFilterSelectTriggerUI>
		</PopoverTrigger>

		{/* Your SelectPopoverContent goes here */}
	</Popover>
)
```


## Sub-Components / Building Blocks
### Popover
Wrap the trigger with `Popover` + `PopoverTrigger` to handle open/close behavior.

### SelectPopoverContent
Place filter UI here. It appears when the trigger is clicked.

### DataViewQueryFilter
Adds include/exclude logic and sorting to the filter content.

## Examples

### Include / Exclude Filter
```tsx
import {
	DataGridFilterSelectTriggerUI,
	DataGridFilterSelectItemUI,
} from '@app/lib/datagrid/ui'
import { Popover, PopoverTrigger } from '@contember/ui'
import { DataViewQueryFilter } from '@contember/react-dataview'

export const UserRoleFilter = ({
	included,
	excluded,
	toggleInclude,
	toggleExclude,
}) => (
	<Popover>
		<PopoverTrigger asChild>
			<DataGridFilterSelectTriggerUI className="flex items-center gap-2 px-2 py-1 bg-white border rounded shadow-sm">
				User role
			</DataGridFilterSelectTriggerUI>
		</PopoverTrigger>

		<SelectPopoverContent>
			<DataViewQueryFilter
				entityName="Role"
				included={included}
				excluded={excluded}
				toggleInclude={toggleInclude}
				toggleExclude={toggleExclude}
			>
				<DataGridFilterSelectItemUI entityId="admin" label="Admin" />
				<DataGridFilterSelectItemUI entityId="editor" label="Editor" />
				<DataGridFilterSelectItemUI entityId="viewer" label="Viewer" />
			</DataViewQueryFilter>
		</SelectPopoverContent>
	</Popover>
)
```
Use `included` and `excluded` arrays to keep track of the current filter state.

## Best Practices & Gotchas
* Always wrap the trigger in `PopoverTrigger asChild`.
* Provide an accessible label for screen readers.
* Keep the triggerâ€™s interactive area large enough for touch devices.
* Avoid heavy content inside the trigger; keep it simple and fast.
* Manage filter state outside the trigger to keep it stateless.
