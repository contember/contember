# DataGridHasManyTooltip

## Overview
DataGridHasManyTooltip wraps any element and adds a filter-aware tooltip.
Users can quickly include or exclude the related entity from a DataGrid filter by clicking items rendered inside the tooltip.

## Styling Guidelines
* All examples use TailwindCSS classes.
* No special Contember theme classes are required.

## Core Concepts
* Tooltip toggles inclusion/exclusion of a “has many” relation in a DataGrid filter.
* Works inside `<HasMany>` and nested relation helpers.
* Accepts arbitrary children; they become the clickable hotspot.
* Optional `actions` slot lets you add custom buttons in the tooltip.
* When you omit `options`, filtering uses the entity behind `field`.

## Quick Start
```tsx
import { HasMany, Field, DataGridHasManyTooltip } from '~/lib/datagrid'

<HasMany field="tags">
	<DataGridHasManyTooltip field="tags">
		<button className="text-sm border rounded px-2 py-1">
			<Field field="name" />
		</button>
	</DataGridHasManyTooltip>
</HasMany>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| field | string &#124; SugarableRelationObject | Yes | — | The relation field to filter by. |
| children | React.ReactNode | Yes | — | Content rendered inside the clickable area. |
| name | string | No | field value | Custom filter name. |
| options | string &#124; OptionsObject | No | — | Predefined list of entities to show in filter. |
| actions | React.ReactNode | No | — | Extra UI rendered inside the tooltip. |

## Examples

### Nested author filter inside comment list
```tsx
<DataGridColumn header="Comment authors">
	<div className="flex flex-wrap gap-2">
		<HasMany field="comments">
			<HasOne field="author">
				<DataGridHasManyTooltip field="comments.author">
					<span className="cursor-pointer border-b border-dashed">
						<Field field="name" />
					</span>
				</DataGridHasManyTooltip>
			</HasOne>
		</HasMany>
	</div>
</DataGridColumn>
```
This renders every comment’s author name. Hovering shows a tooltip where you can include or exclude that author from the DataGrid filter.

### Using `actions` to add “Clear” button
```tsx
<DataGridHasManyTooltip
	field="tags"
	actions={<button className="text-xs text-red-600">Clear</button>}
>
	<Button size="sm" variant="ghost">
		<Field field="name" />
	</Button>
</DataGridHasManyTooltip>
```
The “Clear” button appears inside the tooltip. Implement its `onClick` to reset filters.

## Best Practices & Gotchas
* Wrap only interactive elements; avoid placing tooltips on large containers.
* Always place the component inside the corresponding `<HasMany>` scope.
* Use descriptive `name` when the same relation appears multiple times.
* Keep `options` in sync with possible entities or omit it entirely.
* Do not nest another tooltip inside the children; that confuses users.
