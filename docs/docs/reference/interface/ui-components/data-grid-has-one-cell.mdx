# DataGridHasOneCell

## Overview
`DataGridHasOneCell` renders a single cell bound to a **has-one** relation.
The cell shows your custom content and offers a tooltip with filter actions.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No Contember theme classes are required.

## Core Concepts
* Works only inside a `DataGrid` row.
* Expects a has-one relation path in `field`.
* Renders its `children` in the related entity context.
* Tooltip can display default or custom `tooltipActions`.
* `filterName` links the cell to a DataGrid filter.

## Quick Start
```tsx
import { DataGridHasOneCell, TextField } from '@contember/react-datagrid'

<DataGridHasOneCell field="author">
  <TextField field="name" className="text-sm font-medium text-gray-900" />
</DataGridHasOneCell>
```

## Props Reference
| Name | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| field | string &#124; SugarableHasOneRelation &#124; SugarableHasOneRelation[] | Yes | — | Has-one field to be displayed. |
| filterName | string | No | — | Filter identifier. If omitted, resolved from `field`. |
| children | React.ReactNode | Yes | — | Cell content rendered in the relation context. |
| tooltipActions | React.ReactNode | No | — | Custom actions shown in the tooltip. |

## Examples

### 1. Custom Tooltip Action
```tsx
<DataGridHasOneCell
  field="author"
  tooltipActions={
    <button className="px-2 py-1 text-xs text-blue-600 hover:underline">
      View profile
    </button>
  }
>
  <TextField field="name" className="text-sm" />
</DataGridHasOneCell>
```
Adds a link-like button inside the tooltip while still showing the author’s name in the cell.

### 2. Overriding Filter Name
```tsx
<DataGridHasOneCell field="author" filterName="authorFilter">
  <TextField field="email" className="text-sm text-gray-700" />
</DataGridHasOneCell>
```
You set a custom `filterName` to decouple the filter key from the relation path.

## Best Practices & Gotchas
* Place the component only inside a `DataGrid` column renderer.
* Keep tooltip actions lightweight; long lists slow down rendering.
* Reuse `filterName` across multiple cells to keep filters consistent.
* Do not pass array relations (`hasMany`) into `field`; the component handles only **has-one** paths.
