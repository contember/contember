# DataGridHasOneColumn

## Overview
`DataGridHasOneColumn` renders the content of a single has-one relation inside a `DataGridTable`.  
You supply the relation via the `field` prop and render any nested fields through `children`.

## Styling Guidelines
* All snippets use Tailwind for layout and spacing.  
* The component itself adds no classes; you can still add Contember theme utilities to the cells.

## Core Concepts
* Has-one relations map to exactly one related entity.
* Use inside `DataGridTable`; the grid supplies row context.
* The `field` prop accepts a relation path (`author`, `module.lecturer`, …).
* Put display components (e.g., `<Field>`) inside `children`.
* Override headers, filters, and tooltip actions with the dedicated props.

## Quick Start
```tsx
<DataGridTable className="w-full">
	<DataGridHasOneColumn field="author">
		<Field field="name" className="text-sm font-medium" />
	</DataGridHasOneColumn>
</DataGridTable>
```

## Detailed Guide
### Custom header and filter
```tsx
<DataGridHasOneColumn
	field="author"
	header="Author"
	filter={<DataGridHasOneFilter field="author" />}
>
	<Field field="name" />
</DataGridHasOneColumn>
```
* `header` overrides the auto-generated label.  
* `filter` accepts any React node; here we reuse `DataGridHasOneFilter`.

### Tooltip actions
```tsx
<DataGridHasOneColumn
	field="author"
	tooltipActions={<Button size="xs">Open profile</Button>}
>
	<Field field="name" />
</DataGridHasOneColumn>
```
* Place quick actions next to the rendered name.  
* Keep tooltips concise to avoid clutter.

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| field | string &#124; SugarableHasOneRelation &#124; SugarableHasOneRelation[] | Yes | — | Has-one relation field. |
| filterName | string | No | — | Identifier of the filter. Defaults to the field path. |
| children | React.ReactNode | Yes | — | Cell content rendered in the relation context. |
| header | React.ReactNode | No | — | Custom column header. |
| filter | React.ReactNode | No | — | Custom filter element. |
| tooltipActions | React.ReactNode | No | — | Extra actions displayed in the cell tooltip. |

## Sub-Components / Building Blocks
### DataGridHasOneFilter
Displays a select input that filters the grid by a has-one relation.

### DataGridHasOneTooltip
Shows related entity details or actions inside a tooltip when hovering a cell.

## Examples
### Basic listing with author and category
```tsx
<DataGridTable className="w-full">
	<DataGridHasOneColumn header="Author" field="author">
		<Field field="name" />
	</DataGridHasOneColumn>

	<DataGridHasOneColumn header="Category" field="category">
		<Field field="name" />
	</DataGridHasOneColumn>
</DataGridTable>
```
Each column renders the related entity’s name and provides automatic filtering.

### Nested relation
```tsx
<DataGridHasOneColumn field="module.lecturer">
	<LecturerPreview />
</DataGridHasOneColumn>
```
You can traverse deeper relations by dot-separating the path (`module.lecturer`).

## Best Practices & Gotchas
* Keep cell content small; heavy components slow the grid.
* Always wrap the component in `DataGridTable`.
* Use `filterName` to avoid collisions when two columns share a field.
* Provide `header` when the auto label is unclear.
* Remember: Tailwind classes must be applied to your `children`, not the column itself.
