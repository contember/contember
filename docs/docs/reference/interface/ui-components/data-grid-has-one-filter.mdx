## Overview
`DataGridHasOneFilter` renders a complete “has-one” filter for a Contember `DataGrid`.  
It connects a single entity relation (e.g. `author`, `category`) to user-friendly UI generated from the `children` you pass in.

The component ships with sensible defaults, so you only specify the relation field, a label, and a preview of the related entity.

## Styling Guidelines
* All snippets use Tailwind CSS classes.  
* The component itself adds no visual style; wrap it in your own Tailwind utility classes or rely on your global Contember theme.

## Core Concepts
* You filter by a single has-one relation (one-to-one or many-to-one).
* `children` defines how each option appears (typically with the `Field` component).
* `options` lets you limit selectable entities or remap the relation path.
* Omit `name` to auto-generate it from `field`.

## Quick Start
```tsx
import { DataGrid, DataGridHasOneFilter } from '~/lib/datagrid'
import { Field } from '@contember/react-binding'

<DataGrid>
  <div className="flex gap-2">
    <DataGridHasOneFilter field="author" label="Author">
      <Field field="name" />
    </DataGridHasOneFilter>
  </div>
</DataGrid>
```

## Detailed Guide
### Restrict selectable entities
```tsx
<DataGridHasOneFilter
  field="author"
  label="Author (active only)"
  options={{
    entityName: 'Author',
    filter: { active: true },
  }}
>
  <Field field="name" />
</DataGridHasOneFilter>
```
The `options.filter` limits the dropdown to active authors only.

### Custom filter name
```tsx
<DataGridHasOneFilter
  field="category"
  name="postCategory"
  label="Category"
>
  <Field field="name" />
</DataGridHasOneFilter>
```
Setting `name` avoids collisions when multiple filters reference the same relation.

## Props Reference
Name | Type | Required | Default | Description
-----|------|----------|---------|------------
field | string &#124; SugarableHasOneRelation &#124; SugarableHasOneRelation[] | Yes | — | Relation field to filter by.
label | React.ReactNode | Yes | — | UI label shown to the user.
children | React.ReactNode | Yes | — | Rendered inside each option (usually a `Field`).
name | string | No | field value | Custom filter name (helps with duplicates).
options | string &#124; { filter?: SugaredFilter; hasOneRelationPath?: SugarableHasOneRelation &#124; SugarableHasOneRelation[]; entityName: string } | No | — | Constrain or remap selectable entities.

## Examples
### Popover-based filter group
```tsx
import { Popover, PopoverTrigger, PopoverContent } from '~/ui/popover'

<Popover>
  <PopoverTrigger asChild>
    <button className="btn btn-outline">More filters</button>
  </PopoverTrigger>
  <PopoverContent className="flex flex-col gap-2">
    <DataGridHasOneFilter field="category" label="Category">
      <Field field="name" />
    </DataGridHasOneFilter>

    <DataGridHasOneFilter
      field="author"
      label="Author (inactive)"
      options={{ entityName: 'Author', filter: { active: false } }}
    >
      <Field field="name" />
    </DataGridHasOneFilter>
  </PopoverContent>
</Popover>
```
The popover groups multiple relation filters without crowding the toolbar.

### Inline toolbar usage
```tsx
<div className="flex flex-wrap gap-2 items-center">
  <DataGridQueryFilter />
  <DataGridHasOneFilter field="locale" label="Locale">
    <Field field="code" />
  </DataGridHasOneFilter>
</div>
```
Combine a text query filter with a relation filter for quick searches.

## Best Practices & Gotchas
* Always render `children`; without it users see empty rows.
* Give each filter a unique `name` when the same relation appears twice.
* Wrap filters in a flex container to avoid layout breaks.
* Large datasets need pagination or server-side filtering in `options.filter`.
