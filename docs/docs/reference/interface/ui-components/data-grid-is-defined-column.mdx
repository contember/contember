## Overview
`DataGridIsDefinedColumn` renders a “value present?” column inside a `DataGridTable`.  
It shows either a checkmark or a cross, helping you spot missing data quickly.

Use it when you only care whether a field **exists**, not what it contains. The component supports custom headers, cell renderers, and filters.

## Styling Guidelines
* All examples use Tailwind CSS.  
* No special Contember theme classes are required.

## Core Concepts
* “Is-defined” means the field is **not null**.
* Works only inside `DataGridTable`.
* Default cell shows ✔️ or ✖️.  
* You can override header, cell content, and filter.
* `field` accepts dotted or nested paths (e.g. `coverImage.url`).

## Quick Start
```tsx
import {
	DataGrid,
	DataGridLoader,
	DataGridTable,
	DataGridToolbar,
	DataGridIsDefinedColumn,
} from '~/lib/datagrid'

export default () => (
	<DataGrid entities="Post">
		<DataGridToolbar className="mb-2" />

		<DataGridLoader>
			<DataGridTable className="w-full text-sm">
				<DataGridIsDefinedColumn field="coverImage.url" />
			</DataGridTable>
		</DataGridLoader>
	</DataGrid>
)
```

## Detailed Guide
### Custom header and formatter
```tsx
<DataGridIsDefinedColumn
	field="publishedAt"
	header="Published?"
	format={value => (value ? '✅ Yes' : '❌ No')}
/>
```

### Replacing the default filter
```tsx
<DataGridIsDefinedColumn
	field="archivedAt"
	filterName="archivedState"
	filter={<DataGridIsDefinedFilter field="archivedAt" name="archivedState" />}
/>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| field | string | Yes | – | Displayed field. |
| header | React.ReactNode | No | auto-generated | Custom header label. |
| children | React.ReactNode | No | checkmark / cross | Custom cell content. |
| format | (value: boolean) => React.ReactNode | No | internal formatter | Custom value formatter. |
| filter | React.ReactNode | No | default is-defined filter | Custom filter element. |
| filterName | string | No | field + `__defined` | URL query key for the filter. |

_Inherited props: None_

## Sub-Components / Building Blocks
### DataGridIsDefinedFilter
Default filter used when you omit the `filter` prop.  
It offers “Yes / No” radio buttons to filter rows.

## Examples
### Show whether a product has an image
```tsx
<DataGridIsDefinedColumn
	field="image.url"
	header="Has Image"
	filterName="hasImage"
/>
```
Displays ✔️ for rows with any image URL.

### Hide values but keep filter
```tsx
<DataGridIsDefinedColumn
	field="email"
	header="Email available"
	children={null}          // hide icon
	filter={<DataGridIsDefinedFilter field="email" />} 
/>
```
You only expose the filter to users; cells stay empty.

## Best Practices & Gotchas
* Always set `filterName` when you need stable URL parameters.
* Prefer the default formatter; keep icons consistent across the app.
* Combine with `hidden` prop on other columns to create compact grids.
* Remember: `field` path must resolve to a scalar, not a relation.
