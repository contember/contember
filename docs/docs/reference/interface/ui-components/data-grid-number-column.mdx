# DataGridNumberColumn

## Overview
`DataGridNumberColumn` renders a numeric column inside a `DataGridTable`.
Use it when you need to display numbers, optionally formatted or enriched with custom cell content.

The component accepts `DataGridNumberColumnProps` and fits seamlessly into any Contember data-grid layout.

## Styling Guidelines
* All examples use Tailwind for layout and spacing.
* The component itself adds no visual styles; rely on your global Contember theme or Tailwind utility classes.

## Core Concepts
* Map a database field to a numeric column with `field`.
* Override the column header with `header`.
* Format raw numeric values through `format`.
* Replace the default cell renderer with `children`.
* Swap the default number filter via the `filter` prop.

## Quick Start
```tsx
import { DataGridTable, DataGridNumberColumn } from '~/lib/datagrid'

<DataGridTable className="w-full border border-gray-200">
	<DataGridNumberColumn
		field="price"
		format={value => value.toFixed(2)}
		className="text-right"
	/>
</DataGridTable>
```

## Detailed Guide
```tsx
import { DataGridTable, DataGridNumberColumn, Field } from '~/lib/datagrid'

<DataGridTable className="w-full">
	{/* 1. Simple number with thousands separator */}
	<DataGridNumberColumn
		header="Views"
		field="views"
		format={v => v.toLocaleString()}
	/>

	{/* 2. Combine two fields in one cell */}
	<DataGridNumberColumn field="price" header="Price with currency">
		<div className="flex gap-1 justify-end">
			<Field field="price" /> <Field field="currency" />
		</div>
	</DataGridNumberColumn>

	{/* 3. Inline editable number */}
	<DataGridNumberColumn
		field="rating"
		format={value => (
			<ClickToEdit
				view={value}
				edit={<InputField field="rating" label={undefined} />}
			/>
		)}
	/>
</DataGridTable>
```

## Props Reference
| Name | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| field | string | Yes | — | Displayed field. |
| header | React.ReactNode | No | — | Custom header. If not provided, the label formatter is used. |
| children | React.ReactNode | No | — | Custom cell content. If not provided, the field value is displayed. |
| format | (value: number) => React.ReactNode | No | — | Custom value formatter. |
| filter | React.ReactNode | No | — | Custom filter. If not provided, a default number filter is used. |
## Examples

### Custom Color Coding
Render negative numbers in red and positives in green.

```tsx
<DataGridNumberColumn
	field="balance"
	format={value => (
		<span className={value < 0 ? 'text-red-600' : 'text-green-600'}>
			{value.toFixed(2)}
		</span>
	)}
/>
```
The formatter receives the raw number and returns a styled JSX node.

### Combining Formatters and Children
Provide a default formatter but allow overrides per-row.

```tsx
<DataGridNumberColumn
	field="discount"
	format={v => `${v}%`}
>
	<Field field="discount">
		{({ value }) => (
			<span className={value > 50 ? 'font-bold' : 'opacity-70'}>
				{value} %
			</span>
		)}
	</Field>
</DataGridNumberColumn>
```
If `children` are supplied, the component ignores the `format` prop.

## Best Practices & Gotchas
* Keep formatting functions pure; avoid side effects.
* Right-align numeric cells with `className="text-right"` for readability.
* Use `toLocaleString` for locale-aware formatting.
* Provide a custom `filter` when the default number filter is insufficient.
* Do not mix heavy logic inside `format`; delegate to helper utilities.
