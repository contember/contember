# DataGridNumberFilter

## Overview
`DataGridNumberFilter` renders a numeric filter for `DataGrid`.
It lets users filter rows by numbers without writing queries.

## Styling Guidelines
* All snippets use Tailwind CSS.
* The component does not require extra Contember theme classes.

## Core Concepts
* Filters translate user input into `DataGrid` query variables.
* You pass a `field` path that matches the column to filter.
* `label` appears in the UI.
* `name` identifies the filter in form state and URL params.
* The filter works inside or outside a `<DataGridToolbar>`.

## Quick Start
```tsx
import { DataGrid, DataGridNumberFilter, DataGridNumberColumn } from '~/lib/datagrid'

export const PostsGrid = () => (
	<DataGrid>
		<div className="flex gap-4 mb-4">
			<DataGridNumberFilter
				field="views"
				name="views"
				label="Views"
				className="max-w-xs"
			/>
		</div>

		<DataGridNumberColumn header="Views" field="views" />
	</DataGrid>
)
```

## Detailed Guide
```tsx
import {
	DataGrid,
	DataGridNumberColumn,
	DataGridNumberFilter,
	DataGridToolbar,
} from '~/lib/datagrid'

export const PostsGrid = () => (
	<DataGrid>
		<DataGridToolbar className="flex flex-wrap gap-4 mb-4">
			{/* Exact match */}
			<DataGridNumberFilter
				field="views"
				name="views_eq"
				label="Views equals"
			/>

			{/* Greater than */}
			<DataGridNumberFilter
				field="views >"
				name="views_gt"
				label="Views greater than"
			/>
		</DataGridToolbar>

		<DataGridNumberColumn header="Views" field="views" />
	</DataGrid>
)
```

## Examples
### Filter inside a popover
```tsx
import { Button } from '~/ui'
import {
	Popover,
	PopoverTrigger,
	PopoverContent,
} from '~/ui/popover'
import {
	DataGrid,
	DataGridNumberFilter,
} from '~/lib/datagrid'

export const PostsGrid = () => (
	<DataGrid>
		<Popover>
			<PopoverTrigger asChild>
				<Button variant="outline" className="h-auto">More filters</Button>
			</PopoverTrigger>

			<PopoverContent>
				<div className="flex flex-col gap-2">
					<DataGridNumberFilter
						field="views"
						name="views"
						label="Views"
					/>
				</div>
			</PopoverContent>
		</Popover>
	</DataGrid>
)
```
This pattern keeps secondary filters hidden until needed.

## Best Practices & Gotchas
* Always set `name`; it keeps URL queries predictable.
* Match `field` exactly with the related `DataGrid` column.
* Wrap multiple filters in a `flex gap-…` container for consistent spacing.
* Avoid using the same `name` twice; filters will collide.
* Combine this filter with `DataGridNumberColumn` for a clear UX.
