# DataGridTextColumn

## Overview
`DataGridTextColumn` renders a text cell inside a `DataGridTable` together with a sortable header and an optional filter.
Use it whenever you need to display string data in Contember data grids and optionally format, filter, or fully replace the cell content.

## Styling Guidelines
All examples below include Tailwind utility classes.
`DataGridTextColumn` itself adds no visual styles, so rely on your Tailwind setup or Contember theme classes applied to the surrounding `DataGridTable`.

## Core Concepts
- Column receives data through the `field` prop.
- Header label is auto-generated from `field` unless you pass `header`.
- Pass `format` to transform the raw string into JSX.
- Provide `children` for full control over the cell markup.
- A default text filter appears in the column menu; you can override it with `filter`.

## Quick Start
```tsx
import { DataGridTable, DataGridTextColumn } from '~/lib/datagrid'

<DataGridTable className="w-full">
  <DataGridTextColumn field="title" />
</DataGridTable>
```

## Detailed Guide
### Custom Formatting
```tsx
<DataGridTextColumn
  field="subtitle"
  format={value => (
    <span className="flex items-center gap-1 text-indigo-600">
      <Heading2Icon className="w-4 h-4" />
      {value}
    </span>
  )}
/>
```
### Replacing Cell Content
```tsx
<DataGridTextColumn field="internalName" header="Internal name">
  <Link to="form/articleEdit(id: $entity.id)">
    <AnchorButton variant="link" className="text-blue-700 hover:underline">
      <Field field="internalName" />
    </AnchorButton>
  </Link>
</DataGridTextColumn>
```

## Props Reference
| Name     | Type                                   | Required | Default | Description                                                                          |
|----------|----------------------------------------|----------|---------|--------------------------------------------------------------------------------------|
| field    | `string`                              | Yes      | —       | Displayed field.                                                                     |
| header   | `React.ReactNode`                     | No       | label   | Custom header. If omitted, Contember’s label formatter is used.                      |
| children | `React.ReactNode`                     | No       | —       | Custom cell content. Supersedes `format` and default rendering.                      |
| format   | `(value: string) => React.ReactNode`  | No       | —       | Custom value formatter. Runs only when `children` is not provided.                   |
| filter   | `React.ReactNode`                     | No       | default | Custom filter element rendered in the column menu.                                   |

_Inherited props: none_

## Examples
### Preview First 100 Characters
```tsx
<DataGridTextColumn
  field="description"
  format={txt => txt.slice(0, 100) + '…'}
/>
```
This trims long descriptions while keeping the grid compact.

### Inline Editing with Click-To-Edit
```tsx
<DataGridTextColumn
  header={fieldSchema.name}
  field={fieldSchema.name}
  format={value => (
    <ClickToEdit
      view={<span className="truncate">{value}</span>}
      edit={<TextareaField field={fieldSchema.name} label={undefined} />}
    />
  )}
/>
```
You let users click a cell to switch between read and edit modes without leaving the grid.

### Action Links in a Production Grid
```tsx
<DataGridTextColumn field="name" header="Author">
  <Link to="form/authorEdit(id: $entity.id)">
    <AnchorButton variant="link" className="text-primary hover:underline">
      <Field field="name" />
    </AnchorButton>
  </Link>
</DataGridTextColumn>
```
The cell becomes a navigable button that opens an edit form.

## Best Practices & Gotchas
- Keep formatting functions pure; avoid side effects inside `format`.
- Prefer `children` over `format` when you need multiple elements inside the cell.
- Remember to include `className="truncate"` on cells that might overflow.
- Provide a consistent `header` for readability when the `field` name is not user-friendly.
- Large datasets benefit from simple cells; avoid heavy React components inside `format`.
