# DataGridUnionTextFilter

## Overview
DataGridUnionTextFilter renders a text input that searches across several entity fields at once inside a Contember `DataGrid`.
You pass one or more field names and the component builds a single union filter behind the scenes.

## Styling Guidelines
* All examples use Tailwind classes.
* The component itself adds no styling, so you must style the surrounding layout.

## Core Concepts
* A single text value matches **any** of the listed fields.
* You can pass simple field names or objects with `field` and `hasOneRelationPath`.
* The `name` prop scopes the filter within the parent `DataGrid` filter group.

## Quick Start
```tsx
import { DataGrid, DataGridUnionTextFilter } from '@contember/react-datagrid'

<DataGrid entities="Project">
	<DataGridUnionTextFilter
		name="projectSearch"
		fields={['name', 'description']}
		label="Search"
		className="mb-4 w-full max-w-sm"
	/>
	{/* ...columns */}
</DataGrid>
```


## Props Reference
| Name  | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| name  | `string` | Yes | – | Identifier for the filter inside the DataGrid filter form. |
| fields | `string \| FieldConfig \| Array<string \| FieldConfig>` | Yes | – | Fields that participate in the union search. `FieldConfig` is `{ field: string; hasOneRelationPath?: SugarableHasOneRelation \| SugarableHasOneRelation[] }`. |
| label | `React.ReactNode` | No | `undefined` | Optional label shown next to the input. |


## Examples

### Search on related entity field
```tsx
<DataGridUnionTextFilter
	name="taskSearch"
	fields={[
		{ field: 'title' },
		{ field: 'authorName', hasOneRelationPath: ['author'] },
	]}
	label="Task or Author"
/>
```
The filter now matches the task title or the related author’s name.

### Minimal inline filter with custom width
```tsx
<DataGridUnionTextFilter
	name="quick"
	fields={['code']}
	className="w-48"
/>
```
Using a narrow width saves space in toolbars.

## Best Practices & Gotchas
* Keep the number of fields reasonable to avoid large SQL conditions.
* Apply Tailwind utility classes directly because the component outputs only raw elements.
* Ensure `name` is unique within one `DataGrid`.
* Use `hasOneRelationPath` when the field lives on a related entity.
