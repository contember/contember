# DataGridUuidColumn

## Overview
`DataGridUuidColumn` displays a UUID field inside a `DataGridTable`.
You can customize the header, cell content, or value formatter while keeping the column wired into Contember’s data-fetching layer.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No additional Contember theme classes are required; `DataGrid` already delivers its own styling.

## Core Concepts
* A UUID column renders raw or formatted UUID strings.
* The `field` prop binds the column to an entity field.
* You may override the auto-generated header through `header`.
* `format` lets you present the UUID in a custom way.
* `children` fully replaces the cell content when you need advanced markup.

## Quick Start
```tsx
import {
  DataGrid,
  DataGridTable,
  DataGridUuidColumn,
} from '~/lib/datagrid'

export default () => (
  <DataGrid allowCreate className="border rounded shadow">
    <DataGridTable className="min-w-full divide-y divide-gray-200">
      <DataGridUuidColumn field="id" />
    </DataGridTable>
  </DataGrid>
)
```

## Detailed Guide
When you need a shorter visual for long UUIDs, pass a formatter:

```tsx
<DataGridUuidColumn
  field="id"
  header="Identifier"
  format={value => `${value.slice(0, 8)}…`}
  className="text-sm text-gray-700"
/>
```

## Props Reference
| Name     | Type                                   | Required | Default | Description                                                                 |
|----------|----------------------------------------|----------|---------|-----------------------------------------------------------------------------|
| field    | string                                 | Yes      | —       | Displayed field.                                                            |
| header   | React.ReactNode                        | No       | —       | Custom header. If omitted, the label formatter is used.                     |
| children | React.ReactNode                        | No       | —       | Custom cell content. Overrides the default UUID rendering.                 |
| format   | (value: string) => React.ReactNode     | No       | —       | Custom value formatter. Receives the raw UUID.                              |


## Examples

### 1. Shortened UUID with Tooltip
```tsx
<DataGridUuidColumn
  field="uuid"
  header="UUID"
  format={value => (
    <span className="cursor-help" title={value}>
      {value.slice(0, 8)}…
    </span>
  )}
/>
```
The user sees a compact string while keeping the full UUID accessible on hover.

### 2. Custom Cell Content
```tsx
<DataGridUuidColumn field="uuid">
  {value => (
    <button
      onClick={() => navigator.clipboard.writeText(value)}
      className="text-blue-600 hover:underline"
    >
      Copy {value.slice(0, 4)}
    </button>
  )}
</DataGridUuidColumn>
```
`children` receives the field value, enabling custom actions like copy-to-clipboard.

## Best Practices & Gotchas
* Keep the UUID visible in some form; hiding it can hinder debugging.
* Use `format` for simple string transformations; switch to `children` for interactive content.
* Avoid expensive computations inside `format`—it runs for every cell render.
* Do not forget to include `field`; the column won’t fetch data without it.
