# DataViewFieldLabel

## Overview
`DataViewFieldLabel` renders a schema-aware label for a single entity field.
You pass a field path and the component looks up the correct human-readable label.

## Styling Guidelines
* All snippets use Tailwind classes for layout and typography.
* The component itself adds no styles; wrap it in elements that carry your Contember theme classes.

## Core Concepts
* Schema-driven rendering – labels come from your Contember schema.
* Field paths – dot-notation lets you reach nested relations.
* Presentation separation – keep labels and values in distinct components.

## Quick Start
```tsx
// App.tsx
import { DataViewFieldLabel } from '~/lib/datagrid/labels'

export const JobHeader = () => (
  <header className="flex items-center gap-2">
    <DataViewFieldLabel field="job.name" />
  </header>
)
```

## Detailed Guide
```tsx
// Nested field example
<DataViewFieldLabel field="employee.address.city" />
```
*The component resolves the nested path and prints the schema label for `city`.*

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| field | string | Yes | — | Dot-separated path to the field whose label you want to show |

_No inherited props_


## Examples

### Toggle between labels and values
```tsx
import { DataViewFieldLabel } from '~/lib/datagrid/labels'

export const WorkforceJobHeader = () => (
  <div className="grid grid-cols-2 gap-4">
    <div className="text-gray-500">
      <DataViewFieldLabel field="job.code" />
    </div>
    <div>
      {/* your DataViewValue component here */}
    </div>
  </div>
)
```
This snippet places the label in a muted column while reserving the second column for the value.

### Statistics Dashboard
```tsx
<div className="space-y-1">
  <DataViewFieldLabel field="job.stats.maxFactor" />
  <DataViewFieldLabel field="info.active" />
  <DataViewFieldLabel field="info.absent" />
</div>
```
You can list several labels to create a compact KPI header.

## Best Practices & Gotchas
* Always use dot-notation for nested relations.
* Wrap the component in a Tailwind-styled element rather than passing style props.
* Cache expensive parents; the label itself is lightweight.
