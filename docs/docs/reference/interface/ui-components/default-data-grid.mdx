# DefaultDataGrid

## Overview
`DefaultDataGrid` is a ready-to-use wrapper around `DataGrid`.
It renders a toolbar, loader, table and pagination so you can list entities with almost no setup.

## Styling Guidelines
* All examples use Tailwind utility classes.
* `DefaultDataGrid` itself does not require any special Contember theme class.

## Core Concepts
* `entities` defines which Contember entities the grid fetches.
* Child columns (`DataGridTextColumn`, `DataGridEnumColumn`, …) decide what you see.
* Selection, filtering and sorting states can persist by passing storage props.
* You can extend the grid with a custom `toolbar` node.

## Quick Start
```tsx
import { DefaultDataGrid, DataGridActionColumn, DataGridTextColumn, DataGridEnumColumn } from '~/lib/datagrid'
import { Button } from '@contember/ui'

export const ProjectList = () => (
	<DefaultDataGrid entities="Project" className="w-full">
		<DataGridActionColumn>
			<Button size="sm">Show detail</Button>
		</DataGridActionColumn>

		<DataGridTextColumn header="Title" field="title" />
		<DataGridEnumColumn header="State" field="state" />
	</DefaultDataGrid>
)
```


## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| onSelectHighlighted | (entity: EntityAccessor) => void | Yes | – | Called when you confirm the highlighted row (Enter key). |
| queryField | DataViewUnionFilterFields | Yes | – | Field used by the built-in quick search. |
| dataViewKey | string | Yes | – | Unique key for storing grid state. |
| entities | string &#124; EntitiesSpecifierObject | Yes | – | Entity name or sugar syntax for filtering relations. |
| filterTypes | DataViewFilterHandlerRegistry | Yes | – | Registry of available filter types. |
| initialFilters | DataViewFilteringArtifacts &#124; (stored) => same | Yes | – | Initial filter values or a function producing them. |
| filteringStateStorage | StateStorageOrName | Yes | – | Where to persist filter state. |
| initialSorting | DataViewSortingDirections | Yes | – | Initial sort directions. |
| sortingStateStorage | StateStorageOrName | Yes | – | Where to persist sorting state. |
| initialItemsPerPage | number | Yes | – | Items per page when the grid mounts. |
| currentPageStateStorage | StateStorageOrName | Yes | – | Where to persist the current page index. |
| pagingSettingsStorage | StateStorageOrName | Yes | – | Where to persist items-per-page changes. |
| initialSelection | DataViewSelectionValues &#124; (stored) => same | Yes | – | Initial column visibility / layout selection. |
| selectionStateStorage | StateStorageOrName | Yes | – | Where to persist selection state. |
| layouts | DataViewSelectionLayout[] | Yes | – | Pre-defined column layouts. |
| children | React.ReactNode | Yes | – | Grid columns and custom cells. |
| toolbar | React.ReactNode | No | undefined | Replaces the default toolbar content. |

## Sub-Components / Building Blocks
### DataGridActionColumn
Renders an extra column for row-level actions like “Edit” or “Delete”.

### DataGridTextColumn
Displays a text field.

### DataGridEnumColumn
Displays an enum field with optional value mapping.

_You can use any other `DataGrid*Column`, filter or tooltip component inside `DefaultDataGrid`._

## Examples
### Custom Toolbar
```tsx
<DefaultDataGrid
	entities="FormArticle"
	toolbar={<div className="flex gap-2"><Button size="sm">Export</Button></div>}
>
	<DataGridTextColumn header="Internal name" field="internalName" />
</DefaultDataGrid>
```
Above you inject a custom toolbar with an “Export” button.

### Persisted Filters and Sorting
```tsx
<DefaultDataGrid
	entities="Company"
	dataViewKey="companies"
	initialFilters={{ active: { includeTrue: true } }}
	initialSorting={{ name: 'asc' }}
	filteringStateStorage="local"
	sortingStateStorage="local"
>
	<DataGridTextColumn header="Name" field="name" />
</DefaultDataGrid>
```
The grid remembers filter and sort preferences in `localStorage`.

## Best Practices & Gotchas
* Keep `dataViewKey` stable; otherwise stored state resets.
* Wrap the grid in a `Binding` when you need `$entity` in links.
* Use `StateStorageOrName = "session"` for per-tab persistence.
* Do not place heavy logic inside cell render functions; keep them pure.
* Always provide a height strategy (e.g., `overflow-auto`) when the surrounding layout is flexible.
