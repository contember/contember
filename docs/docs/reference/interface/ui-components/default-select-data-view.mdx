## Overview
`DefaultSelectDataView` renders a ready-made `DataView` optimised for select and multiselect pop-overs.  
You provide an optional search field, an initial sort order, and a child renderer for list items.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* The component adds no Contember theme classes, so you rely on your project’s defaults.

## Core Concepts
* Integrates `DataView` into a select pop-over.  
* `queryField` enables full-text search.  
* `initialSorting` sets first ordering.  
* `children` renders each entity row.  
* The component is headless—style it in parent UI.

## Quick Start
```tsx
import { DefaultSelectDataView } from './list'

<DefaultSelectDataView>
  {entity => <span className="block px-2 py-1">{entity.name}</span>}
</DefaultSelectDataView>
```

## Detailed Guide
```tsx
<DefaultSelectDataView
  queryField="fullName"
  initialSorting={{ fullName: 'asc' }}
>
  {entity => (
    <div className="flex items-center gap-2 px-2 py-1">
      <img src={entity.avatarUrl} className="w-6 h-6 rounded-full" />
      <span>{entity.fullName}</span>
    </div>
  )}
</DefaultSelectDataView>
```
* `queryField` links search input to `fullName`.  
* `initialSorting` keeps list alphabetical.  
* The child function renders each row with an avatar and name.

## Props Reference
| Name | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| queryField | DataViewUnionFilterFields | No | — | Field used for full-text filtering. |
| initialSorting | DataViewSortingDirections | No | — | Sorting applied on first render. |
| children | React.ReactNode | Yes | — | Renderer for each entity. |
| *…DataView props* | *Inherited* | No | — | All other `DataView` props are forwarded. |

## Best Practices & Gotchas
* Wrap the component in `SelectPopoverContent` or similar pop-over.  
* Index `queryField` in your database for fast search.  
* Keep row markup light to ensure smooth scrolling.  
* Pass a stable object to `initialSorting` to avoid unnecessary re-renders.
