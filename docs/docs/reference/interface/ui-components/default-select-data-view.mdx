## Overview
`DefaultSelectDataView` renders a standard `DataView` optimised for select and multiselect pop-overs.  
It accepts a search field, an initial sort order, and children that define individual list items.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* The component itself adds no Contember theme classes, so you only need your project’s defaults.

## Core Concepts
* It plugs a ready-made `DataView` into a select pop-over.  
* `queryField` enables text search in the list.  
* `initialSorting` decides the first ordering of items.  
* You pass `children` to describe how every entity renders.  
* The component is headless—styling happens in surrounding UI elements.

## Quick Start
```tsx
import { DefaultSelectDataView } from './list'

<DefaultSelectDataView>
  {entity => <span className="block px-2 py-1">{entity.name}</span>}
</DefaultSelectDataView>
```

## Detailed Guide
```tsx
<DefaultSelectDataView
  queryField="fullName"
  initialSorting={{ fullName: 'asc' }}
>
  {entity => (
    <div className="flex items-center gap-2 px-2 py-1">
      <img src={entity.avatarUrl} className="w-6 h-6 rounded-full" />
      <span>{entity.fullName}</span>
    </div>
  )}
</DefaultSelectDataView>
```
* `queryField` links the search input to the `fullName` column.  
* `initialSorting` keeps the list alphabetical.  
* The child function renders each row with an avatar and name.

## Props Reference
| Name | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| queryField | DataViewUnionFilterFields | No | — | Field used for full-text filtering. |
| initialSorting | DataViewSortingDirections | No | — | Sorting applied on first render. |
| children | React.ReactNode | Yes | — | Row renderer for each entity. |

The example uses `DefaultSelectDataView` to choose an author while keeping validation logic separate.

## Best Practices & Gotchas
* Always wrap the component in a `SelectPopoverContent` or similar pop-over.  
* Keep `queryField` indexed in the database for fast search.  
* Avoid heavy layouts inside children; simpler markup keeps scrolling smooth.  
* Pass a stable object to `initialSorting` to prevent unnecessary re-renders.
