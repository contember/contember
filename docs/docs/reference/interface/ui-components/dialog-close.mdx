# DialogClose

## Overview
DialogClose renders a wrapper element that closes the nearest `<Dialog>` when the user activates it.
It forwards props to its single child, so you usually wrap a `<Button>` or any clickable element inside it.

## Styling Guidelines
* All snippets use Tailwind CSS classes.
* No mandatory Contember theme classes are required for DialogClose itself.

## Core Concepts
* DialogClose must live inside an open `<Dialog>` hierarchy.
* It **does not** render visible UI; the child element provides the visual part.
* The child receives all accessibility attributes automatically.
* You can combine `asChild` on the child element for fine-grained control.

## Quick Start
```tsx
import { Dialog, DialogTrigger, DialogContent, DialogClose } from '~/lib/ui/dialog'

export const BasicDialog = () => (
  <Dialog>
    <DialogTrigger asChild>
      <button className="px-4 py-2 bg-blue-600 text-white rounded">Open</button>
    </DialogTrigger>

    <DialogContent className="p-6 bg-white rounded shadow">
      <p className="text-sm">Dialog content goes here.</p>

      <DialogClose asChild>
        <button className="mt-4 px-4 py-2 bg-gray-200 rounded">Close</button>
      </DialogClose>
    </DialogContent>
  </Dialog>
)
```

## Examples

### Confirm-Cancel Dialog
Used to confirm a dangerous action and close the dialog on cancel.

```tsx
import { Dialog, DialogTrigger, DialogContent, DialogClose, DialogFooter } from '~/lib/ui/dialog'

export const ConfirmArchive = () => (
  <Dialog>
    <DialogTrigger asChild>
      <button className="text-red-600 underline">Archive item</button>
    </DialogTrigger>

    <DialogContent>
      <p>Do you really want to archive this item?</p>

      <DialogFooter className="flex gap-4 justify-end">
        <button className="px-4 py-2 bg-red-600 text-white rounded">Confirm</button>

        <DialogClose asChild>
          <button className="px-4 py-2 bg-gray-200 rounded">Cancel</button>
        </DialogClose>
      </DialogFooter>
    </DialogContent>
  </Dialog>
)
```

Explanation
* Confirm button performs the action but **does not** close the dialog automatically.
* DialogClose wraps the cancel button, ensuring the dialog closes without extra code.

## Best Practices & Gotchas
* Always provide a single child to DialogClose; multiple children break `asChild`.
* Wrap only interactive elements (button, link) to keep accessibility intact.
* Need to close the dialog from code? Use the `onOpenChange` prop on `<Dialog>` instead.
* Combine with variants like `variant="secondary"` to keep visual consistency.
