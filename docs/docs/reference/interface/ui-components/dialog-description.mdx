# DialogDescription

## Overview
`DialogDescription` renders descriptive text inside a dialog.
Use it to give users extra context about the dialog’s purpose or required action.

## Styling Guidelines
* All examples use Tailwind CSS.
* No additional Contember theme classes are required for `DialogDescription`.

## Core Concepts
* `DialogDescription` must be nested inside a `DialogContent`.
* It is purely presentational—no state or logic.
* Content is passed as children.

## Quick Start
```tsx
import { Dialog, DialogContent, DialogDescription, DialogHeader, DialogTitle } from '~/lib/ui/dialog'

export function Example() {
	return (
		<Dialog open={true}>
			<DialogContent className="max-w-md space-y-4">
				<DialogHeader>
					<DialogTitle className="text-xl font-bold">Invite user</DialogTitle>
					<DialogDescription className="text-sm text-gray-500">
						We’ll send an email with an invitation link.
					</DialogDescription>
				</DialogHeader>
				{/* your form goes here */}
			</DialogContent>
		</Dialog>
	)
}
```

## Examples

### Locale Availability Dialog
```tsx
<DialogDescription>{componentLabels.entityLocaleAvailability.dialog.description}</DialogDescription>
```
Renders a localized description explaining why the dialog appeared.

### Sign-In Prompt
```tsx
<DialogDescription>{labels.signDescription}</DialogDescription>
```
Provides helpful text below the dialog title on a sign-in screen.

## Best Practices & Gotchas
* Keep descriptions short and actionable.
* Use `text-sm` Tailwind class to visually separate it from the title.
* Do not place interactive elements inside `DialogDescription`.
