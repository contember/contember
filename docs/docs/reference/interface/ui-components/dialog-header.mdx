# DialogHeader

## Overview
DialogHeader renders the top section of a dialog.
Use it inside `DialogContent` to group the dialog title and optional descriptive text.

## Styling Guidelines
* All snippets use Tailwind CSS.
* No special Contember theme classes are required for DialogHeader.

## Core Concepts
* DialogHeader is purely structural; it adds no visual styles.
* Place `DialogTitle` (and optional text) inside DialogHeader.
* DialogHeader must be nested in `DialogContent`.
* Combine with `Dialog`, `DialogTrigger`, and `DialogFooter` for a complete flow.

## Quick Start
```tsx
import { Dialog, DialogTrigger, DialogContent, DialogHeader, DialogTitle } from '~/lib/ui/dialog'

export const Basic = () => (
	<Dialog>
		<DialogTrigger asChild>
			<button className="px-4 py-2 bg-primary text-white rounded">Open dialog</button>
		</DialogTrigger>

		<DialogContent className="max-w-md">
			<DialogHeader>
				<DialogTitle className="text-2xl font-bold">Hello!</DialogTitle>
				<p className="mt-1 text-sm text-gray-500">This is a DialogHeader example.</p>
			</DialogHeader>
		</DialogContent>
	</Dialog>
)
```

## Examples

### Header with Additional Description
```tsx
<DialogHeader>
	<DialogTitle className="text-xl font-semibold">Delete item</DialogTitle>
	<p className="mt-2 text-sm text-red-600">This action cannot be undone.</p>
</DialogHeader>
```
Use a paragraph under the title to warn users before a destructive action.

### Dynamic Title
```tsx
const ItemDialogHeader = ({ name }: { name: string }) => (
	<DialogHeader>
		<DialogTitle className="text-2xl font-bold">{name}</DialogTitle>
	</DialogHeader>
)
```
Pass dynamic data to the header so the dialog reflects current context.

## Best Practices & Gotchas
* Keep the header concise; long text belongs in `DialogContent`.
* Always place DialogHeader as the first child of `DialogContent`.
* Do not apply heavy padding; let `DialogContent` handle general spacing.
* Combine semantic HTML (`h1`, `p`) with Tailwind for accessibility.
