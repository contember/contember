# DimensionsSwitcher

## Overview
`DimensionsSwitcher` lets you switch between multiple data dimensions such as languages or markets.  
You provide an entity list, a slug field, and child fields; the component renders selectable options.

It supports optional initial sorting and multi‑selection, so you can handle both simple and complex editing screens.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* The component itself adds no styles; wrap it in Tailwind‑styled containers as needed.  
* No special Contember theme classes are required.

## Core Concepts
* Dimension – arbitrary context key (for example, `locale`) stored in Contember dimensions.
* `options` – entity list supplying selectable records.
* `slugField` – unique identifier field of each option.
* Child fields – what renders inside each option (commonly a `Field` component).
* `orderBy` – optional initial sorting of options.
* Multi‑selection – enabled with `isMulti`.

## Quick Start
```tsx
import { DimensionsSwitcher } from '~/lib/dimensions'

<div className="flex gap-2">
  <DimensionsSwitcher
    dimension="locale"
    slugField="code"
    options="DimensionsLocale"
  >
    <Field field="label" />
  </DimensionsSwitcher>
</div>
```

## Examples

### 1. Basic Locale Switcher
```tsx
<DimensionsSwitcher
  dimension="locale"
  slugField="code"
  options="DimensionsLocale"
>
  <Field field="label" />
</DimensionsSwitcher>
```
You render a simple list of locales. The user can switch the current locale with one click.

### 2. Sorted, Multi‑Select Locale Switcher
```tsx
<DimensionsSwitcher
  dimension="locale"
  slugField="code"
  options="DimensionsLocale"
  orderBy={{ label: 'desc' }}
  isMulti
>
  <Field field="label" />
</DimensionsSwitcher>
```
You allow selecting several locales. Options are sorted alphabetically descending by label.

### 3. Full Page Integration with SideDimensions
```tsx
<>
  <Slots.Title>
    <Title icon={<LanguagesIcon />}>Dimensions</Title>
  </Slots.Title>

  <Binding>
    <div>
      <DimensionsSwitcher
        options="DimensionsLocale"
        slugField="code"
        dimension="locale"
        isMulti
      >
        <Field field="label" />
      </DimensionsSwitcher>
    </div>
  </Binding>

  <Binding>
    <Slots.Actions>
      <PersistButton />
    </Slots.Actions>

    <EntitySubTree entity="DimensionsItem(unique = unique)">
      <SideDimensions
        dimension="locale"
        as="currentLocale"
        field="locales(locale.code = $currentLocale)"
      >
        <Card>
          <CardHeader>
            <CardTitle><Variable name="currentLocale" /></CardTitle>
          </CardHeader>

          <CardContent>
            <InputField field="title" />
            <TextareaField field="content" />
          </CardContent>
        </Card>
      </SideDimensions>
    </EntitySubTree>
  </Binding>
</>
```
You combine `DimensionsSwitcher` with `SideDimensions`, `PersistButton`, and header slots.  
Users pick one or more locales, edit locale‑specific fields, and persist changes.

## Best Practices & Gotchas
* Always pass a stable `slugField`; unreliable slugs break selection tracking.
* Wrap the component in Tailwind layouts instead of adding props that do not exist.
* Use `orderBy` sparingly; client sorting can confuse users if the server returns a different order.
* Enable `isMulti` only when downstream components understand multiple dimension values.
