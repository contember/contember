# Divider

## Overview
Divider renders a thin horizontal rule that visually separates groups of related content.
Use it to break long forms, lists, or panels into digestible sections without adding semantic meaning.

## Styling Guidelines
* All examples use Tailwind CSS classes.
* Divider requires no special Contember theme classes.

## Core Concepts
* Divider adds visual separation only; it carries no semantics.
* It takes up the full width of its parent container.
* Keep vertical spacing consistent with surrounding elements.

## Quick Start
```tsx
import { Divider } from '~/lib/ui/divider'

export default function Example() {
	return (
		<div className="space-y-4">
			<p>Section one</p>
			<Divider />
			<p>Section two</p>
		</div>
	)
}
```

## Examples

### Split a settings form into logical blocks
```tsx
import { Divider } from '~/lib/ui/divider'

function SettingsForm() {
	return (
		<form className="max-w-xl space-y-4">
			<AccountFields />
			<Divider />
			<NotificationFields />
			<Divider />
			<SecurityFields />
		</form>
	)
}
```
Use Divider to visually chunk the form into three manageable parts.

### Combine with action rows
```tsx
<div className="flex flex-col gap-4 p-4 border rounded-md bg-white">
	<UserProfileSummary />
	<Divider />
	<div className="flex justify-end">
		<SaveButton />
	</div>
</div>
```
Place Divider before the action row to make primary actions stand out.

## Best Practices & Gotchas
* Place Divider inside a flex-column or block container to avoid unexpected orientation.
* Do not rely on Divider for semantic grouping; use headings or ARIA landmarks instead.
* Keep consistent vertical padding (`space-y-*`) around Divider for visual balance.
