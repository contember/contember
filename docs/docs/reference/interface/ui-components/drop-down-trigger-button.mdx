# DropDownTriggerButton

## Overview
DropDownTriggerButton renders a button used as the visual trigger for a Radix UI–style `DropdownMenu`.
You usually wrap it with `DropdownMenuTrigger asChild` so the root dropdown handles all interactions.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No Contember-specific theme classes are required by default.

## Core Concepts
* Wrap the button with `DropdownMenuTrigger asChild`.
* Combine it with `DropdownMenu`, `DropdownMenuContent`, and related items.
* Pass additional HTML button props directly to the component (inherits standard button props).

## Quick Start
```tsx
import {
  DropdownMenu,
  DropdownMenuTrigger,
  DropdownMenuContent,
  DropdownMenuItem,
  DropDownTriggerButton,
} from '~/lib/ui/dropdown'

export const SimpleDropdown = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <DropDownTriggerButton className="px-3 py-2 bg-primary-500 text-white rounded" />
    </DropdownMenuTrigger>

    <DropdownMenuContent className="w-40">
      <DropdownMenuItem>Profile</DropdownMenuItem>
      <DropdownMenuItem>Settings</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

## Examples

### Repeater Action Menu
```tsx
import {
  DropdownMenu,
  DropdownMenuTrigger,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuSeparator,
  DropDownTriggerButton,
} from '~/lib/ui/dropdown'
import { DeleteEntityDialog } from '~/lib/ui/dialogs'

export const RepeaterMenu = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <DropDownTriggerButton className="p-2 rounded hover:bg-gray-100" />
    </DropdownMenuTrigger>

    <DropdownMenuContent className="w-[160px]">
      <DropdownMenuItem>Edit</DropdownMenuItem>
      <DropdownMenuItem>Make a copy</DropdownMenuItem>
      <DropdownMenuSeparator />
      <DeleteEntityDialog
        trigger={
          <DropdownMenuItem
            onSelect={e => e.preventDefault()}
            className="text-danger-600"
          >
            Delete
          </DropdownMenuItem>
        }
      />
    </DropdownMenuContent>
  </DropdownMenu>
)
```
Explanation
* `DropDownTriggerButton` provides a consistent trigger style.
* `DeleteEntityDialog` nests a dialog inside the dropdown while preventing default selection behavior.

## Best Practices & Gotchas
* Always wrap the trigger button with `DropdownMenuTrigger asChild`; otherwise, clicks won’t propagate.
* Apply Tailwind classes directly to customize size, color, and hover states.
* Prevent default selection on destructive actions to keep the dropdown open until the dialog takes over.
