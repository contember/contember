# DropdownMenuContent

## Overview
`DropdownMenuContent` renders the visible panel of a dropdown menu.
It positions itself relative to `DropdownMenuTrigger` and handles alignment, offsets, and focus management.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No mandatory Contember theme classes are required; you may add project-specific classes as needed.

## Core Concepts
* The component must be wrapped in a `<DropdownMenu>` context.
* Positioning is controlled with `side`, `align`, and `sideOffset` props.
* The element accepts standard `className` for Tailwind styling.
* You can nest `DropdownMenuItem`, `DropdownMenuGroup`, `DropdownMenuLabel`, and `DropdownMenuSeparator` inside.
* Mobile layouts often change the `side` prop to `"bottom"`.

## Quick Start
```tsx
import { DropdownMenu, DropdownMenuTrigger, DropdownMenuContent, DropdownMenuItem } from '~/lib/ui/dropdown'

export const SimpleDropdown = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <button className="px-3 py-1.5 rounded-md bg-blue-600 text-white">Actions</button>
    </DropdownMenuTrigger>

    <DropdownMenuContent className="min-w-40 rounded-md bg-white shadow-lg p-1">
      <DropdownMenuItem className="px-2 py-1 rounded hover:bg-gray-100">Edit</DropdownMenuItem>
      <DropdownMenuItem className="px-2 py-1 rounded hover:bg-gray-100">Delete</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

## Props Reference
| Name | Type | Required | Default | Description |
| ---- | ---- | -------- | ------- | ----------- |
| children | ReactNode | Yes | — | Elements rendered inside the menu. |
| side | `"top" \| "right" \| "bottom" \| "left"` | No | `"bottom"` | Which side of the trigger to place the menu. |
| align | `"start" \| "center" \| "end"` | No | `"start"` | Horizontal alignment relative to the trigger. |
| sideOffset | number | No | `0` | Pixel offset from the chosen side. |
| className | string | No | — | Additional Tailwind or custom classes. |
| ...HTMLDivProps | — | No | — | All standard `<div>` props are forwarded. |

## Sub-Components / Building Blocks
### DropdownMenuItem
Interactive row inside the content. Supports keyboard navigation and selection callbacks.

### DropdownMenuGroup
Logical grouping of items. It does not affect styling by default.

### DropdownMenuLabel
Non-interactive text header, often used to describe the following group.

### DropdownMenuSeparator
Thin horizontal rule that separates logical groups.

## Examples
```tsx
// Avatar menu with user actions
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <button className="flex items-center gap-2 px-2 py-1 rounded-md hover:bg-gray-50">
      <img src={avatarUrl} className="w-6 h-6 rounded-full" />
      <span>{userEmail}</span>
    </button>
  </DropdownMenuTrigger>

  <DropdownMenuContent className="w-56 rounded-lg" align="end" sideOffset={4}>
    <DropdownMenuLabel className="px-3 py-1.5 text-sm font-semibold">{userEmail}</DropdownMenuLabel>
    <DropdownMenuSeparator />
    <DropdownMenuItem className="flex gap-2">
      <BadgeCheckIcon size={16} /> Security
    </DropdownMenuItem>
    <DropdownMenuItem className="flex gap-2">
      <BellIcon size={16} /> API keys
    </DropdownMenuItem>
    <DropdownMenuSeparator />
    <DropdownMenuItem className="flex gap-2">
      <LogOutIcon size={16} /> Logout
    </DropdownMenuItem>
  </DropdownMenuContent>
</DropdownMenu>
```
This pattern shows nested items, separators, and the use of `align="end"` for right alignment.

## Best Practices & Gotchas
* Always place `DropdownMenuContent` inside a `DropdownMenu` provider.
* Use `sideOffset` instead of custom margin utilities to avoid layout shifts.
* Wrap complex interactive children in `DropdownMenuItem asChild` to preserve keyboard support.
* Don’t apply focus styles directly on items; Radix handles focus states.
* Avoid heavy padding on `DropdownMenuContent`; pad inner items for better click targets.
