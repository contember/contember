# DropdownMenuGroup

## Overview
DropdownMenuGroup visually and semantically groups related dropdown items.
Use it inside `DropdownMenuContent` to keep menus organized and easier to scan.

## Styling Guidelines
* All examples use Tailwind CSS.
* DropdownMenuGroup itself adds no visual styles; style individual items instead.
* No special Contember theme classes are required.

## Core Concepts
* Place only `DropdownMenuItem`, `DropdownMenuLabel`, or `DropdownMenuSeparator` inside the group.
* Nest the group within `DropdownMenuContent`.
* The group has no props; it only affects structure and accessibility.
* You can render multiple groups in a single dropdown.

## Quick Start
```tsx
import {
  DropdownMenu,
  DropdownMenuTrigger,
  DropdownMenuContent,
  DropdownMenuGroup,
  DropdownMenuItem,
} from '~/lib/ui/dropdown'

export const SettingsButton = () => (
  <DropdownMenu>
    <DropdownMenuTrigger className="px-3 py-1 rounded bg-blue-600 text-white">
      Settings
    </DropdownMenuTrigger>

    <DropdownMenuContent className="p-2 bg-white shadow-lg rounded">
      <DropdownMenuGroup>
        <DropdownMenuItem className="flex gap-2 px-2 py-1 rounded hover:bg-gray-100">
          Profile
        </DropdownMenuItem>
        <DropdownMenuItem className="flex gap-2 px-2 py-1 rounded hover:bg-gray-100">
          Logout
        </DropdownMenuItem>
      </DropdownMenuGroup>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

## Examples

### Grouping tenant-level actions
```tsx
<DropdownMenuGroup>
  <DropdownMenuItem>
    <Link to="tenant/security">
      <AnchorButton variant="ghost" size="xs" className="flex gap-2">
        <BadgeCheckIcon size={16} />
        Security
      </AnchorButton>
    </Link>
  </DropdownMenuItem>

  <DropdownMenuItem>
    <Link to="tenant/apiKeys">
      <AnchorButton variant="ghost" size="xs" className="flex gap-2">
        <BellIcon size={16} />
        API keys
      </AnchorButton>
    </Link>
  </DropdownMenuItem>
</DropdownMenuGroup>
```
The group collects tenant-related links. Users understand that both items act on the same context.

## Best Practices & Gotchas
* Keep each group focused on one subject area.
* Use `DropdownMenuSeparator` between groups, not inside them.
* Remember: the group does not add padding; apply spacing classes as needed.
