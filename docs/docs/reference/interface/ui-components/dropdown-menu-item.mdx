## Overview
DropdownMenuItem renders a selectable row inside a DropdownMenu component.
Use it to wrap links, buttons, or any interactive element you want to appear in the pop-up menu.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No special Contember theme classes are required.

## Core Concepts
* A menu item **forwards focus** to the enclosed element.
* `onSelect` lets you intercept selection without closing the menu.
* `asChild` passes props to the child element instead of rendering its own wrapper.

## Quick Start
```tsx
import { DropdownMenu, DropdownMenuTrigger, DropdownMenuContent, DropdownMenuItem } from '~/lib/ui/dropdown'
import { Button } from '~/lib/ui/button'

export const BasicMenu = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <Button className="flex gap-1">Options</Button>
    </DropdownMenuTrigger>

    <DropdownMenuContent className="bg-white shadow-lg rounded">
      <DropdownMenuItem className="px-3 py-1.5 hover:bg-gray-100">Edit</DropdownMenuItem>
      <DropdownMenuItem className="px-3 py-1.5 hover:bg-gray-100">Duplicate</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

## Detailed Guide
```tsx
// Prevent menu from closing when the item is clicked
<DropdownMenuItem
  onSelect={e => e.preventDefault()}
  className="px-3 py-1.5 hover:bg-gray-100"
>
  Persistent action
</DropdownMenuItem>
```

```tsx
// Use `asChild` when you already have a styled component
<DropdownMenuItem asChild>
  <a href="/profile" className="flex items-center gap-2 px-3 py-1.5">
    <UserIcon size={16} />
    Profile
  </a>
</DropdownMenuItem>
```

## Props Reference

| Name       | Type                            | Required | Default | Description                                                                       |
|------------|---------------------------------|----------|---------|-----------------------------------------------------------------------------------|
| onSelect   | `(event: Event) => void`        | No       | –       | Called when the item is selected. Call `event.preventDefault()` to keep the menu open. |
| asChild    | `boolean`                       | No       | `false` | Render the child element directly without an extra wrapper.                       |
| className  | `string`                        | No       | –       | Tailwind or custom CSS classes.                                                   |
| ...HTMLDivAttributes | See `React.HTMLAttributes<HTMLDivElement>` | No | – | Inherited DOM attributes.                                                         |

## Examples
### Wrapping a React Router Link
```tsx
<DropdownMenuItem onSelect={e => e.preventDefault()} asChild>
  <Link to="tenant/apiKeys" className="flex gap-2 px-3 py-1.5">
    <KeyIcon size={16} />
    API Keys
  </Link>
</DropdownMenuItem>
```
Use `onSelect` to stop the menu from closing so React Router can handle navigation.

### Confirm-delete Flow
```tsx
<DeleteEntityDialog
  trigger={
    <DropdownMenuItem onSelect={e => e.preventDefault()} className="text-red-600">
      Delete
    </DropdownMenuItem>
  }
/>
```
Keep the menu open while the confirmation dialog appears by preventing the default `onSelect` behaviour.

## Best Practices & Gotchas
* Call `event.preventDefault()` inside `onSelect` to keep the dropdown open.
* Use `asChild` when you need semantic elements like `<a>` or custom buttons.
* Keep each item’s padding consistent for visual harmony.
* Avoid heavy interaction logic inside the item; delegate it to parent components.
