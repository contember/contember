## Overview
`DropdownMenuRadioItem` renders a selectable option inside a dropdown menu.
Use it when you need mutually exclusive choices within a menu.

## Styling Guidelines
* All snippets below include Tailwind classes.
* No Contember-specific theme classes are required, but you may add your own.

## Core Concepts
* A radio item represents one choice in a set.
* Only one radio item in the same group can be active.
* The component lives inside a dropdown menu context.

## Quick Start
```tsx
import { DropdownMenuRadioItem } from 'your-library';

export const QuickDemo = () => (
	<DropdownMenuRadioItem className="px-3 py-1.5 hover:bg-gray-100">
		Dark mode
	</DropdownMenuRadioItem>
);
```

## Examples

### Custom hover color
```tsx
<DropdownMenuRadioItem className="px-3 py-2 hover:bg-blue-100">
	Blue theme
</DropdownMenuRadioItem>
```
Hovering over the item changes its background to light blue.

### Disabled state
```tsx
<DropdownMenuRadioItem className="px-3 py-2 text-gray-400 cursor-not-allowed" disabled>
	Coming soon
</DropdownMenuRadioItem>
```
You can style a disabled item to signal unavailability.

## Best Practices & Gotchas
* Place radio items inside a logical group component for accessibility.
* Keep labels conciseâ€”long text breaks layout.
* Use `cursor-not-allowed` on disabled items to clarify intent.
* Test keyboard navigation; radio items should be reachable via arrow keys.
* Avoid mixing radio and non-radio items in one block.
