# DropdownMenuSeparator

## Overview
DropdownMenuSeparator renders a thin horizontal line that groups related items inside a dropdown menu.
It improves visual clarity by separating logical sections of `<DropdownMenuContent>` without adding extra spacing or padding.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* DropdownMenuSeparator inherits colors from the Contember theme via Radix primitives—no additional theme classes are required.

## Core Concepts
* Separates menu items into logical groups.
* Renders only inside `<DropdownMenuContent>`.
* Uses Radix UI defaults; you usually add only `className` for width or color tweaks.
* Accepts no custom props—behaves as a purely presentational component.

## Quick Start
```tsx
import {
  DropdownMenu,
  DropdownMenuTrigger,
  DropdownMenuContent,
  DropdownMenuItem,
  DropdownMenuSeparator,
} from '~/lib/ui/dropdown'

export const BasicDropdown = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <button className="px-3 py-2 rounded-md bg-primary-600 text-white">
        Actions
      </button>
    </DropdownMenuTrigger>

    <DropdownMenuContent className="w-48 rounded-lg shadow-lg bg-white">
      <DropdownMenuItem className="px-3 py-2">Edit</DropdownMenuItem>
      <DropdownMenuItem className="px-3 py-2">Duplicate</DropdownMenuItem>

      <DropdownMenuSeparator />

      <DropdownMenuItem className="px-3 py-2 text-destructive">Delete</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

### Repeater Item Actions
```tsx
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <DropDownTriggerButton />
  </DropdownMenuTrigger>

  <DropdownMenuContent className="w-[160px]">
    <DropdownMenuItem>Edit</DropdownMenuItem>
    <DropdownMenuItem>Make a copy</DropdownMenuItem>

    <DropdownMenuSeparator />

    <DeleteEntityDialog trigger={
      <DropdownMenuItem onSelect={e => e.preventDefault()}>
        Delete
      </DropdownMenuItem>
    } />
  </DropdownMenuContent>
</DropdownMenu>
```
Explanation: The separator isolates destructive `Delete` from routine actions, reducing accidental clicks.

## Best Practices & Gotchas
* Place separators only between items—not at the start or end of the menu.
* Combine destructive actions after a separator for safer UX.
* Avoid over-using separators; too many lines reduce readability.
* Do not pass onClick or other props—DropdownMenuSeparator ignores them.
