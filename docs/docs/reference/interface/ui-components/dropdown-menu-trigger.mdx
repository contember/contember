# DropdownMenuTrigger

## Overview
`DropdownMenuTrigger` toggles a `DropdownMenu` when the user interacts with its child element.  
It forwards events and refs to the wrapped element, letting you style or replace the trigger freely.

## Styling Guidelines
* All examples use Tailwind CSS.  
* No Contember-specific theme classes are required, but you can add project styles through the wrapped element.

## Core Concepts
* The trigger is **only a wrapper**—it renders nothing when `asChild` is true.  
* Interaction state is exposed through `data-[state=open|closed]` attributes for styling.  
* You must place `DropdownMenuTrigger` **inside** a `DropdownMenu` provider.  
* Any element (button, div, icon) can become a trigger via `asChild`.

## Quick Start
```tsx
import { DropdownMenu, DropdownMenuContent, DropdownMenuItem, DropdownMenuTrigger } from '~/lib/ui/dropdown'

export const SimpleDropdown = () => (
  <DropdownMenu>
    <DropdownMenuTrigger asChild>
      <button className="inline-flex items-center gap-1 rounded border px-3 py-1 text-sm hover:bg-gray-100">
        Actions
        <svg className="size-3" viewBox="0 0 10 6" fill="none"><path d="M1 1l4 4 4-4" stroke="currentColor"/></svg>
      </button>
    </DropdownMenuTrigger>

    <DropdownMenuContent className="w-40">
      <DropdownMenuItem>Profile</DropdownMenuItem>
      <DropdownMenuItem>Logout</DropdownMenuItem>
    </DropdownMenuContent>
  </DropdownMenu>
)
```

## Examples

### Icon Button Trigger
```tsx
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <button className="rounded-full p-2 hover:bg-gray-100">
      <svg className="size-4" viewBox="0 0 20 20" fill="currentColor"><path d="M10 3a1 1 0 110 2 1 1 0 010-2zm0 5a1 1 0 110 2 1 1 0 010-2zm0 5a1 1 0 110 2 1 1 0 010-2z"/></svg>
    </button>
  </DropdownMenuTrigger>
  <DropdownMenuContent side="bottom" className="w-32">
    <DropdownMenuItem>Edit</DropdownMenuItem>
    <DropdownMenuItem>Delete</DropdownMenuItem>
  </DropdownMenuContent>
</DropdownMenu>
```
Use `asChild` so the button remains untouched and accessible.

### Trigger Inside a Repeater
```tsx
<DropdownMenu>
  <DropdownMenuTrigger asChild>
    <Button variant="link" size="sm" className="gap-1 px-0">
      <PlusCircleIcon size={16} />
      <span>Add item</span>
    </Button>
  </DropdownMenuTrigger>

  <DropdownMenuContent side="top">
    <DropdownMenuItem>Add internal</DropdownMenuItem>
    <DropdownMenuItem>Add external</DropdownMenuItem>
  </DropdownMenuContent>
</DropdownMenu>
```
Mix Contember UI buttons with Tailwind utilities to keep design consistent.

## Best Practices & Gotchas
* Always wrap the trigger in a `DropdownMenu`; otherwise, it does nothing.  
* When the child already handles `onClick`, **call `event.preventDefault()`** inside menu items, not on the trigger.  
* Use the `data-[state=open]` attribute for styling active state instead of manual state tracking.  
* Set `asChild` for custom elements to avoid extra DOM nodes that could break flex layouts.  
* Ensure the trigger is **focusable** for keyboard users (e.g., `button` or `tabIndex={0}`).
