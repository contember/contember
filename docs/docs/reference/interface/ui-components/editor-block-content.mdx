# EditorBlockContent

## Overview
`EditorBlockContent` renders the editable region inside a block.
It manages `contentEditable`, placeholder text, and padding so you can focus on authoring.

## Styling Guidelines
* All examples use Tailwind CSS.
* No special Contember theme classes are required.

## Core Concepts
* Editable region lives inside each `EditorBlock`.
* Placeholder appears when the block is empty.
* Component handles focus and text-presence detection.
* Style the area with standard Tailwind utilities.

## Quick Start
```tsx
import { EditorBlock, EditorBlockContent } from '~/lib/editor'

export default () => (
	<EditorBlock name="paragraph" label="Paragraph" className="border p-4">
		<EditorBlockContent className="prose max-w-none" />
	</EditorBlock>
)
```


## Examples

### Standalone content area
```tsx
import { EditorBlockContent } from '~/lib/editor'

export const PlainContent = () => <EditorBlockContent />
```
Use this only inside a parent `EditorBlock`. It shows the simplest possible usage.

### Minimal quoted block (matches playground)
```tsx
import { EditorBlock, EditorBlockContent } from '~/lib/editor'

export const MinimalQuoteBlock = () => (
	<EditorBlock name="quote" label="Quote" className="border-l-4 pl-4">
		<EditorBlockContent />
	</EditorBlock>
)
```
The block gains a left border. `EditorBlockContent` takes care of editing and placeholder logic.

### Quoted block with custom background
```tsx
import {
	EditorBlock,
	EditorBlockContent,
} from '~/lib/editor'

export const QuoteBlock = () => (
	<EditorBlock
		name="quote"
		label="Quote"
		className="bg-gray-50 border-l-4 border-blue-500 p-4"
	>
		<EditorBlockContent className="italic text-gray-700" />
	</EditorBlock>
)
```
The block gets background and border styling, while `EditorBlockContent` handles inline editing.

## Best Practices & Gotchas
* Wrap `EditorBlockContent` only inside `EditorBlock`.
* Add padding on the parent block, not on the content, to keep the placeholder aligned.
* Use `prose` or similar Tailwind typography classes for rich text.
* Do not set `contentEditable` manually; the component already does that.
