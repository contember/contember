# EditorBlockToolbar

## Overview
EditorBlockToolbar renders a horizontal action bar inside the Contember rich-text editor.  
You place buttons or triggers inside it to let users insert block elements or references.

## Styling Guidelines
* All examples use Tailwind utility classes.  
* No Contember-specific theme classes are required; you may add your design system classes freely.

## Core Concepts
* EditorBlockToolbar is only a container – it renders its children unchanged.  
* Combine it with EditorReferenceTrigger or EditorElementTrigger to create actionable buttons.  
* Position the toolbar where your editor plugin expects block controls.  
* Tailwind flex utilities provide the quickest layout wins.

## Quick Start
```tsx
import { EditorBlockToolbar, EditorReferenceTrigger } from '~/lib/editor'

const BlockToolbar = () => (
	<EditorBlockToolbar>
		<EditorReferenceTrigger referenceType="image">
			<button className="px-2 py-1 bg-blue-600 text-white rounded">
				Add image
			</button>
		</EditorReferenceTrigger>
	</EditorBlockToolbar>
)
```

## Detailed Guide
```tsx
import {
	EditorBlockToolbar,
	EditorReferenceTrigger,
	EditorElementTrigger,
} from '~/lib/editor'
import { ImageIcon, TableIcon } from '~/icons'

export const RichTextBlockToolbar = () => (
	<EditorBlockToolbar>
		{/* Reference buttons */}
		<EditorReferenceTrigger referenceType="image">
			<button className="flex items-center gap-1 px-3 py-1 bg-gray-800 text-white rounded">
				<ImageIcon className="w-4 h-4" /> Image
			</button>
		</EditorReferenceTrigger>

		{/* Element buttons */}
		<EditorElementTrigger elementType="table">
			<button className="flex items-center gap-1 px-3 py-1 bg-gray-800 text-white rounded">
				<TableIcon className="w-4 h-4" /> Table
			</button>
		</EditorElementTrigger>
	</EditorBlockToolbar>
)
```

## Sub-Components / Building Blocks
### EditorReferenceTrigger
Wraps a button and inserts a reference (image, quote, etc.) when clicked.

### EditorElementTrigger
Inserts a custom block element such as a table or horizontal rule.

### BlockButton
Simple presentational button with Contember’s editor styling.

## Examples
### Insert Quote and Image
```tsx
<EditorBlockToolbar>
	<EditorReferenceTrigger referenceType="quote">
		<BlockButton><QuoteIcon /> Quote</BlockButton>
	</EditorReferenceTrigger>
	<EditorReferenceTrigger referenceType="image">
		<BlockButton><ImageIcon /> Image</BlockButton>
	</EditorReferenceTrigger>
</EditorBlockToolbar>
```
The toolbar now lets users add quoted text or upload an image.

### Extended Toolbar with Table and HR
```tsx
<EditorBlockToolbar>
	<EditorReferenceTrigger referenceType="quote">
		<BlockButton><QuoteIcon /> Quote</BlockButton>
	</EditorReferenceTrigger>
	<EditorElementTrigger elementType={tableElementType}>
		<BlockButton><TableIcon /> Table</BlockButton>
	</EditorElementTrigger>
	<EditorElementTrigger elementType={horizontalRuleElementType}>
		<BlockButton><MinusIcon /> Horizontal rule</BlockButton>
	</EditorElementTrigger>
</EditorBlockToolbar>
```
Users can now insert tables and horizontal rules alongside quotes.

## Best Practices & Gotchas
* Render EditorBlockToolbar inside the editor context; outside it triggers fail silently.  
* Keep button labels short; the toolbar should stay on one line.  
* Use flex-wrap if you expect many buttons.  
* Add hover and focus styles for accessibility.  
* Do not place complex layouts inside the toolbar – keep it lightweight.
