# EditorBlock

## Overview
EditorBlock registers reusable content blocks inside a Slate‑powered rich‑text editor.
You pass a unique `name`, a human‑readable `label`, and JSX defining the block body.

The component supports an optional alternate pop‑over for configuration. Drag‑and‑drop works out of the box, so authors reorder blocks effortlessly.

## Styling Guidelines
* All examples use Tailwind CSS.
* No Contember theme classes are required for basic usage.

## Core Concepts
* Give every block a stable, unique `name`.
* `label` appears in toolbars and pop‑overs.
* Render block internals through the `children` prop.
* Provide `alternate` to show a secondary settings UI.
* Wrap blocks with `<BlockEditorField>` for persistence and references.

## Quick Start
```tsx
import {
  BlockEditorField,
  EditorBlock,
  EditorBlockToolbar,
  EditorReferenceTrigger,
  EditorBlockContent,
} from '~/lib/editor'
import { ImageIcon } from 'lucide-react'

const PostEditor = () => (
  <BlockEditorField
    field="data"
    referencesField="references"
    referenceDiscriminationField="type"
    className="space-y-4"
  >
    <EditorBlockToolbar className="flex gap-2">
      <EditorReferenceTrigger referenceType="image">
        <button className="btn btn-primary flex items-center gap-1">
          <ImageIcon size={16} /> Image
        </button>
      </EditorReferenceTrigger>
    </EditorBlockToolbar>

    <EditorBlock name="image" label="Image">
      <ImageField baseField="image" urlField="url" />
    </EditorBlock>
  </BlockEditorField>
)
```

## Sub‑Components / Building Blocks
### EditorBlockContent
Renders a default rich‑text area inside a block.

### EditorBlockToolbar
Horizontal toolbar displayed above the editor.

### EditorInlineToolbar
Contextual toolbar appearing near selected text.

### EditorReferenceTrigger
Element that inserts a reference block, such as an image.

## Examples
### Quote Block
```tsx
<EditorBlock name="quote" label="Quote">
  <EditorBlockContent />
</EditorBlock>
```
This block gives authors a rich‑text area styled as a quote.

### Image Block
```tsx
<EditorBlock name="image" label="Image">
  <ImageField baseField="image" urlField="url" />
</EditorBlock>
```
Authors upload or select an image. The block stores the file and URL.

## Best Practices & Gotchas
* Keep `name` constant; changing it breaks stored content.
* Always nest `<EditorBlock>` inside one `<BlockEditorField>`.
* Prefer Tailwind classes via `className`; avoid global CSS.
* Use short `label`s to keep toolbars readable.
* Use `alternate` only for advanced settings to reduce UI complexity.
