## Overview
`EditorCanvas` renders an arbitrary React component—often Slate’s `Editable`—and forwards any props you supply.  
It lets you decorate the wrapped component with Contember-specific logic, focus handling, and optional visual focus rings.

You pass the target component through `underlyingComponent` and its props through `componentProps`; `EditorCanvas` takes care of the rest.

## Styling Guidelines
* All examples below use Tailwind CSS.  
* No special Contember theme classes are required, but you can add any Tailwind utilities to `className`.

## Core Concepts
* Canvas acts as a bridge between your editor logic and the UI.  
* All rendering responsibility stays inside the `underlyingComponent`.  
* You keep full control of props by passing them through `componentProps`.  
* `focusRing` toggles an optional visual outline.  
* Children render inside the same tree, so portals are unnecessary.

## Quick Start
```tsx
import { EditorCanvas, EditorEditableCanvas } from '~/lib/editor'
import { useSlateStatic } from 'slate-react'

const SimpleEditor = () => {
  const editor = useSlateStatic()

  return (
    <EditorCanvas
      underlyingComponent={EditorEditableCanvas}
      componentProps={{
        renderElement: editor.renderElement,
        renderLeaf: editor.renderLeaf,
        onKeyDown: editor.onKeyDown,
      }}
      className="border rounded p-4 focus:outline-none"
    >
      {/* Your Slate nodes go here */}
    </EditorCanvas>
  )
}
```

| focusRing | `boolean` | No | `false` | Shows a focus outline when `true`. |

## Examples
### Sortable Slate Blocks
```tsx
({ children, placeholder }: { children: ReactNode; placeholder?: string }) => {
  const editor = useSlateStatic()
  const entities = useRepeaterSortedEntities()

  const onDragEnd = ({ active, over }: DragEndEvent) => {
    if (!over) return
    const from = entities.findIndex(e => e.id === active.id)
    const to = entities.findIndex(e => e.id === over.id)
    if (from === undefined || to === undefined || from === to) return
    Transforms.moveNodes(editor, { at: [from], to: [to] })
  }

  return (
    <RepeaterSortable onDragEnd={onDragEnd} autoScroll={{ layoutShiftCompensation: false }}>
      <EditorCanvas
        underlyingComponent={EditorEditableCanvas}
        componentProps={{
          renderElement: editor.renderElement,
          renderLeaf: editor.renderLeaf,
          onKeyDown: editor.onKeyDown,
          onFocusCapture: editor.onFocus,
          onBlurCapture: editor.onBlur,
          onDOMBeforeInput: editor.onDOMBeforeInput,
          onDrop: e => e.preventDefault(),
          placeholder,
        }}
        className="border rounded p-4 bg-white"
      >
        {children}
      </EditorCanvas>
    </RepeaterSortable>
  )
}
```
This pattern lets users reorder blocks while preserving full Slate editing capabilities.

## Best Practices & Gotchas
* Keep `componentProps` memoized to avoid unnecessary rerenders.  
* Pass every editor callback (`renderElement`, `onKeyDown`, etc.) through `componentProps`; do not attach them directly to `EditorCanvas`.  
* Use `focusRing` only for visual cues; it does not manage focus state.  
* Avoid heavy logic in children; place it in the wrapped component instead.  
* Wrap sortable or draggable canvases with layout containers to keep DOM structure predictable.
