## Overview
`EditorEditableCanvas` renders an interactive canvas that supports live editing inside Contember’s editor environment.  
Use it when you need users to modify the content displayed by an `EditorCanvas`.

## Styling Guidelines
All examples on this page use Tailwind CSS utility classes.  
This component does not require any special Contember theme classes.

## Core Concepts
- Editable canvas shares the same visual space as `EditorCanvas`.
- Works only within a Contember `Editor` provider.
- Emits changes as users interact with the canvas.

## Quick Start
```tsx
import { Editor, EditorEditableCanvas } from '~/lib/editor'

export default function Page() {
  return (
    <Editor>
      <div className="border rounded p-4">
        <EditorEditableCanvas className="w-full h-64 bg-white" />
      </div>
    </Editor>
  )
}
```

## Best Practices & Gotchas
- Render `EditorEditableCanvas` inside an `Editor` context; otherwise, interactions fail silently.
- Wrap the canvas with a fixed height to prevent layout shifts.
- Avoid heavy inline styles; leverage Tailwind utilities for performance.
