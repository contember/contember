# EditorInlineToolbar

## Overview
EditorInlineToolbar groups inline formatting controls for Contember’s rich-text editor.  
You wrap buttons like `EditorMarkTrigger` or `EditorElementTrigger` inside it, and the toolbar handles focus management and keyboard navigation.

## Styling Guidelines
* All examples use Tailwind CSS.  
* EditorInlineToolbar renders no default styles—add Tailwind utility classes to the toolbar itself or to its children as needed.

## Core Concepts
* Toolbar is a plain `div` that hosts trigger components.
* Use inside `RichTextField` or any Slate-based editor in the Contember UI.
* Order of children defines button layout; group buttons with regular HTML elements (`div`, `span`) for logical sections.
* You can nest interactive components such as `Popover` for advanced inputs (e.g., link picker).

## Quick Start
```tsx
import { EditorInlineToolbar, EditorMarkTrigger } from '~/lib/editor'
import { boldMark } from '~/lib/editor/marks'
import { BoldIcon, Toggle } from '~/components/ui'

<EditorInlineToolbar className="flex gap-2">
  <EditorMarkTrigger mark={boldMark}>
    <Toggle><BoldIcon className="h-4 w-4" /></Toggle>
  </EditorMarkTrigger>
</EditorInlineToolbar>
```

## Detailed Guide
```tsx
import {
  EditorInlineToolbar,
  EditorMarkTrigger,
  EditorElementTrigger,
  EditorGenericTrigger,
} from '~/lib/editor'
import {
  boldMark,
  italicMark,
  paragraphElementType,
  createAlignHandler,
} from '~/lib/editor/registry'
import { BoldIcon, ItalicIcon, AlignLeftIcon, Toggle } from '~/components/ui'

<RichTextField field="content">
  <EditorInlineToolbar className="flex flex-wrap gap-3">
    {/* Text style group */}
    <div className="flex gap-1">
      <EditorMarkTrigger mark={boldMark}>
        <Toggle><BoldIcon className="h-3 w-3" /></Toggle>
      </EditorMarkTrigger>

      <EditorMarkTrigger mark={italicMark}>
        <Toggle><ItalicIcon className="h-3 w-3" /></Toggle>
      </EditorMarkTrigger>
    </div>

    {/* Alignment group */}
    <div className="flex gap-1 pl-3 border-l border-gray-300">
      <EditorGenericTrigger {...createAlignHandler('start')}>
        <Toggle><AlignLeftIcon className="h-3 w-3" /></Toggle>
      </EditorGenericTrigger>
    </div>

    {/* Block style group */}
    <EditorElementTrigger elementType={paragraphElementType}>
      <Toggle className="ml-3">P</Toggle>
    </EditorElementTrigger>
  </EditorInlineToolbar>
</RichTextField>
```

| children | React.ReactNode | Yes      | –       | Toolbar content: usually trigger components and groups.             |

**Inherited props:** _None_

## Sub-Components / Building Blocks
### EditorMarkTrigger
Wraps a button that sets or toggles a text mark (bold, italic, etc.).

### EditorElementTrigger
Switches the current block to a specific Slate element type (heading, list, paragraph).

### EditorGenericTrigger
Executes a custom command returned by helper functions such as `createAlignHandler`.

### Toggle
Reusable button that reflects active editor state through its `data-[state]` attribute.

### EditorInlineReferencePortal
Used inside popovers to edit object references (e.g., link target).

## Best Practices & Gotchas
* Keep groups small; overflowing buttons wrap or scroll depending on the container’s CSS.
* Wrap related triggers in `div` containers to apply group styling (borders, spacing).
* Add `aria-label` to toggles for accessibility.
* Toolbar itself is unstyled—forgetting to add `flex` or `gap` results in cramped icons.
* Do not place non-focusable elements (plain text) directly inside the toolbar—they break keyboard navigation order.
