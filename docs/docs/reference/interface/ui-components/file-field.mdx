# FileField

## Overview
`FileField` lets you upload any file type and persist its URL and optional metadata.
It wraps Contember’s upload client and renders a drag-and-drop zone with progress, validation and removal controls.

## Styling Guidelines
* All examples below use Tailwind utility classes.
* `FileField` does not require special Contember theme classes, but you may add custom styles as needed.

## Core Concepts
* Map each file attribute (URL, name, size, type, etc.) to a Contember field.
* Use `baseField` when the file lives in a related entity.
* Pass an `accept` object or `acceptFile` function to restrict uploads.
* Disable deletion with `noDestroy` when the file must remain permanent.
* Provide optional `dropzonePlaceholder` to guide users.

## Quick Start
```tsx
<FileField
  label="Document"
  urlField="file.url"
  className="w-full rounded border border-dashed border-gray-300 p-6"
/>
```

## Detailed Guide
```tsx
<FileField
  label="Document"
  description="Upload PDF only"
  baseField="document"
  urlField="file.url"
  fileNameField="fileName"
  fileSizeField="fileSize"
  fileTypeField="fileType"
  lastModifiedField="lastModified"
  dropzonePlaceholder="Drag PDF here"
  accept={{ 'application/*': ['.pdf'] }}
  className="w-full rounded border-2 border-dashed border-indigo-400 p-8 text-center"
/>
```
The snippet:
* Stores the actual file in the related `document` entity (`baseField`).
* Saves additional metadata into dedicated fields.
* Limits uploads to any PDF MIME type.
* Shows a branded dropzone with Tailwind classes.

## Examples

### Attachments in a Repeater
```tsx
<DefaultRepeater field="files" label="Attachments" orderBy="url" className="space-y-4">
  <FileField
    label="Attachment file"
    urlField="url"
    fileNameField="fileName"
    fileSizeField="fileSize"
    fileTypeField="fileType"
    className="w-full"
  />
  <RepeaterItemActions>
    <RepeaterRemoveItemButton />
  </RepeaterItemActions>
</DefaultRepeater>
```
Use this pattern when an entity can contain multiple files.

### Read-only Mandate Upload
```tsx
<FileField
  baseField="mandate"
  urlField="url"
  fileNameField="fileName"
  fileSizeField="fileSize"
  fileTypeField="fileType"
  label="Mandate"
  noDestroy
  className="w-full"
/>
```
The `noDestroy` flag prevents accidental deletion of legally binding documents.

## Best Practices & Gotchas
* Always set `urlField`—nothing uploads without it.
* Reserve `baseField` for relations; omit it for scalar fields.
* Validate file types with both `accept` and server-side rules.
* Do not store large blobs inside Contember, only URLs and metadata.
* Pass a stable `uploader` instance to reuse connections.
* Remember to handle `errors` in parent forms.
