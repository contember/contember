## Overview
`FormContainerUI` wraps one or more form fields and their labels.  
It guarantees consistent vertical spacing and a full-width layout.

## Styling Guidelines
* All examples use Tailwind.
* The component already includes `flex flex-col gap-2 w-full`; you do not need to add them.

## Core Concepts
* Group related form inputs visually.
* Apply consistent vertical rhythm across the whole form.
* Keep markup flat; place labels and inputs as direct children.

## Quick Start
```tsx
import { FormContainerUI, FormLabelUI, InputField } from '~/lib/form'

export const EmailField = () => (
  <FormContainerUI>
    <FormLabelUI>Email</FormLabelUI>
    <InputField field="email" />
  </FormContainerUI>
)
```

## Detailed Guide
```tsx
<FormContainerUI>
  <FormLabelWrapperUI>
    <FormLabelUI required>Password</FormLabelUI>
  </FormLabelWrapperUI>

  <InputField field="password" type="password" className="max-w-sm" />

  <FormDescriptionUI>
    Use at least 8 characters and one symbol.
  </FormDescriptionUI>
</FormContainerUI>
```
* Wrap `FormLabelUI` in `FormLabelWrapperUI` when you need extra layout control.  
* Place optional helpers like `FormDescriptionUI` after the input.

## Examples

### Toggle-controlled date picker
```tsx
<FormContainerUI>
  <FormLabelUI className="flex items-start">
    <Switch checked={!!value} onCheckedChange={toggle} />
    <span className="ml-2">Activate from</span>
  </FormLabelUI>

  {value && wantsSelectDate && (
    <FormInput field="activeFrom">
      <Input className="max-w-md" />
    </FormInput>
  )}
</FormContainerUI>
```
The switch sets or clears the date. When active, the user can pick a custom date.

### Read-only preview inside a relation
```tsx
<FormContainerUI>
  <FormLabelWrapperUI>
    <FormLabelUI>Lecturer</FormLabelUI>
  </FormLabelWrapperUI>

  <HasOne field="lecturer">
    <LecturerPreview />
  </HasOne>
</FormContainerUI>
```
You can combine the container with Contember relation helpers to show previews instead of inputs.

## Best Practices & Gotchas
* Always place labels inside the container; do not float them outside.
* Keep child hierarchy shallow. Deep nesting breaks the flex gap.
* Do not override `w-full` unless you truly need narrower content.
* Use one container per logical field group for better accessibility.
