# FormContainer

## Overview
`FormContainer` wraps form inputs with consistent markup and accessibility helpers.
It renders an optional label, description, and error list while managing the required state inside Contember forms.

The component accepts any React children. You embed inputs, editors, or even nested form scopes to keep your UI coherent.

## Styling Guidelines
* All snippets use Tailwind CSS.
* No Contember-specific theme class is required; style with regular Tailwind utilities or your own classes.

## Core Concepts
* Wrap each input into a single `FormContainer`.
* Provide `label`, `description`, and `errors` to improve accessibility.
* Pass `required` to show an asterisk and enable HTML validation.
* Nest arbitrary React nodes inside the `children` prop.

## Quick Start
```tsx
import { FormContainer } from '~/lib/form'

<FormContainer
  label="Email"
  description="Enter a valid email address"
  required
  errors={errors.email}
>
  <FormInput field="email" className="max-w-md" />
</FormContainer>
```

## Examples

### Checkbox list with field scope
```tsx
<FormFieldScope field="roles">
  <FormContainer label="User roles" description="Select at least one role">
    <CheckboxListEnumFieldInner field="roles" />
  </FormContainer>
</FormFieldScope>
```
The scope binds nested inputs to the `roles` field. `FormContainer` adds a label and description automatically.

### Select or type pattern
```tsx
<FormFieldScope field="category">
  <FormContainer label="Category">
    {showSelect ? (
      <Select
        value={value ?? undefined}
        onValueChange={handleSelect}
        className="max-w-md"
      >
        {/* ...options */}
      </Select>
    ) : (
      <FormInput field="category">
        <Input className="max-w-md" />
      </FormInput>
    )}
  </FormContainer>
</FormFieldScope>
```
The component hosts either a select or an input based on state, yet keeps consistent labeling and error handling.

## Best Practices & Gotchas
* Always pass `errors` from your form validation logic.
* Keep one `FormContainer` per logical field.
* Avoid wrapping multiple unrelated fields; it harms accessibility.
* Put layout classes such as `flex gap-2` on the child elements, not on `FormContainer`.
