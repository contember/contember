## Overview
FormDescriptionUI shows secondary help text under a form field.
It renders muted, small text to guide users without stealing focus.

## Styling Guidelines
* All examples use Tailwind.
* The component applies `text-[0.8rem] text-muted-foreground`.
  No extra Contember theme classes are required.

## Core Concepts
* Display contextual help below any input.
* Keep wording short; it supplements, not replaces, labels.
* Combine with FormLabelUI and FormContainerUI for consistent spacing.

## Quick Start
```tsx
import { FormDescriptionUI } from '~/lib/form'

export const PasswordField = () => (
  <div className="flex flex-col gap-1">
    <input type="password" className="border rounded px-2 py-1" />
    <FormDescriptionUI>
      Must be at least 8 characters
    </FormDescriptionUI>
  </div>
)
```

## Examples

### Conditional description inside a form container
You can show the description only when the author provides one.

```tsx
interface FieldProps {
  label: string
  description?: string
}

export const TextField = ({ label, description }: FieldProps) => (
  <FormContainerUI>
    <FormLabelWrapperUI>
      <FormLabelUI>{label}</FormLabelUI>
    </FormLabelWrapperUI>

    <input className="border rounded w-full px-2 py-1" />

    {description && (
      <FormDescriptionUI>{description}</FormDescriptionUI>
    )}
  </FormContainerUI>
)
```
The wrapper keeps vertical rhythm while the description remains optional.

### Muted help text with interactive controls
Combine FormDescriptionUI with switches or buttons.

```tsx
export const PublishSwitch = ({ description }: { description: string }) => {
  const [enabled, setEnabled] = useState(false)

  return (
    <div className="flex flex-col gap-1">
      <label className="flex items-center gap-2">
        <input
          type="checkbox"
          checked={enabled}
          onChange={e => setEnabled(e.target.checked)}
        />
        Publish now
      </label>

      {!enabled && (
        <FormDescriptionUI>
          {description}
        </FormDescriptionUI>
      )}
    </div>
  )
}
```
The text explains why a switch is off, improving clarity.

## Best Practices & Gotchas
* Keep descriptions short—one sentence is usually enough.
* Do not duplicate the label text. Describe how or why, not what.
* Avoid actions inside the description. Use buttons or links instead.
* Ensure sufficient color contrast against backgrounds.
