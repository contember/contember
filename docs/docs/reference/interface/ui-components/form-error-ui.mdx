## Overview
FormErrorUI displays form validation errors in a unified destructive style.  
It renders its children as red, bold, small text, making issues easy to spot.

## Styling Guidelines
* All examples use Tailwind CSS.  
* The component already applies `text-[0.8rem] font-medium text-destructive`. No additional Contember theme classes are required.

## Core Concepts
* Error messages must stand out yet stay compact.  
* Consistent styling improves accessibility and user trust.  
* You usually combine FormErrorUI with higher-level `FormError` logic.

## Quick Start
```tsx
import { FormErrorUI } from '~/lib/form'

export default function Example() {
	return (
		<FormErrorUI>Invalid email format</FormErrorUI>
	)
}
```

## Detailed Guide
When you validate a form, you often collect errors at field level. Wrap FormErrorUI in a higher-order component that decides when to show the message.

```tsx
import { FormError } from '@contember/react-form'
import { FormErrorUI } from '~/lib/form'

export default function EmailField() {
	return (
		<FormError field="email">
			<FormErrorUI />
		</FormError>
	)
}
```

`FormError` injects the current error text as children. FormErrorUI then handles the visual presentation.

## Examples
### With custom formatter
```tsx
import { FormError, useErrorFormatter } from '@contember/react-form'
import { FormErrorUI } from '~/lib/form'

export default function PasswordField() {
	const formatter = useErrorFormatter()

	return (
		<FormError field="password" formatter={formatter}>
			<FormErrorUI />
		</FormError>
	)
}
```
The `formatter` converts error codes to readable messages. FormErrorUI keeps styling consistent.

## Best Practices & Gotchas
* Show only one error message per field to avoid clutter.  
* Keep the message short; the component uses small font size.  
* Do not add extra margins; let the parent control layout spacing.  
* Combine with ARIA attributes on the related form field for accessibility.
