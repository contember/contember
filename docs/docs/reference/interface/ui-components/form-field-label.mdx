## Overview
`FormFieldLabel` renders a label for the current form field inside a Contember form.
You typically place it inside a field scope so it can automatically resolve the field’s metadata and translation key.

## Styling Guidelines
* All examples use Tailwind CSS.
* No special Contember theme classes are required.

## Core Concepts
* Form scope – the component reads the current `field` context.
* Accessibility – the component outputs a standard `<label>` element.
* Composition – you can combine it with cards, inputs, or error components.

## Quick Start
```tsx
import { FormFieldLabel } from '~/lib/form/labels'

export const SimpleInput = () => (
	<div className="flex flex-col gap-2">
		<FormFieldLabel />
		<input type="text" className="border rounded px-3 py-2" />
	</div>
)
```
## Examples
```tsx
<FormHasOneRelationScope field="profile">
	<HasOne field="profile">
		<DefaultCard
			title={<FormFieldLabel />}
			description={<Errors />}
			seamless
			className="mb-6"
		>
			<ResetFieldState>
				{/* form fields go here */}
			</ResetFieldState>
		</DefaultCard>
	</HasOne>
</FormHasOneRelationScope>
```
Inside a relation scope, `FormFieldLabel` automatically picks up the `profile` field and displays its localized label.

## Best Practices & Gotchas
* Always place `FormFieldLabel` inside a field scope; otherwise it renders nothing.
* Pair it with `Errors` to keep labels and validation messages together.
* Keep label text short. Long labels break layouts on small screens.

## Further Reading
* `HasOne` – wrapper for one-to-one relations.
* `DefaultCard` – opinionated container for form sections.
* `Errors` – component for field validation messages.
