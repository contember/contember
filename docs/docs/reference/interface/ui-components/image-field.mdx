# ImageField

## Overview
`ImageField` lets you upload and preview images inside a Contember form. It wraps the generic file upload with image–specific conveniences such as drag-and-drop and an automatic thumbnail preview.

Use it only inside an entity context (`<EntitySubTree />` or `<EntityListSubTree />`). The component stores the upload metadata (url, dimensions, etc.) into the current entity fields.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* Contember’s default theme already styles form labels, inputs, and error messages—no extra classes are required unless you want custom layouts.

## Core Concepts
* Works **only** inside an entity subtree.
* Persists the uploaded file’s URL (and optional metadata) to designated fields.
* Shows a live preview via `UploadedImageView`.
* Accepts custom drag-and-drop UI through `dropzonePlaceholder`.
* You can disable the “remove” button with `noDestroy`.

## Quick Start
```tsx
import { Binding, EntitySubTree, PersistButton } from '@contember/react-binding'
import { ImageField } from '~/lib/form'

const UserForm = () => (
  <Binding>
    <EntitySubTree entity="User(id = $id)">
      <ImageField
        label="Profile picture"
        urlField="avatar.url"
        dropzonePlaceholder="Drag image here"
      />
      <PersistButton className="mt-4" />
    </EntitySubTree>
  </Binding>
)
```

## Sub-Components / Building Blocks
### UploadedImageView
Renders the thumbnail preview once the upload is complete. You rarely use it directly—it is integrated into `ImageField`.

### UploaderDropzoneAreaUI
Reusable drop-area wrapper. Pass it to `dropzonePlaceholder` to replace the default minimal placeholder.

## Examples

### Custom drop-zone with browse button
```tsx
<ImageField
  baseField="image"
  urlField="url"
  fileNameField="fileName"
  dropzonePlaceholder={(
    <UploaderDropzoneAreaUI className="w-60 p-4 border rounded-md text-center">
      <UploadIcon className="w-12 h-12 text-gray-400 mx-auto" />
      <div className="font-semibold text-sm">Drop image here</div>
      <div className="text-xs">or</div>
      <Button size="sm" variant="outline" className="mt-1">Browse</Button>
    </UploaderDropzoneAreaUI>
  )}
/>
```
The custom placeholder keeps the default uploader logic while giving you full control over the visuals.

### Disabling file removal
```tsx
<ImageField
  label="Company logo"
  urlField="logo.url"
  noDestroy
/>
```
Set `noDestroy` to prevent accidental deletion of mandatory files.

## Best Practices & Gotchas
* Always wrap the component inside an entity subtree; otherwise it has nowhere to save data.
* Prefer dedicated image entities (`baseField`) when you need extra metadata (width/height).
* Supply `accept` to limit formats, e.g. `{ 'image/*': [] }`.
* Stop event propagation in custom placeholders if you embed interactive elements (buttons, selects).
* When using `acceptFile`, throw `RejectFileError` to display a human-friendly error.
