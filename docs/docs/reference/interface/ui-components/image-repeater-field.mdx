## Overview
ImageRepeaterField lets you upload, reorder, and manage multiple images inside an Entity context.  
It displays previews, upload progress, and removal controls out-of-the-box.

## Styling Guidelines
* All snippets use Tailwind CSS utility classes.  
* The component respects Contember UI theme classes applied higher in the tree; no extra theme classes are required.

## Core Concepts
* Runs only inside `<EntitySubTree />` or `<EntityListSubTree />`.
* Manages a list of images: upload, preview, reorder, remove.
* Supports drag-and-drop plus manual file browsing.
* Maps entity fields via props (e.g. `urlField`, `widthField`).
* Accepts a custom `dropzonePlaceholder` to fully style the drop area.

## Quick Start
```tsx
<EntitySubTree entity="Gallery(unique = unique)">
  <ImageRepeaterField
    label="Gallery images"
    field="images"
    urlField="image.url"
    orderBy="createdAt"
    className="w-full"
  />
</EntitySubTree>
```

## Detailed Guide
### Custom dropzone and extended metadata
The example below stores additional image metadata and replaces the default dropzone with a bespoke UI.

```tsx
<ImageRepeaterField
  field="imageList.items"
  baseField="image"
  sortableBy="order"
  urlField="url"
  widthField="width"
  heightField="height"
  fileNameField="fileName"
  fileSizeField="fileSize"
  fileTypeField="fileType"
  lastModifiedField="lastModified"
  label="Image file"
  description="Store every detail about your image."
  dropzonePlaceholder={(
    <UploaderDropzoneAreaUI className="w-60">
      <UploadIcon className="w-12 h-12 text-gray-400" />
      <div className="font-semibold text-sm">Drop files here</div>
      <div className="text-xs">or</div>
      <div className="flex gap-2 items-center text-xs">
        <Button size="sm" variant="outline">Browse</Button>
        <div onClick={e => e.stopPropagation()}>
          <SelectImageRepeater />
        </div>
      </div>
    </UploaderDropzoneAreaUI>
  )}
/>
```

## Examples
### Use inside a multidimensional form locale
```tsx
<FormLocales>
  <ImageRepeaterField
    field="gallery.items"
    baseField="image"
    sortableBy="displayOrder"
    urlField="url"
    widthField="width"
    heightField="height"
    fileNameField="fileName"
    fileTypeField="fileType"
    fileSizeField="fileSize"
    lastModifiedField="lastModified"
    className="mt-4"
  />
</FormLocales>
```
This snippet shows how to manage translated galleries. The component lives in each locale’s entity subtree and writes metadata in every language context.

### Display custom overlay controls
```tsx
<ImageRepeaterField
  field="photos"
  urlField="image.url"
  sortableBy="order"
  withoutUploader
>
  <div className="relative">
    <div className="absolute top-1 right-1 z-10 bg-white/80 py-0.5 px-1 rounded">
      <CheckboxField field="isUsed" label="Use image" />
    </div>

    <UploadedImageView
      urlField="image.url"
      DestroyAction={RepeaterRemoveItemTrigger}
    />
  </div>
</ImageRepeaterField>
```
`withoutUploader` hides the default upload UI. You supply custom children to render each item, including your own removal trigger.

## Best Practices & Gotchas
* Always wrap the component in an Entity or EntityList subtree; otherwise nothing will render.  
* Keep `sortableBy` and `orderBy` in sync with your backend ordering logic.  
* Use `withoutUploader` when you only need to edit metadata, not upload new images.  
* Provide a stable key for each image in the database to avoid accidental re-ordering.  
* Large files slow the preview generation; consider client-side compression before upload.
