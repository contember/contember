# `InputBare`

## Overview
`InputBare` renders a raw HTML `<input>` element.  
The component ships **no** visual styles—no border, shadow, focus ring, or spacing.

Use `InputBare` when you want complete control over an input’s appearance or when composing higher-level inputs.

## Styling Guidelines
* All code samples use Tailwind CSS classes.  
* No Contember theme classes are applied by default; you must add your own.

## Core Concepts
* `InputBare` is a thin wrapper around `<input>`.
* It forwards every native input attribute.
* You must add padding, borders, focus rings, and colors yourself.
* Combine it with `InputLike` or custom wrappers for complex inputs.

## Quick Start
```tsx
import { InputBare } from '@app/lib/ui/input'

export default function EmailField() {
  return (
    <InputBare
      type="email"
      placeholder="you@example.com"
      className="w-full px-3 py-2 border rounded-md focus:outline-none focus:ring focus:border-blue-500"
    />
  )
}
```

## Examples

### Prefixed Input with External Link
Adds a prefix and a shortcut link using `InputLike`.

```tsx
import { InputBare, InputLike } from '@app/lib/ui/input'
import { ExternalLinkIcon } from '@heroicons/react/outline'

export function UrlInput(props) {
  return (
    <InputLike className="relative">
      <span className="-my-2 -ml-2 text-gray-400 self-stretch py-1 pl-2 flex items-center">https://</span>
      <InputBare
        {...props}
        className="pr-1 flex-1"
      />
      <a
        href={`https://${props.value ?? ''}`}
        target="_blank"
        rel="noreferrer"
        className="ml-auto self-stretch flex items-center px-2 text-gray-600 bg-gray-50 rounded-r-md border-l hover:bg-gray-100 -my-2 -mr-2"
      >
        <ExternalLinkIcon className="h-4 w-4" />
      </a>
    </InputLike>
  )
}
```
This pattern keeps the prefix and link outside the editable area while sharing a unified border.

### Command Palette Path Input
Shows a static path separator before the editable part.

```tsx
<InputLike className="relative">
  <span className="-my-2 -ml-2 text-gray-400 self-stretch py-1 pl-2 flex items-center">/</span>
  <InputBare
    placeholder="type-command"
    className="pr-1 w-full"
  />
</InputLike>
```
Use this when the first character of the path is fixed and should not be editable.

## Best Practices & Gotchas
* Always add focus styles (`focus:ring`, `focus:border-*`) for accessibility.
* Set `className="flex-1"` when you need the input to fill remaining space.
* Use `InputLike` to share borders between addons, prefixes, or suffixes.
* Remember to forward refs when wrapping `InputBare` in your own component.

## Further Reading
