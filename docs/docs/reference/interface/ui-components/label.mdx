## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* The real-world snippets rely on Contember theme tokens such as `fill-foreground` and `fill-muted-foreground`.

## Core Concepts
* `Label` renders SVG or text inside Recharts primitives.  
* You can pass either plain children or a `content` render function to gain full control.  
* Tailwind classes let you style fills, typography, and spacing without extra CSS.

## Quick Start
```tsx
import { Label } from 'recharts'

export const Example = () => (
	<Label className="fill-foreground text-sm">Total</Label>
)
```

## Examples

### Centered number inside a pie chart
```tsx
<Pie data={chartData} dataKey="users" nameKey="status" innerRadius={60} strokeWidth={5}>
	<Label
		content={({ viewBox }) => (
			viewBox && 'cx' in viewBox && 'cy' in viewBox ? (
				<text x={viewBox.cx} y={viewBox.cy} textAnchor="middle" dominantBaseline="middle">
					<tspan x={viewBox.cx} y={viewBox.cy} className="fill-foreground text-3xl font-bold">
						{totalOnlineCourseProgressCount}
					</tspan>
					<tspan x={viewBox.cx} y={(viewBox.cy || 0) + 24} className="fill-muted-foreground">
						{componentLabels.dashboard.eLearningPie.insideDescription}
					</tspan>
				</text>
			) : null
		)}
	/>
</Pie>
```
This pattern places dynamic numbers inside a donut chart. The first `tspan` shows the main figure, while the second displays a subtitle.

### Stand-alone label in forms
```tsx
<Label className="text-muted-foreground">{entityLabels.Job.fields.name}</Label>
```
Use `Label` outside charts when you only need styled text pulled from localization files.

## Best Practices & Gotchas
* Always check `viewBox` keys (`cx`, `cy`) before accessing them.  
* Keep `tspan` offsets relative, not absolute, for responsive layouts.  
* Prefer Tailwind color tokens (`fill-foreground`) so themes remain swappable.  
* Wrap long strings in `tspan` elements to avoid SVG text clipping.
