## Overview
`LinkBackButton` is a navigational button that routes users to a target path.  
It wraps its content in `Slots.Back` and renders an `AnchorButton`, giving you a consistent place for “Back” actions across your app.

## Styling Guidelines
* All examples use Tailwind CSS.  
* No special Contember theme classes are required; use your own Tailwind utilities.

## Core Concepts
* Routing link target (`to`) defines where the button navigates.
* `Slots.Back` ensures consistent placement in Contember layouts.
* `AnchorButton` provides anchor semantics with button styling.
* Passing `children` lets you replace the default label and icon.

## Quick Start
```tsx
import { LinkBackButton } from '@contember/react'

export const ArticleListHeader = () => (
	<LinkBackButton
		to="articles"
		label="Back to Articles"
		className="inline-flex items-center gap-2 text-blue-600 hover:underline"
	/>
)
```

## Detailed Guide
### Replace the Label with a Custom Icon
```tsx
<LinkBackButton to="dashboard">
	<ArrowLeftIcon className="w-4 h-4 text-gray-600" />
</LinkBackButton>
```
The icon-only button saves space in dense toolbars.

### Combine Icon and Text
```tsx
<LinkBackButton to="dashboard">
	<ArrowLeftIcon className="w-4 h-4" /> Dashboard
</LinkBackButton>
```
You keep a recognisable icon while providing text for accessibility.

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| children | React.ReactNode | No | `undefined` | Custom button content that replaces the label and default icon |
| label | React.ReactNode | Yes | – | Text or node displayed as the button label when `children` is not provided |
| to | RoutingLinkTarget | Yes | – | Destination route |
| className | string | No | `undefined` | Tailwind classes forwarded to the underlying anchor |
| All AnchorButton props | See AnchorButton | No | – | Any additional props accepted by `AnchorButton` |

## Sub-Components / Building Blocks
### Slots.Back
Wraps the button so that layout components can render all “back” links in a consistent place.

### AnchorButton
Underlying component that renders an `<a>` element styled like a button.  
All non-specific props pass through to it.

## Examples
### Back to Article List in a Detail Page
```tsx
const ArticleDetailHeader = () => (
	<header className="flex items-center justify-between border-b pb-2 mb-4">
		<LinkBackButton to="articles" label="Back" className="text-sm text-blue-500" />
		<h1 className="text-lg font-semibold">Article detail</h1>
	</header>
)
```
Use the button in headers to maintain consistent navigation.

### Compact Sidebar Back Button
```tsx
<LinkBackButton to="dashboard" className="p-2 hover:bg-gray-100 rounded">
	<ArrowLeftIcon className="w-5 h-5" />
</LinkBackButton>
```
Icon-only variant fits sidebars where space is limited.

## Best Practices & Gotchas
* Always pass a valid `to` target; the component will not fallback to `history.back()`.
* Provide either `label` or `children`, not both, to avoid duplicate content.
* Forward `className` for Tailwind customization instead of wrapping in extra elements.
* Place the button inside `Slots.Back` aware layouts to avoid unexpected placement.
