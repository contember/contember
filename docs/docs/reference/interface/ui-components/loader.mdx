## Styling Guidelines
* All code samples use Tailwind classes.
* `Loader` has no mandatory Contember theme classes.

## Core Concepts
* Show a visual indicator while data loads.
* Render conditionally based on fetch or mutation state.
* Keep the spinner lightweight to avoid blocking the main thread.

## Quick Start
```tsx
import { Loader } from '~/lib/ui/loader'

export const Page = () => (
	<div className="flex items-center justify-center h-screen">
		<Loader />
	</div>
)
```

## Examples

### Conditional entity load
```tsx
const UserCard = () => {
	const [user, setUser] = useState<User | null>(null)

	useEffect(() => {
		fetch('/api/user')
			.then(res => res.json())
			.then(setUser)
	}, [])

	if (!user) {
		return <Loader />
	}

	return (
		<div className="p-4 rounded border shadow">
			<p className="font-semibold">{user.name}</p>
		</div>
	)
}
```
The component shows `Loader` until the user object arrives, then renders the card.

### Inline form feedback
```tsx
const SaveButton = ({ onSave }: { onSave: () => Promise<void> }) => {
	const [saving, setSaving] = useState(false)

	const handleClick = async () => {
		setSaving(true)
		await onSave()
		setSaving(false)
	}

	return (
		<button
			onClick={handleClick}
			className="flex items-center gap-2 px-4 py-2 rounded bg-blue-600 text-white"
		>
			{saving && <Loader size="sm" />}
			<span>{saving ? 'Saving…' : 'Save'}</span>
		</button>
	)
}
```
`Loader` augments the button label without disrupting layout.

## Best Practices & Gotchas
* Place the spinner close to the data it represents.
* Memoize expensive child elements to prevent unnecessary re-renders.
* Prefer the `sm` size inside buttons to keep alignment tidy.
* Do not rely on undocumented props; stick to `size`.
