# LoginWithEmail

## Overview
`LoginWithEmail` authenticates users by email.  
It works with session tokens to establish a signed‑in state.

## Styling Guidelines
* All examples use Tailwind.
* The component itself has no required Contember theme classes.

## Core Concepts
* Email‑based authentication flow.
* Session token creation and storage.
* Drop‑in usage: no props required.
* Composition inside application shells, DevBars, or standalone pages.

## Quick Start
```tsx
import { LoginWithEmail } from '~/lib/dev'

export default function Page() {
	return (
		<div className="flex min-h-screen items-center justify-center bg-gray-50">
			<LoginWithEmail />
		</div>
	)
}
```

## Examples

### Basic Usage
```tsx
import { LoginWithEmail } from '~/lib/dev'

export function Login() {
	return <LoginWithEmail />
}
```
Render the component wherever you need a simple login form.  
It handles validation and submission internally.

### Embedded in a DevBar
```tsx
import { DevBar, DevPanel } from '@contember/ui-devbar'
import { LogInIcon } from 'lucide-react'
import { LoginWithEmail } from '~/lib/dev'

export function DevTools() {
	return (
		<DevBar>
			<DevPanel heading="Login" icon={<LogInIcon />} >
				<LoginWithEmail />
			</DevPanel>
		</DevBar>
	)
}
```
Place the login form in a development toolbar.  
You can switch accounts without leaving the current page.

### Full Application Entrypoint
```tsx
import { ApplicationEntrypoint } from '@contember/react-client'
import { Pages } from '@contember/react-pages'
import { Toaster } from '@contember/ui'
import { DevBar, DevPanel } from '@contember/ui-devbar'
import { LogInIcon } from 'lucide-react'
import { LoginWithEmail } from '~/lib/dev'
import { Layout } from './Layout'
import { OutdatedApplicationDialog } from './OutdatedApplicationDialog'
import { SlotsProvider } from '@contember/ui-slots'
import { FieldLabelFormatterProvider } from '@contember/react-form'
import { EnumOptionsFormatterProvider } from '@contember/react-form'
import { getConfig } from './config'

export function App() {
	return (
		<FieldLabelFormatterProvider value={fieldLabelFormatter}>
			<EnumOptionsFormatterProvider value={enumOptionsFormatter}>
				<SlotsProvider>
					<ApplicationEntrypoint
						{...getConfig()}
						children={
							<>
								<Toaster>
									<Pages
										layout={Layout}
										children={import.meta.glob<PageModule>(
											'./pages/**/*.tsx',
											{ eager: true },
										)}
									/>
									{import.meta.env.DEV && (
										<DevBar>
											<DevPanel heading="Login" icon={<LogInIcon />}>
												<LoginWithEmail />
											</DevPanel>
										</DevBar>
									)}
								</Toaster>
								<OutdatedApplicationDialog />
							</>
						}
					/>
				</SlotsProvider>
			</EnumOptionsFormatterProvider>
		</FieldLabelFormatterProvider>
	)
}
```
In this real‑world setup you conditionally show the login panel only during development.  
Production builds do not include the DevBar, keeping the UI clean.

## Best Practices & Gotchas
* Keep the component high in the tree to avoid nested authentication prompts.
* Wrap it in `Suspense` if you load authentication logic lazily.
* Do not place it inside forms; it manages its own submit events.
* Avoid styling overrides that hide default validation states.
