## Overview
MemberList renders a table of project members.
You configure the table headers, columns, labels, and an optional controller.

## Styling Guidelines
* All examples use Tailwind CSS.
* No extra Contember theme classes are required.

## Core Concepts
* Use `filter` to limit displayed members.
* `tableHeaders` defines visible column titles.
* `tableColumns` maps a member entity to JSX.
* `labels` localize delete-related texts.
* `controller` lets you access imperative actions.

## Quick Start
```tsx
import { MemberList } from '@contember/react-ui'
import { ProjectMembersFilter } from '@contember/graphql-client-tenant'

export const Page = () => {
	const filter: ProjectMembersFilter = {} // show all members

	return (
		<MemberList
			filter={filter}
			tableHeaders={['ID']}
			tableColumns={member => (
				<span className="font-mono text-sm">{member.identity.id}</span>
			)}
			labels={{
				deleteConfirmation: 'Remove this member?',
				deleted: 'Member removed',
				deleteFailed: 'Failed to remove member',
			}}
			className="w-full border rounded"
		/>
	)
}
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| filter | ProjectMembersFilter | Yes | — | GraphQL filter for members. |
| labels | `{ deleteConfirmation: string; deleted: string; deleteFailed: string; }` | Yes | — | Localized strings for delete flow. |
| tableColumns | `(member) => React.ReactNode` | Yes | — | Function returning JSX for each row. |
| tableHeaders | `string[]` | Yes | — | Titles shown above columns. |
| controller | `{ current?: MemberListController }` | No | — | Ref object exposing list controls. |
| roles | `RolesConfig` | No | — | Config that enriches role rendering. |

## Examples
### Custom Role Badges
```tsx
<MemberList
	filter={{}}
	tableHeaders={['Name', 'Roles']}
	tableColumns={member => (
		<div className="flex gap-4">
			<span>{member.person?.name ?? '—'}</span>
			<div className="flex gap-1">
				{member.roles?.map(role => (
					<span
						key={role}
						className="rounded bg-blue-100 px-2 py-0.5 text-xs text-blue-800"
					>
						{role}
					</span>
				))}
			</div>
		</div>
	)}
	labels={{
		deleteConfirmation: 'Really delete?',
		deleted: 'Done',
		deleteFailed: 'Oops',
	}}
/>
```
The example shows how to decorate each role with a badge.

## Best Practices & Gotchas
* Keep `tableHeaders` length equal to `tableColumns` output cells.
* Memoize `tableColumns` to avoid re-render loops.
* Provide clear delete labels to prevent accidents.
* Use the `controller` ref for refreshes instead of remounting.
