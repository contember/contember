# MenuItem

## Overview
_None_

## Styling Guidelines
* All examples use Tailwind utility classes.
* The component respects Contember theme classes applied to its parent `Menu`.

## Core Concepts
* Menu items can be nested to create hierarchical navigation.
* `label` renders visible text or a React node.
* `icon` adds a visual indicator before the label.
* `to` accepts a routing target handled by `@contember/react-routing`.
* `expandedByDefault` shows child items without user interaction.
* `role` conditionally renders the item based on identity roles.

## Quick Start
```tsx
import { Menu, MenuItem } from '~/lib/ui/menu'

export const Navigation = () => (
	<Menu>
		<MenuItem label="Dashboard" to="dashboard" />
	</Menu>
)
```

## Examples

### Hierarchical Menu with Icons
```tsx
import { Menu, MenuItem } from '~/lib/ui/menu'
import { HomeIcon, UserIcon, LockKeyholeIcon } from 'lucide-react'

export const TenantMenu = () => (
	<Menu>
		<MenuItem icon={<HomeIcon size={16} />} label="Home" to="index" />
		<MenuItem icon={<UserIcon size={16} />} expandedByDefault label="Tenant">
			<MenuItem icon={<LockKeyholeIcon />} label="Security" to="tenant/security" />
			<MenuItem label="Members" to="tenant/members" />
		</MenuItem>
	</Menu>
)
```
Use `expandedByDefault` to reveal nested links on page load.

### Role-Based Item
```tsx
<MenuItem label="Admin" to="admin" role={{ admin: true }} />
```
The item renders only for users satisfying the `RoleCondition`.

## Best Practices & Gotchas
* Keep labels short to prevent overflow in narrow sidebars.
* Pass consistent `size` maps to all items in one menu.
* Avoid deep nesting; two levels are usually enough for usability.
* Wrap icons in `<span className="shrink-0">` if their size differs from text.
* Do not mix `to` and `children` on the same item—use one or the other.
