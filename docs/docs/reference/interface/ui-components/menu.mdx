# Menu

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* The component follows Contember theme tokens where relevant. No additional theme classes are required.

## Core Concepts
* Menus group navigation links into vertical trees.
* Each `MenuItem` can act as a link or expandable node.
* You can nest `MenuItem` components to any depth.
* Icons are optional React nodes placed before the label.
* Role-based wrappers (`HasRole`, `HasCompanyRole`) can wrap `MenuItem` or whole `Menu`.

## Quick Start
```tsx
import { Menu, MenuItem } from '~/lib/ui/menu'

export const SimpleMenu = () => (
	<div className="w-64 border-r border-gray-200">
		<Menu>
			<MenuItem label="Home" to="home" />
			<MenuItem label="Profile" to="profile" />
		</Menu>
	</div>
)
```

## Detailed Guide
```tsx
import { Menu, MenuItem } from '~/lib/ui/menu'
import { HomeIcon, UserIcon, SettingsIcon } from 'lucide-react'

export const NestedMenu = () => (
	<aside className="w-72 bg-white">
		<Menu>
			<MenuItem icon={<HomeIcon size={16} />} label="Dashboard" to="dashboard" />

			<MenuItem icon={<UserIcon size={16} />} label="Account" expandedByDefault>
				<MenuItem label="Profile" to="account/profile" />
				<MenuItem label="Security" to="account/security" />
			</MenuItem>

			<MenuItem icon={<SettingsIcon size={16} />} label="Settings" to="settings" />
		</Menu>
	</aside>
)
```

## Sub-Components / Building Blocks
### MenuItem
Reusable item that represents a link or expandable branch inside a `Menu`. See examples above for typical props like `label`, `to`, `icon`, and `expandedByDefault`.

## Best Practices & Gotchas
* Wrap the menu in a fixed-width container to avoid layout shift.
* Keep nesting depth shallow; deep trees confuse users.
* Use meaningful icons; avoid decorative clutter.
* Combine `Menu` with role guards (`HasRole`, `HasCompanyRole`) for secure navigation.
* Use consistent `size` values for sibling `MenuItem` elements.
* Do not mix interactive and disabled states in one group without clear styling.
