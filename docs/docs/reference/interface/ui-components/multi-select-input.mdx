# MultiSelectInput Component Group

## Overview
MultiSelectInput lets you pick multiple related entities from a dropdown list.  
You can filter, sort, and even create new entities without leaving the input.

## Core Concepts
- Multi-value selection bound to a relation field.
- Filtered option list supplied by Contember query objects.
- Initial ascending or descending sorting.
- Inline entity creation through an embedded form.

## Installation
```bash
npm install @contember/admin
```

```tsx
import { MultiSelectInput, Field } from '@contember/admin'
```

## Quick Start
```tsx
import { MultiSelectInput, Field } from '@contember/admin'

<MultiSelectInput
  field="tags"
  placeholder="Select tags"
  options={[
    { field: 'id', operator: 'eq', value: '1', label: 'Tag 1' },
    { field: 'id', operator: 'eq', value: '2', label: 'Tag 2' },
  ]}
  createNewForm={<div className="p-4">Form to create a new tag</div>}
  initialSorting="ASC"
  className="w-full"
/>
```

## Detailed Guide
1. Pass `field` with the relation name, e.g. `"tags"`.  
2. Build `options` with filter objects containing a `label`.  
3. Add a `<Field>` child when you need custom rendering inside each option.  
4. Control list order with `initialSorting="ASC"` or `"DESC"`.  
5. Supply `createNewForm` to open an inline form at the bottom of the menu.  

```tsx
<MultiSelectInput
  field="tags"
  placeholder="Select tags"
  options={[
    { field: 'id', operator: 'eq', value: '1', label: 'Tag 1' },
    { field: 'id', operator: 'eq', value: '2', label: 'Tag 2' },
  ]}
  createNewForm={<div>New tag form</div>}
  initialSorting="DESC"
  className="w-full"
>
  <Field field="name" />
</MultiSelectInput>
```

## Best Practices & Gotchas
- Keep the `options` array short to reduce query cost.
- Provide a `<Field>` child to avoid showing raw IDs.
- Use Tailwind `truncate` on option labels for long names.
- Embed `createNewForm` sparingly to keep focus on selection.
