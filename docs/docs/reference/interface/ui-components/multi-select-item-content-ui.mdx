## Overview
`MultiSelectItemContentUI` displays the visual content of a selected value inside a `MultiSelect` field.  
You wrap arbitrary JSX (usually a text label) with it to keep spacing and consistent styling among selected chips.

## Styling Guidelines
All examples use Tailwind CSS.  
The component itself adds no extra Contember theme classes; feel free to extend it with custom classes.

## Core Concepts
- Every selected value renders inside `MultiSelectItemUI`, which nests `MultiSelectItemContentUI`.
- You control what appears in the chip by passing children.
- Pair it with `MultiSelectItemRemoveButtonUI` to offer a delete action.

## Quick Start
```tsx
import {
  MultiSelect,
  MultiSelectItemUI,
  MultiSelectItemContentUI,
  MultiSelectItemRemoveButtonUI,
} from '@contember/react-select' // adjust path

<MultiSelect field="tags">
  <MultiSelectItemUI className="flex items-center gap-1 bg-gray-200 px-2 py-1 rounded">
    <MultiSelectItemContentUI>Tag label</MultiSelectItemContentUI>
    <MultiSelectItemRemoveButtonUI />
  </MultiSelectItemUI>
</MultiSelect>
```

## Best Practices & Gotchas
- Keep the chip width minimal. Long texts break layouts.
- Stop event propagation inside custom buttons to avoid opening the dropdown.
- Style all items consistently for better UX.
