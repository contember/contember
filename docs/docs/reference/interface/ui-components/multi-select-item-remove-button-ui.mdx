# MultiSelectItemRemoveButtonUI

## Overview
MultiSelectItemRemoveButtonUI renders a small button inside each selected item. You let users remove a chosen value without reopening the dropdown. The component usually sits inside `SelectItemTrigger`, inheriting focus and keyboard handling.

## Styling Guidelines
* All examples use Tailwind CSS.  
* The button has no required Contember theme classes; style it as you prefer.

## Core Concepts
* A multi-select input can hold many chosen values.  
* Each value is rendered by `MultiSelectItemUI`.  
* `MultiSelectItemRemoveButtonUI` lives inside the item and fires a remove command.  
* Call `e.stopPropagation()` to stop unwanted dropdown toggling.  
* Wrap the button with `SelectItemTrigger` for accessibility.

## Quick Start
```tsx
<SelectItemTrigger>
	<MultiSelectItemRemoveButtonUI
		onClick={e => e.stopPropagation()}
		className="text-red-500 hover:text-red-700"
	/>
</SelectItemTrigger>
```
The snippet removes the clicked value while keeping the popover closed.

## Best Practices & Gotchas
* Always call `e.stopPropagation()` in `onClick`.  
* Add a descriptive `aria-label`, for example `"Remove item"`.  
* Keep the button small (`w-4 h-4`) to reduce visual clutter.  
* Wrap the button in `SelectItemTrigger` to inherit focus styles.  
* Test keyboard removal flows to ensure accessibility.
