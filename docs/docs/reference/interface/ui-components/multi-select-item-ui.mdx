# MultiSelectItemUI

## Overview
MultiSelectItemUI wraps one selected value inside a MultiSelect input.  
It groups the item content and any related triggers (remove, drag-handle) so you can style or reorder the selection consistently.

## Styling Guidelines
* All snippets below use Tailwind CSS.  
* No mandatory Contember theme classes exist for this component. Adjust Tailwind classes to match your design system.

## Core Concepts
* Wrap each selected value with `<MultiSelectItemUI>`.
* Nest content in `<MultiSelectItemContentUI>` to display the label.
* Add `<MultiSelectItemRemoveButtonUI>` to let users unselect a value.
* Optionally insert sortable or drag overlay variants for drag-and-drop reordering.

## Quick Start
```tsx
import {
  MultiSelectItemUI,
  MultiSelectItemContentUI,
  MultiSelectItemRemoveButtonUI,
} from '@contember/ui'

<MultiSelectItemUI className="flex items-center gap-1 rounded bg-gray-100 px-2 py-1">
  <MultiSelectItemContentUI className="text-sm">Company A</MultiSelectItemContentUI>
  <MultiSelectItemRemoveButtonUI
    onClick={e => e.stopPropagation()}
    className="text-gray-500 hover:text-red-500"
  />
</MultiSelectItemUI>
```

## Sub-Components / Building Blocks

### MultiSelectItemContentUI
Wraps the human-readable label of the selected entity.

### MultiSelectItemRemoveButtonUI
Clickable icon/button that removes the item from the selection.  
Call `stopPropagation()` in its `onClick` to avoid toggling the parent popover.

### MultiSelectItemDragOverlayUI
Overlay rendered while the item is dragged. Combine with `dnd-kit` or Contember repeater helpers.

### MultiSelectSortableItemContentUI
Drop-in replacement for `MultiSelectItemContentUI` when items are sortable.

## Examples

### 1. MultiSelect inside a Popover
```tsx
import {
  MultiSelectItemUI,
  MultiSelectItemContentUI,
  MultiSelectItemRemoveButtonUI,
  SelectInputUI,
  SelectEachValue,
} from '@contember/ui'

<SelectInputUI className="w-full">
  <SelectEachValue>
    <MultiSelectItemUI className="flex gap-1 items-center bg-primary-50 rounded px-2 py-0.5">
      <MultiSelectItemContentUI>Tag name</MultiSelectItemContentUI>
      <MultiSelectItemRemoveButtonUI
        onClick={e => e.stopPropagation()}
        className="w-4 h-4 text-primary-600 hover:text-red-600"
      />
    </MultiSelectItemUI>
  </SelectEachValue>
</SelectInputUI>
```
The item component sits inside `SelectEachValue` so it renders once for every selected record.

### 2. Sortable MultiSelect
```tsx
import {
  MultiSelectItemUI,
  MultiSelectSortableItemContentUI,
  MultiSelectItemRemoveButtonUI,
  RepeaterSortableItemActivator,
} from '@contember/ui'

<MultiSelectItemUI className="flex items-center gap-2">
  <RepeaterSortableItemActivator>
    <MultiSelectSortableItemContentUI>Lesson module</MultiSelectSortableItemContentUI>
  </RepeaterSortableItemActivator>
  <MultiSelectItemRemoveButtonUI onClick={e => e.stopPropagation()} />
</MultiSelectItemUI>
```
`RepeaterSortableItemActivator` provides the drag handle; `MultiSelectSortableItemContentUI` updates the cursor and opacity while dragging.

## Best Practices & Gotchas
* Always call `e.stopPropagation()` in the remove buttonâ€™s `onClick` to prevent opening/closing the select popover.
* Keep the entire item clickable only if you want to toggle the option; wrap internal buttons to isolate events.
* Apply padding and rounded corners directly on `MultiSelectItemUI` for predictable hit areas.
* Use the sortable variants only when the parent repeater or list supports drag-and-drop.
