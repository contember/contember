# NavigationGuardDialog

## Overview
`NavigationGuardDialog` prompts users with a confirmation dialog when they attempt to leave a page that has unsaved changes.
It integrates with `useBlockNavigationOnDirtyState` to prevent accidental data loss.

## Styling Guidelines
* All examples use Tailwind CSS.
* No special Contember theme classes are required.

## Core Concepts
* Detect a “dirty” state with `useBlockNavigationOnDirtyState`.
* Block navigation and show a confirmation dialog.
* Provide save, discard, or cancel actions.

## Quick Start
```tsx
import { NavigationGuardDialog } from '@contember/react-components'

export const Page = () => (
	<>
		{/* …your page content… */}
		<NavigationGuardDialog />
	</>
)
```

## Sub-Components / Building Blocks
### useBlockNavigationOnDirtyState
Hook that marks the page as “dirty” and blocks navigation until changes are saved or discarded.

## Examples
```tsx
// Basic usage — mount once at page level
<NavigationGuardDialog />
```
The component renders a modal dialog when navigation is blocked.

```tsx
import { useState } from 'react'
import { useBlockNavigationOnDirtyState, NavigationGuardDialog } from '@contember/react-components'

export const ArticleEditor = () => {
	const [content, setContent] = useState('')
	const isDirty = content.length > 0

	useBlockNavigationOnDirtyState(isDirty)

	return (
		<div className="p-6 flex flex-col gap-4">
			<textarea
				className="border rounded p-2 w-full"
				value={content}
				onChange={e => setContent(e.target.value)}
				placeholder="Start typing…"
			/>
			<NavigationGuardDialog />
		</div>
	)
}
```
The textarea sets the page to a “dirty” state.
Attempting to navigate away shows `NavigationGuardDialog`, letting users save, discard, or cancel.

## Best Practices & Gotchas
* Mount `NavigationGuardDialog` only once, near the root of your page.
* Keep dialog text short and action-oriented.
* Always pair the dialog with `useBlockNavigationOnDirtyState`.
* Don’t rely solely on the browser’s `beforeunload`; some users disable it.
