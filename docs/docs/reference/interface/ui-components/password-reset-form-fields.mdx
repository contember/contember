# PasswordResetFormFields

## Overview
`PasswordResetFormFields` renders the input fields required to reset a user password inside `PasswordResetForm`.
The component decides whether to show the token field based on the `hasToken` prop.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No additional Contember theme classes are required.

## Core Concepts
* You embed `PasswordResetFormFields` inside a `<form>` element that is wrapped by `PasswordResetForm`.
* The `hasToken` prop controls whether the **reset-token** input is displayed.
* Place the component within any card, modal, or plain container—styling is entirely up to you.

## Quick Start
```tsx
import { PasswordResetForm, PasswordResetFormFields } from '~/lib/tenant'

export default () => (
  <PasswordResetForm token="URL_TOKEN" onSuccess={() => console.log('reset')}>
    <form className="flex flex-col gap-4">
      <PasswordResetFormFields hasToken />
    </form>
  </PasswordResetForm>
)
```

## Props Reference
| Name     | Type    | Required | Default | Description                            |
|----------|---------|----------|---------|----------------------------------------|
| hasToken | boolean | No       | false   | When `true`, the component renders a hidden token input. |
| *Inherited React props* | — | — | — | Standard DOM props like `className`, `style`, and `id`. |

## Examples

### Token supplied in URL
```tsx
const token = useCurrentRequest()?.parameters.token as string | undefined

<PasswordResetForm
  token={token}
  onSuccess={() => showToast(<ToastContent>Password reset</ToastContent>, { type: 'success' })}
>
  <form className="space-y-4">
    <PasswordResetFormFields hasToken={!!token} />
  </form>
</PasswordResetForm>
```
This pattern extracts the token from the current request and passes it to both `PasswordResetForm` and `PasswordResetFormFields`.

### Reset without pre-filled token
```tsx
<PasswordResetForm onSuccess={() => navigate('index')}>
  <form className="grid gap-3">
    <PasswordResetFormFields hasToken={false} />
  </form>
</PasswordResetForm>
```
Use this variant when your workflow expects users to paste the reset token manually.

## Best Practices & Gotchas
* Always wrap the fields in a `<form>` element; the component itself is not a form tag.
* Pass the same token to `PasswordResetForm` and `PasswordResetFormFields` to avoid validation errors.
* Keep your success handler short—trigger a toast and navigate away quickly.
