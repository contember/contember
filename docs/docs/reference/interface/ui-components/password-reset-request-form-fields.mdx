# PasswordResetRequestFormFields

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No additional Contember theme classes are required.

## Core Concepts
* Password reset flow starts with a form that collects the user’s e-mail.
* `PasswordResetRequestFormFields` renders the default field set (typically just the e-mail input).
* You embed the fields inside a parent `<PasswordResetRequestForm>` component that handles submission.
* The parent form fires `onSuccess` when the reset request is accepted.

## Quick Start
```tsx
import { PasswordResetRequestForm, PasswordResetRequestFormFields } from '@app/lib/tenant'

export default () => (
  <PasswordResetRequestForm onSuccess={() => console.log('Reset link sent')}>
    <form className="space-y-4">
      <PasswordResetRequestFormFields />
    </form>
  </PasswordResetRequestForm>
)
```

## Examples

### Placing the form inside a Card component
```tsx
import {
  PasswordResetRequestForm,
  PasswordResetRequestFormFields,
} from '@app/lib/tenant'
import { Card, CardHeader, CardTitle, CardContent } from '@/components/ui/card'

export default () => (
  <Card className="w-96">
    <CardHeader>
      <CardTitle className="text-2xl">Reset your password</CardTitle>
    </CardHeader>

    <CardContent>
      <PasswordResetRequestForm onSuccess={() => redirect('resetRequestSuccess')}>
        <form>
          <PasswordResetRequestFormFields />
        </form>
      </PasswordResetRequestForm>
    </CardContent>
  </Card>
)
```
*The Card provides visual framing while `PasswordResetRequestFormFields` supplies the input.*

### Adding a back-to-login link
```tsx
<Link to="index">
  <AnchorButton variant="link" className="ml-auto">
    Back to login
  </AnchorButton>

```
*Place the link near the form footer so users can easily return to the login page.*

## Best Practices & Gotchas
* Wrap the fields in a `<form>` element—even inside a parent form component.
* Keep the form width narrow (`w-96`) for better readability.
* Provide clear success feedback via the `onSuccess` callback.
* Do not add extra props to `PasswordResetRequestFormFields`; it accepts none.

## Further Reading
* `PasswordResetRequestForm` – parent that handles submission.
* `PasswordResetFormFields` – renders fields used after the user receives the reset token.
* `LoginFormFields` – renders fields for the standard login form.
