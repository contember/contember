## Styling Guidelines
* All code examples use Tailwind CSS utility classes.
* The component adds no implicit Contember theme classes, so you style it just like any other form control.

## Core Concepts
* Passwordless authentication requires either an e-mailed token or an OTP code.
* `PasswordlessSignInFormFields` renders only the inputs needed for the selected validation type.
* The component must live inside `<PasswordlessSignInForm>` so that form context and validation work.

## Quick Start
```tsx
import { PasswordlessSignInForm, PasswordlessSignInFormFields } from '~/lib/tenant/forms/passwordless-signin'

export default () => (
	<PasswordlessSignInForm requestId="abc" validationType="otp" onSuccess={() => {}}>
		<form className="flex flex-col gap-4">
			<PasswordlessSignInFormFields type="otp" />
		</form>
	</PasswordlessSignInForm>
)
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| type | `'otp' \| 'token'` | Yes | — | Determines whether the user enters an OTP code (`'otp'`) or a token from a magic link (`'token'`). |

## Examples

### Token-based sign-in
```tsx
<PasswordlessSignInForm
	requestId={requestId}
	validationType="token"
	onSuccess={() => redirect('index')}
>
	<form className="space-y-4">
		<PasswordlessSignInFormFields type="token" />
	</form>
</PasswordlessSignInForm>
```
This variant renders a single text input where the user pastes the token they received via e-mail.

### OTP sign-in with conditional UI
```tsx
{showOtp && (
	<PasswordlessSignInForm
		requestId={requestId}
		validationType="otp"
		onSuccess={() => redirect('index')}
	>
		<form className="space-y-4">
			<PasswordlessSignInFormFields type="otp" />
		</form>
	</PasswordlessSignInForm>
)}
```
Toggle `showOtp` to prompt the user only when they choose to enter the OTP manually.

## Best Practices & Gotchas
* Always wrap the fields in `<form>` to enable native browser submission shortcuts.
* Match the `type` prop with the surrounding `validationType` on `<PasswordlessSignInForm>`.
* Add `className` directly on `<form>` or a wrapper—`PasswordlessSignInFormFields` itself doesn’t expose styling hooks.
* Handle success in `onSuccess` of the parent form instead of inside the fields component.
