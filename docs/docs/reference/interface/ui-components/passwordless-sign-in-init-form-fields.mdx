# `PasswordlessSignInInitFormFields`

## Overview
`PasswordlessSignInInitFormFields` renders the input fields required to start a password-less sign-in flow.  
You usually place it inside `PasswordlessSignInInitForm` and wrap both with a standard `<form>` element.

## Styling Guidelines
* All examples below use Tailwind classes.  
* The component follows Contember’s default form theme, so your global `@contember/ui` styles (if installed) will apply automatically.

## Core Concepts
* Password-less flow needs an initiation step (collect email or phone).  
* `PasswordlessSignInInitFormFields` handles field rendering, validation, and error messages.  
* You supply the surrounding `<form>` tag and any submit button.  
* Styling relies on Tailwind; override with utility classes or custom CSS.

## Quick Start
```tsx
import { PasswordlessSignInInitForm, PasswordlessSignInInitFormFields } from '~/lib/tenant/forms/passwordless-signin-init'

export default () => (
	<PasswordlessSignInInitForm onSuccess={() => {/* redirect user */}}>
		<form className="grid gap-4">
			<PasswordlessSignInInitFormFields />
			<button type="submit" className="btn-primary w-full">Send Magic Link</button>
		</form>
	</PasswordlessSignInInitForm>
)
```

## Examples

### Add a descriptive headline
```tsx
<PasswordlessSignInInitForm onSuccess={handleSuccess}>
	<form className="space-y-6">
		<h2 className="text-xl font-semibold text-center">Sign in without a password</h2>
		<PasswordlessSignInInitFormFields />
		<button type="submit" className="btn-primary w-full">Send Link</button>
	</form>
</PasswordlessSignInInitForm>
```
This pattern adds a headline above the fields while keeping the form self-contained.

### Combine with social logins
```tsx
<IDP>
	<Card className="w-96">
		<CardContent className="space-y-4">
			<PasswordlessSignInInitForm onSuccess={handleSuccess}>
				<form className="grid gap-4">
					<PasswordlessSignInInitFormFields />
				</form>
			</PasswordlessSignInInitForm>

			<IDPInitTrigger identityProvider="google">
				<Button variant="outline" className="w-full">Sign in with Google</Button>
			</IDPInitTrigger>
		</CardContent>
	</Card>
</IDP>
```
You can freely mix password-less fields with OAuth identity providers for a unified login screen.

## Best Practices & Gotchas
* Wrap the component in a single `<form>` tag; otherwise validation may break.  
* Handle `onSuccess` on the parent `PasswordlessSignInInitForm` to control redirects.  
* Keep the submit button outside `PasswordlessSignInInitFormFields`; the component renders only inputs.  
* Use Tailwind’s `space-y-*` or `grid gap-*` utilities to maintain consistent spacing.  
* Do not nest multiple `PasswordlessSignInInitFormFields` instances in one form.
