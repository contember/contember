# PersistButton

## Overview
`PersistButton` renders a button that saves unsaved data in the current `Binding` context.  
The button uses `usePersistSuccessHandler` to handle post-save feedback.

## Styling Guidelines
* All examples use Tailwind CSS.  
* `PersistButton` itself adds no mandatory theme classes.

## Core Concepts
* The component must live inside a `Binding` provider.  
* It triggers the Contember persistence pipeline.  
* You can replace the default label with the `label` prop.  
* Success feedback flows through `usePersistSuccessHandler`.

## Quick Start
```tsx
import { Binding, PersistButton } from '~/lib/binding'

export default () => (
  <Binding>
    <div className="flex justify-end">
      <PersistButton />
    </div>
  </Binding>
)
```

## Detailed Guide
### Custom label
```tsx
<PersistButton label="Save Article" />
```
The prop overrides the text that defaults to `dictionary.persist.persistButton`.

## Props Reference
| Name | Type | Required | Default | Description |
| ---- | ---- | -------- | ------- | ----------- |
| label | React.ReactNode | No | dictionary.persist.persistButton | Custom button text |

_Inherited props:_ None

## Sub-Components / Building Blocks
### usePersistSuccessHandler
Hook that lets you run code after a successful save.

```tsx
const { usePersistSuccessHandler } from '@contember/react'

const MyComponent = () => {
  usePersistSuccessHandler(() => toast.success('Saved!'))
  return <PersistButton />
}
```

## Examples
### Add save action to a create form
```tsx
import { Binding, EntitySubTree, PersistButton } from '~/lib/binding'

<Binding>
  <EntitySubTree entity="Post" isCreating>
    <AutoFields />
    <div className="flex justify-end">
      <PersistButton />
    </div>
  </EntitySubTree>
</Binding>
```
The button appears bottom-right and persists the new `Post`.

### Edit page with custom label
```tsx
<Binding>
  <EntitySubTree entity="Article(id = $id)">
    <AutoFields />
    <PersistButton label="Update Article" />
  </EntitySubTree>
</Binding>
```
Use a more descriptive caption to match the page action.

## Best Practices & Gotchas
* Always wrap the button in `Binding`; otherwise nothing happens.  
* Place the button in `Slots.Actions` to follow Contember layouts.  
* Provide visual feedback with `usePersistSuccessHandler`.  
* Avoid multiple `PersistButton`s in one subtree; it can confuse users.
