# PersonList

## Overview
PersonList renders a paginated table of project members.  
You use it to view, filter, and refresh user memberships inside a Contember project.

## Styling Guidelines
* All code samples use Tailwind classes.  
* PersonList itself adds no visual styles; wrap it in your theme’s `Card`, `div`, or layout utilities.

## Core Concepts
* MemberListController lets you refresh the list programmatically.  
* RolesConfig supplies human-friendly labels for each role.  
* PersonList reads project context from `ProjectSlugProvider` higher in the tree.

## Quick Start
```tsx
import { PersonList } from '~/lib/tenant'
import { useRef } from 'react'

export default () => {
  const controller = useRef<MemberListController>()

  return (
    <div className="p-6">
      <PersonList controller={controller} />
    </div>
  )
}
```

## Examples

### Refresh after Invite
```tsx
() => {
  const projectSlug = useProjectSlug()!
  const showToast = useShowToast()
  const controller = useRef<MemberListController>()

  return (
    <div className="flex flex-col gap-4">
      <InviteForm
        projectSlug={projectSlug}
        onSuccess={args => {
          showToast(
            <ToastContent>
              Invitation sent to {args.result.person?.email}
            </ToastContent>,
            { type: 'success' }
          )
          controller.current?.refresh()
        }}
      >
        <form className="grid gap-4">
          <InviteFormFields projectSlug={projectSlug} />
        </form>
      </InviteForm>

      <PersonList controller={controller} />
    </div>
  )
}
```
The invite form triggers `controller.current?.refresh()` which reloads the list so the newly invited member appears immediately.

### List with Custom Roles
```tsx
<PersonList controller={controller} roles={rolesConfig} />
```
Pass `roles` to display labels and colors specific to your application’s role scheme.

## Best Practices & Gotchas
* Always wrap PersonList in a container with overflow handling.  
* Reuse one MemberListController per page; avoid new refs on each render.  
* Call `refresh` after any mutation to keep the list in sync.  
* Provide `roles` so users understand each member’s permissions.
