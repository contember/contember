## Overview
`PopoverAnchor` wraps any element and marks it as the reference point for a popover.
It provides no behavior on its own; it only supplies layout context for `PopoverContent`.

## Styling Guidelines
* All examples use Tailwind utility classes.
* No special Contember theme classes are required.

## Core Concepts
* An *anchor* defines where the popover should align.
* The anchor itself does **not** open or close the popover.
* Place any interactive element (button, icon) inside the anchor.

## Quick Start
```tsx
import { PopoverAnchor } from '@contember/react-components'

export const Demo = () => (
  <PopoverAnchor>
    <button className="px-4 py-2 bg-blue-600 text-white rounded">
      Toggle details
    </button>
  </PopoverAnchor>
)
```

## Best Practices & Gotchas
* Wrap exactly the element that should act as the reference point.
* Keep anchors small; large anchors cause unpredictable popover placement.
* Manage show/hide logic outside of `PopoverAnchor`.

## Further Reading
* `PopoverContent` – renders the actual popover.
* `usePopover` hook – handles open/close logic.
