# PropertyRow

## Overview
_None_

## Styling Guidelines
* All examples use Tailwind CSS utility classes.  
* Contember’s default theme adds typography helpers such as `font-normal`. Keep them if you rely on consistent weight across the list.

## Core Concepts
* PropertyRow groups one label with one value inside a PropertyList.  
* Combine it with PropertyLabel and PropertyValue for clear two-column layout.  
* Use composition instead of props to control spacing and order.  
* Nest any React elements inside PropertyValue, including rich text or formatted fields.

## Quick Start
```tsx
import { PropertyRow, PropertyLabel, PropertyValue } from '~/lib/ui/property-list'

export const QuickStart = () => (
  <PropertyRow className="flex gap-4 py-2">
    <PropertyLabel className="w-40 text-sm font-medium">
      Subject
    </PropertyLabel>

    <PropertyValue className="flex-1 font-normal">
      Welcome email
    </PropertyValue>
  </PropertyRow>
)
```

## Detailed Guide
```tsx
import {
  PropertyList,
  PropertyRow,
  PropertyLabel,
  PropertyValue,
} from '~/lib/ui/property-list'

export const EmailDetail = () => (
  <PropertyList className="divide-y divide-gray-200">
    {/* Other PropertyItem rows */}

    <PropertyRow>
      <PropertyLabel>Content</PropertyLabel>
      <PropertyValue className="font-normal">
        <FormattedField field="content" formatter={formatStringToHtmlDoc} />
      </PropertyValue>
    </PropertyRow>

    {/* More PropertyItem rows */}
  </PropertyList>
)
```
* Place PropertyRow among PropertyItem components for mixed layouts.  
* Add `divide-y` to the parent list for subtle separators.  
* Override spacing with Tailwind’s padding utilities on each row.

## Sub-Components / Building Blocks
### PropertyLabel
* Renders the descriptive text.  
* Accepts standard `className` and children.

### PropertyValue
* Holds the actual value.  
* Accepts any React nodes.  
* Use typography classes like `font-normal` for consistency.

## Examples
```tsx
<PropertyList>
  <PropertyItem label={entityLabels.Email.fields.recipients}>
    <FormattedField field="recipients" />
  </PropertyItem>

  <PropertyRow>
    <PropertyLabel>{entityLabels.Email.fields.content}</PropertyLabel>
    <PropertyValue className="font-normal">
      <FormattedField field="content" formatter={formatStringToHtmlDoc} />
    </PropertyValue>
  </PropertyRow>

  <PropertyItem label={entityLabels.Email.fields.sentAt}>
    <FormattedField field="sentAt" />
  </PropertyItem>
</PropertyList>
```
This pattern mixes PropertyItem (label prop) and PropertyRow (explicit label component) in one list. Use it when the value needs custom styling or multiple elements.

## Best Practices & Gotchas
* Keep label width consistent with the Tailwind `w-*` utilities.  
* Avoid wrapping PropertyRow inside another flex container; it already exposes a flex layout.  
* Do not pass props that belong to PropertyItem; PropertyRow ignores them.  
* Always pair PropertyLabel and PropertyValue to preserve accessibility.
