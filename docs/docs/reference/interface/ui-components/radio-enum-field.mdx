## Overview
RadioEnumField renders enum values as a group of radio buttons inside an Entity context.
You can let the component discover enum values automatically or pass your own `options` array.

## Styling Guidelines
All examples use Tailwind utility classes.
RadioEnumField works with any Contember theme; no special theme classes are required.

## Core Concepts
- Entity context is mandatory (`<EntitySubTree />` or `<EntityListSubTree />`).
- Each option needs a `value` and a `label`.
- The `orientation` prop controls horizontal or vertical layout.
- Omit the `options` prop to auto-generate choices from the schema enum.
- You can forward standard `<input>` attributes through `inputProps`.

## Quick Start
```tsx
<EntitySubTree entity="Article" setOnCreate="(status = draft)">
	<RadioEnumField
		field="status"
		label="Status"
		className="space-y-2"
		options={[
			{ value: 'draft', label: 'Draft' },
			{ value: 'published', label: 'Published' },
		]}
	/>
</EntitySubTree>
```

## Detailed Guide
### Auto-detect enum values
```tsx
<RadioEnumField field="category" label="Category" />
```
The component queries the Contember schema and renders all enum labels automatically.

### Switching orientation
```tsx
<RadioEnumField
	field="visibility"
	label="Visibility"
	orientation="horizontal"
	className="flex gap-4"
/>
```
Use `orientation="horizontal"` for a compact inline layout. The default orientation is vertical.

## Examples
### Conditional rendering with `If`
```tsx
<RadioEnumField field="loginBackgroundType" orientation="horizontal" />
<If condition="[loginBackgroundType = image]">
	<ImageField baseField="loginBackgroundImage" label=" " />
</If>
<If condition="[loginBackgroundType = color]">
	<InputField field="loginBackgroundColor" inputProps={{ type: 'color' }} label=" " />
</If>
```
This pattern shows how you can branch the form based on the selected radio value.

### Mixing data types in options
```tsx
<RadioEnumField
	field="enumValue"
	label="Mixed types"
	options={[
		{ value: 'a', label: 'Option A' },
		{ value: 1, label: 'One' },
		{ value: true, label: 'True' },
		{ value: null, label: 'None' },
	]}
/>
```
RadioEnumField handles strings, numbers, booleans, and `null` without extra code.

## Best Practices & Gotchas
- Always nest the component in an Entity context; otherwise, it cannot bind to data.
- Provide `inputProps={{ disabled: true }}` instead of wrapping the whole field in `fieldset disabled`.
- Keep option values immutable to avoid React key warnings.
- Use the schema enum labels to localize automatically; avoid hard-coding strings when possible.
- Combine `required` with server-side validation rules for full safety.
