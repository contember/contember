# RadioInput

## Overview
RadioInput renders an HTML `<input type="radio">` wrapped with Contember-specific styles.
Use it when users must choose exactly one option from a set.

## Styling Guidelines
* All snippets below rely on Tailwind CSS.
* The component ships with Contember theme classes, so you rarely add more than layout utilities.

## Core Concepts
* RadioInput is a controlled component – you pass `checked` and `onChange`.
* It forwards most props to the underlying `<input>` element.
* Pair it with `<label>` or Contember `Label` for accessible text.
* Group radios by sharing the same `name`.

## Quick Start
```tsx
import { RadioInput } from '~/lib/ui/input'

export const ThemePicker = () => {
	const [theme, setTheme] = React.useState<'light' | 'dark'>('light')

	return (
		<div className="flex gap-4">
			<label className="inline-flex items-center gap-2">
				<RadioInput
					name="theme"
					checked={theme === 'light'}
					onChange={() => setTheme('light')}
				/>
				Light
			</label>

			<label className="inline-flex items-center gap-2">
				<RadioInput
					name="theme"
					checked={theme === 'dark'}
					onChange={() => setTheme('dark')}
				/>
				Dark
			</label>
		</div>
	)
}
```
## Props Reference
| Name | Type | Required | Default | Description |
|---|---|---|---|---|
| name | string | No | — | Radio group identifier. Radios that share the same `name` behave as one group. |
| checked | boolean | No | — | Controls the checked state. |
| onChange | `(e: React.ChangeEvent<HTMLInputElement>) => void` | No | — | Called when the user selects the radio. |
| disabled | boolean | No | `false` | Disables the radio and makes it unfocusable. |
| type | `"radio"` | No | `"radio"` | Exposed only for completeness; rarely overridden. |
| …rest | HTMLInputElement props | No | — | All other standard `<input>` attributes. |

## Examples

### Inside a Popover settings panel
```tsx
() => {
	const [settings, setSettings] = useSettings()

	return (
		<Popover>
			<PopoverTrigger asChild>
				<Button variant="ghost" size="xs">
					<SettingsIcon size={16} />

							name="area"
							checked={settings.area === 'sqm'}
							onChange={() => setSettings({ area: 'sqm' })}

						m²
					</Label>

					<Label className="inline-flex gap-2">
						<RadioInput
							name="area"
							checked={settings.area === 'sqft'}
							onChange={() => setSettings({ area: 'sqft' })}
						/>
						ft²
					</Label>
```

Use unique `name` per setting group so each popover works independently.

## Best Practices & Gotchas
* Always supply a `name`; otherwise, the browser treats each radio as a standalone control.
* Keep RadioInput controlled (`checked` + `onChange`) to avoid React warnings.
* Wrap the input and label in the same clickable area for better UX.
* Avoid setting `type` – the default `"radio"` is already correct.
