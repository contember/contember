## Overview
`RepeaterRemoveItemButton` renders a button that removes the current repeater item.  
It wraps `RepeaterRemoveItemTrigger`, so you can replace the default trash icon (which turns red on hover) with any child element.

## Styling Guidelines
* All examples use Tailwind CSS.  
* No special Contember theme classes are required; inherit your project’s design tokens.

## Core Concepts
* Contember repeaters manage collections of entities.  
* Triggers encapsulate data mutations (add, remove, reorder).  
* `RepeaterRemoveItemButton` is a trigger shortcut wrapped in a button.  
* You can override the default icon by passing `children`.

## Quick Start
```tsx
import { Binding } from '@contember/react-binding'
import { DefaultRepeater, RepeaterItemActions, RepeaterRemoveItemButton } from '~/lib/repeater'

export const Page = () => (
	<Binding>
		<DefaultRepeater entities="Item" orderBy="order" title="Items">
			<RepeaterItemActions className="flex gap-2">
				<RepeaterRemoveItemButton />
			</RepeaterItemActions>
		</DefaultRepeater>
	</Binding>
)
```

## Detailed Guide
1. Place `RepeaterRemoveItemButton` inside `RepeaterItemActions` or any element rendered for each repeater row.  
2. Pass custom content to change the appearance:

```tsx
<RepeaterRemoveItemButton>
	<span className="text-red-600 hover:text-red-800">Delete</span>
</RepeaterRemoveItemButton>
```

3. Combine with dropdown menus:

```tsx
<RepeaterItemActions className="flex gap-2">
	{repeaterDropdown}
	<RepeaterRemoveItemButton />
</RepeaterItemActions>
```

## Examples
### Sortable repeater with relations
```tsx
<DefaultRepeater entities="RepeaterItem" sortableBy="order" title="Foo items" addButtonPosition="around">
	<InputField field="title" />

	<SelectField field="relation">
		<Field field="id" /> /
		<Field field="name" />
	</SelectField>

	<RepeaterItemActions className="flex gap-2">
		{repeaterDropdown}
		<RepeaterRemoveItemButton />
	</RepeaterItemActions>
</DefaultRepeater>
```
The button removes the current row while the repeater remains sortable.

Use the same button pattern even when the repeater points to a relation.

### Non‑sortable repeater
```tsx
<DefaultRepeater entities="RepeaterItem" orderBy="order" title="Foo items">
	<Field field="title" />

	<RepeaterItemActions className="flex gap-2">
		{repeaterDropdown}

```
Even without sorting, the button cleanly removes the selected item.

## Best Practices & Gotchas
* Keep destructive actions visually distinct (e.g., red icon).  
* Wrap the button in `RepeaterItemActions` to maintain layout.  
* Do not attach extra `onClick`; use `children` for custom UI only.  
* Removing an item does not persist until you press `PersistButton`.
