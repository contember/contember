## Overview
`richTextFieldPlugins` is a predefined array of Slate plugins that adds essential inline formatting to Contember’s `RichTextEditor`.  
You use it when you need links, emphasis, code blocks, highlights, and other common text styles without hand‑crafting every Slate plugin yourself.

## Styling Guidelines
* All code examples use Tailwind CSS.  
* No Contember‑specific theme classes are required for the plugins to work.

## Core Concepts
* Slate plugins extend editor behavior.
* `richTextFieldPlugins` is an array; you pass it to `RichTextEditor` via the `plugins` prop.
* The plugins cover links, bold/italic/underline, strikethrough, code blocks, highlights, and manual line breaks.
* Order matters: Contember already arranges the plugins in a safe sequence.

## Quick Start
```tsx
import { RichTextEditor } from '@contember/react';
import { richTextFieldPlugins } from '@contember/react'; // adjust import path if needed

export default function ArticleEditor() {
	return (
		<div className="max-w-xl mx-auto">
			<RichTextEditor
				plugins={richTextFieldPlugins}
				className="border rounded-md p-4 focus:outline-none"
			/>
		</div>
	);
}
```

## Examples

### Blog post body with save button
```tsx
import { useCallback } from 'react';
import { RichTextEditor } from '@contember/react';
import { richTextFieldPlugins } from '@contember/react';

export default function BlogPostBody({ onSave }: { onSave: (value: unknown) => void }) {
	const handleSave = useCallback((value: unknown) => {
		onSave(value);
	}, [onSave]);

	return (
		<div className="flex flex-col gap-4">
			<RichTextEditor
				plugins={richTextFieldPlugins}
				className="border rounded p-4 min-h-[300px]"
				onBlur={handleSave}
			/>
			<button
				onClick={() => handleSave(/* you could read editor state here */)}
				className="self-end px-4 py-2 bg-blue-600 text-white rounded"
			>
				Save
			</button>
		</div>
	);
}
```
`onBlur` triggers automatic saving; the button provides a manual fallback.

## Best Practices & Gotchas
* Always memoize plugin arrays when you create custom ones to avoid re‑mounting the editor.
* Keep the array stable: pass the literal `richTextFieldPlugins` variable instead of recreating it on every render.
* Wrap the editor in a container with a defined width to prevent unexpected line‑wrap behavior.
* Use Tailwind’s focus utilities (`focus:outline-none`) so keyboard users get proper feedback.
