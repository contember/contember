# ScrollArea

_None_

## Styling Guidelines
* All examples use Tailwind CSS classes.
* ScrollArea does not require any Contember theme classes out of the box.

## Core Concepts
* Wrap long content in a scrollable container.
* Control vertical height with Tailwind utility classes (`max-h-*`, `overflow-y-auto`).
* Nest any React elements as children.

## Quick Start
```tsx
import { ScrollArea } from '~/lib/ui/scroll-area'

export const Demo = () => (
	<ScrollArea className="max-h-64 overflow-y-auto">
		<p className="p-4">
			Scrollable content goes here.
		</p>
	</ScrollArea>
)
```

## Examples

```tsx
<ScrollArea className="max-h-80">
	{isLoading && (
		<SelectListItemUI>
			<Loader size="md" />
		</SelectListItemUI>
	)}
	{error && <SelectListItemUI>{error}</SelectListItemUI>}
	{!isLoading && filteredItems?.length === 0 && (
		<SelectListItemUI className="hover:cursor-default hover:bg-transparent">
			{componentLabels.simpleSelect.noData}
		</SelectListItemUI>
	)}
	{filteredItems?.map((item, index) => (
		<SelectListItemUI
			key={getItemValue ? String(getItemValue(item)) : `item-${index}`}
			onClick={() => handleItemClick(item)}
		>
			{getItemLabel(item)}
		</SelectListItemUI>
	))}
</ScrollArea>
```
The example shows a scrollable dropdown list. You limit the height with `max-h-80`, display loaders or error states, and map items to interactive rows.

## Best Practices & Gotchas
* Always set a `max-h-*` or explicit height; otherwise the container may not scroll.
* Combine `overflow-y-auto` with vertical padding for smooth scrolling.
* Do not place fixed-height elements inside if you expect dynamic content growth.
* Pass className directly; ScrollArea forwards it to the root element.
