# SelectDefaultFilter

## Overview
`SelectDefaultFilter` filters data inside a `SelectDataView`.  
The component renders a text input only when a matching default filter exists, letting users refine the list quickly.

## Styling Guidelines
* All examples use Tailwind CSS.
* `SelectDefaultFilter` adds no Contember theme classes by itself.

## Core Concepts
* Filters live inside a `SelectDataView`.
* `SelectDefaultFilter` appears only when the data view provides a relevant filter type.
* The component outputs a plain text input for inline filtering.

## Quick Start
```tsx
import { SelectDataView } from '@contember/react-components'
import { SelectDefaultFilter } from '@app/lib/select'

export const UserSelect = () => (
	<SelectDataView className="flex flex-col gap-4">
		<SelectDefaultFilter />
		{/* …rest of your view… */}
	</SelectDataView>
)
```

## Examples

### Example 1 – Relation Filter in a Learning Platform
```tsx
({ children, toggleExclude, toggleInclude, included, excluded, entityName, initialSorting }: {
	children: ReactNode
	toggleInclude: (entityId: EntityId) => void
	toggleExclude: (entityId: EntityId) => void
	included: string[]
	excluded: string[]
	entityName: string
	initialSorting?: DataViewSortingDirections
}) => {
	return (
		<>
			<DataViewRelationFilterOptions entities={entityName} initialSorting={initialSorting}>
				<SelectListInner filterToolbar={<SelectDefaultFilter />}>
					<DataGridRelationFilterSelectItem toggleInclude={toggleInclude} toggleExclude={toggleExclude} included={included} excluded={excluded}>
						{children}
					</DataGridRelationFilterSelectItem>
				</SelectListInner>
			</DataViewRelationFilterOptions>
		</>
	)
}
```
The filter sits in the `filterToolbar` position of `SelectListInner`.  
Users type text to narrow entities before including or excluding them in the relation filter.

## Best Practices & Gotchas
* Place `SelectDefaultFilter` at the top of the list for better visibility.
* Combine it with relation filters to let users search referenced entities.
* Ensure the target `SelectDataView` defines a compatible default filter; otherwise, the component renders nothing.
* Avoid wrapping it in extra form controls; it manages its own input state.
