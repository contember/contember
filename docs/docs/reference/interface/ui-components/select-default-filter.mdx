# SelectDefaultFilter

## Overview
`SelectDefaultFilter` filters data in a `SelectDataView`.
It checks whether a default filter type exists and, when it does, renders a text input for user-driven filtering.

## Styling Guidelines
* All examples use Tailwind CSS.
* The component adds no Contember theme classes by itself.

## Core Concepts
* Filter widgets live inside a `SelectDataView`.
* `SelectDefaultFilter` renders only when a matching filter type exists.
* The component outputs a plain text input for quick, inline filtering.

## Quick Start
```tsx
import { SelectDataView } from '@contember/react-components'
import { SelectDefaultFilter } from './filter'

export const UserSelect = () => (
	<SelectDataView className="flex flex-col gap-4">
		<SelectDefaultFilter />
		{/* …rest of your view… */}
	</SelectDataView>
)
```

### Example 1 – Basic List Filtering

The filter appears above the list and lets users narrow results with free-text input.

### Example 2 – Multi-Select Picker With Filter
```tsx
({ field, queryField, options, children, placeholder, createNewForm, initialSorting }) => {
	const [open, setOpen] = React.useState(false)
	const id = useFormFieldId()

	return (
		<MultiSelect field={field} options={options}>
			<div className="flex gap-1 items-center">
				<Popover open={open} onOpenChange={setOpen}>
					<SelectInputWrapperUI>
						<PopoverTrigger asChild>
							<SelectInputUI id={id ? `${id}-input` : undefined}>
								<SelectPlaceholder>
									{placeholder ?? <SelectDefaultPlaceholderUI />}
								</SelectPlaceholder>

								<MultiSelectItemsUI>
									<SelectEachValue>
										<MultiSelectItemUI>
											<MultiSelectItemContentUI>{children}</MultiSelectItemContentUI>
											<SelectItemTrigger>
												<MultiSelectItemRemoveButtonUI onClick={e => e.stopPropagation()} />
											</SelectItemTrigger>
										</MultiSelectItemUI>
									</SelectEachValue>
								</MultiSelectItemsUI>

								<SelectInputActionsUI>
									{open ? <ChevronUpIcon className="w-4 h-4" /> : <ChevronDownIcon className="w-4 h-4" />}
								</SelectInputActionsUI>
							</SelectInputUI>
						</PopoverTrigger>
					</SelectInputWrapperUI>

					<SelectPopoverContent>
						<SelectDataView initialSorting={initialSorting} queryField={queryField}>
							<SelectListInner filterToolbar={<SelectDefaultFilter />}>
								<SelectOption>
									<SelectItemTrigger>
										<SelectListItemUI>{children}</SelectListItemUI>
									</SelectItemTrigger>
								</SelectOption>
							</SelectListInner>
						</SelectDataView>
					</SelectPopoverContent>
				</Popover>
				{createNewForm && (
					<CreateEntityDialog trigger={<SelectCreateNewTrigger />}>
						{createNewForm}
					</CreateEntityDialog>
				)}
			</div>
		</MultiSelect>
	)
}
```
You embed the filter inside a popover so users can search large datasets without leaving the form.

## Best Practices & Gotchas
* Place `SelectDefaultFilter` at the top of the view for better UX.
* Combine it with other filters for complex queries.
* Avoid wrapping it in additional forms; it manages its own input state.
* Verify the target `SelectDataView` supports the expected filter type.
* Keep the filter visible; hidden filters reduce discoverability.
