# SelectDefaultFilter

## Overview
`SelectDefaultFilter` filters data in a `SelectDataView`.
It checks whether a default filter type exists and, when it does, renders a text input for user‑driven filtering.

## Styling Guidelines
* All examples use Tailwind CSS.
* The component adds no Contember theme classes by itself.

## Core Concepts
* Filter widgets live inside a `SelectDataView`.
* `SelectDefaultFilter` renders only when a matching filter type exists.
* The component outputs a plain text input for quick, inline filtering.

## Quick Start
```tsx
import { SelectDataView, SelectDefaultFilter } from '@contember/react-components'

export const UserSelect = () => (
	<SelectDataView className="flex flex-col gap-4">
		<SelectDefaultFilter />
		{/* …rest of your view… */}
	</SelectDataView>
)
```

## Best Practices & Gotchas
* Place `SelectDefaultFilter` at the top of the view for better UX.
* Combine it with other filters for complex queries.
* Avoid wrapping it in additional forms; it manages its own input state.
* Verify the target `SelectDataView` supports the expected filter type.
