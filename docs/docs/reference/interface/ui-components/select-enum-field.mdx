# SelectEnumField

## Overview
SelectEnumField lets you pick an enum value inside a Contember form.
The component can auto-detect enum options from the schema or render options that you pass manually.

## Styling Guidelines
* All examples use Tailwind classes.
* The component itself respects the Contember theme classes on `select` and `label` elements. No extra setup is required.

## Core Concepts
* Auto-detection pulls valid enum values directly from the project schema.
* Manual options support two shapes: object map `{ admin: 'Administrator' }` or array `{ value, label }[]`.
* Mixed value types are fine—string, number, boolean, and `null`.
* `required` prop enforces validation before persist.
* Use `placeholder` to render an empty option.
* You can supply `defaultValue` for initial form state.

## Quick Start
```tsx
import { SelectEnumField } from '~/lib/form'

export const ArticleStatusSelect = () => (
  <SelectEnumField
    field="articleStatus"
    label="Publication Status"
    required
    className="w-full max-w-sm"
  />
)
```

## Examples

### 1. Mixed value types
```tsx
<SelectEnumField
  field="notificationSettings"
  label="Alert Preferences"
  placeholder="Select preference..."
  options={[
    { value: 'email', label: 'Email Notifications' },
    { value: 1, label: 'SMS Alerts' },
    { value: null, label: 'No Notifications' },
  ]}
  className="w-64"
/>
```
`null` becomes an explicit "no value" choice. Contember will store `null` when the option is selected.

### 2. Object-based options
```tsx
<SelectEnumField
  field="userRole"
  label="Account Type"
  options={{
    admin: 'Administrator',
    user: 'Standard User',
    guest: 'Temporary Access',
  }}
  className="w-56"
/>
```
When you provide an object map, keys are stored while values are shown to users.

### 3. Required validation
```tsx
<SelectEnumField
  field="enumValue"
  label="Some enum"
  required
  className="w-full"
/>
```
When the user leaves the field empty, Contember highlights the control and blocks persisting.

## Best Practices & Gotchas
* Keep enum names lowercase and without spaces to prevent serialization issues.
* Always set `required` when the database column is `NOT NULL`.
* Use `defaultValue` instead of pre-setting `options[0]` to avoid dirty state on form open.
* Do not mix object map and array options in the same instance.
* Place wider selects (`className="w-full"`) inside responsive grids for mobile friendliness.
