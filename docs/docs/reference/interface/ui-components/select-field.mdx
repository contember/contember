## Overview
SelectField lets you pick a single related entity (hasOne) through a dropdown.
You must use it inside an Entity or EntityList context.

## Styling Guidelines
* All examples use Tailwind classes.
* The rendered `<select>` element already carries Contember’s form theme classes (`ct-select`).

## Core Concepts
* Works only inside `<EntitySubTree />` or `<EntityListSubTree />`.
* Binds the chosen option to a hasOne relation.
* Accepts children that define how each option renders.
* Can sort, filter, or query options.
* Can embed a form for creating a new related entity.

## Quick Start
```tsx
import { EntitySubTree, Field, SelectField } from '~/lib/form'

<EntitySubTree entity="User">
	<SelectField
		field="country"
		label="Country"
		className="w-full ct-select"
	>
		<Field field="name" />
	</SelectField>
</EntitySubTree>
```

## Detailed Guide
```tsx
import { EntitySubTree, Field, SelectField } from '~/lib/form'
import { CountryForm } from './CountryForm'

<EntitySubTree entity="Author">
	<SelectField
		field="country"
		label="Home country"
		placeholder="Choose a country"
		initialSorting={{ name: 'asc' }}
		createNewForm={<CountryForm />}
		className="w-72 ct-select"
	>
		<Field field="name" />
	</SelectField>
</EntitySubTree>
```
* `initialSorting` orders the dropdown alphabetically.
* `createNewForm` opens `CountryForm` in a dialog when the user wants to add a missing country.
* `placeholder` shows until a value is selected.

## Examples
### 1. Filter archived authors
```tsx
<SelectField
	field="author"
	label="Author"
	options="Author[archived != false]"
	className="w-64 ct-select"
>
	<Field field="name" />
</SelectField>
```
This query hides archived authors.

### 2. Add new article on the fly
```tsx
<SelectField
	field="article"
	label="Article"
	createNewForm={<ArticleForm />}
	className="w-full ct-select"
>
	<Field field="internalName" />
</SelectField>
```
`ArticleForm` appears in a modal when the user clicks “Create new”.

## Best Practices & Gotchas
* Always wrap SelectField in an Entity context; otherwise it throws.
* Provide a child `<Field>` for readable option labels.
* Prefer `initialSorting` to keep large lists predictable.
* Avoid heavy `createNewForm` components; keep the dialog lightweight.
* Combine `queryField` with options filtering for fast search.
