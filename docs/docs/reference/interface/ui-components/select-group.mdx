# SelectGroup

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No special Contember theme classes are required for `SelectGroup`.

## Core Concepts
* `SelectGroup` groups related `SelectItem` elements inside a `Select` dropdown.
* It does not render visible UI by itself; it wraps children in a semantic `<div>`.
* Use groups to improve keyboard navigation and screen-reader structure.

## Quick Start
```tsx
import {
  Select,
  SelectTrigger,
  SelectValue,
  SelectContent,
  SelectGroup,
  SelectItem,
} from '~/lib/ui/select'

export const CountryPicker = () => (
  <Select>
    <SelectTrigger className="w-56">
      <SelectValue placeholder="Choose a country" />
    </SelectTrigger>

    <SelectContent className="bg-white shadow-md">
      <SelectGroup>
        <SelectItem value="us">United States</SelectItem>
        <SelectItem value="de">Germany</SelectItem>
        <SelectItem value="jp">Japan</SelectItem>
      </SelectGroup>
    </SelectContent>
  </Select>
)
```

## Sub-Components / Building Blocks
### SelectItem
Represents a single option inside the group.

### SelectTrigger
Button the user clicks to open the dropdown.

### SelectContent
Popup element that houses one or more `SelectGroup` components.

### SelectValue
Displays the currently selected item.

## Examples

### Country list with flags
```tsx
<SelectGroup>
  {COUNTRIES.map(country => (
    <SelectItem key={country.code} value={country.code}>
      <span className="flex items-center gap-2">
        <span className="text-base">{country.flag}</span>
        <span>{country.dialCode}</span>
        <span className="text-muted-foreground text-xs">{country.name}</span>
      </span>
    </SelectItem>
  ))}
</SelectGroup>
```
This example shows how you can render emojis, phone codes, and muted helper text inside each item.

### Custom "add new" action
```tsx
<SelectGroup>
  <SelectItem value="add-custom">
    <span className="flex items-center gap-2">
      <PlusCircle className="w-4 h-4" />
      <span>Other country</span>
    </span>
  </SelectItem>
</SelectGroup>
```
The action appears as the last item. Handle the `"add-custom"` value in your `onValueChange` callback.

## Best Practices & Gotchas
* Keep each group focused on one category.
* Avoid mixing action items and data items in the same group.
* Wrap heavy item content (icons, flags) in `flex` containers for alignment.
* Do not apply margins to `SelectGroup`; style `SelectContent` instead.
