## Overview  
`SelectInputActionsUI` renders the right-hand action slot inside `SelectInputUI`.  
Place icons, buttons, or triggers here. The component holds no state; it only positions its children.

## Styling Guidelines  
* All examples use Tailwind utility classes.  
* No special Contember theme classes are required for `SelectInputActionsUI`.

## Core Concepts  
* Sits inside `SelectInputUI` next to the selected label.  
* Accepts arbitrary React children (icons, buttons, loaders, etc.).  
* Relies on `flex` alignment from the parent.  
* Does **not** manage click handling; handle events inside the children.  

## Quick Start  
```tsx
import { SelectInputUI, SelectInputActionsUI } from '@contember/ui'
import { ChevronDownIcon } from '@radix-ui/react-icons'

export const SimpleSelectInput = () => (
  <SelectInputUI className="flex items-center gap-2 px-3 py-2 border rounded">
    Label or selected value
    <SelectInputActionsUI>
      <ChevronDownIcon className="w-4 h-4" />
    </SelectInputActionsUI>
  </SelectInputUI>
)
```

## Examples  

### Display a collapse / expand icon  
```tsx
<SelectInputActionsUI>
  <ChevronDownIcon className="w-4 h-4" />
</SelectInputActionsUI>
```
Use this when the select is closed. Swap it with `ChevronUpIcon` when open.

### Action group with remove button and dropdown icon  
```tsx
<SelectInputActionsUI className="flex items-center gap-1">
  <Button size="xs" variant="ghost">
    <XIcon className="w-4 h-4" />
  </Button>
  <ChevronDownIcon className="w-4 h-4 pointer-events-none" />
</SelectInputActionsUI>
```
Combine interactive and decorative elements. Apply `pointer-events-none` to non-interactive icons to let clicks reach the input.

### Loader with dynamic open/close icons  
```tsx
<SelectInputActionsUI className="flex items-center gap-1">
  {isLoading && <Loader size="sm" />}
  {open ? (
    <ChevronUpIcon className="w-4 h-4" />
  ) : (
    <ChevronDownIcon className="w-4 h-4" />
  )}
</SelectInputActionsUI>
```
Show a spinner while fetching options, then toggle the chevron based on the `open` state.

## Best Practices & Gotchas  
* Keep the slot compact; large children break vertical alignment.  
* Stop event propagation inside buttons to avoid closing the popover unintentionally.  
* Decorate only; never place heavy logic inside this presentational component.  
* Hide loaders when not needed to avoid layout shifts.
