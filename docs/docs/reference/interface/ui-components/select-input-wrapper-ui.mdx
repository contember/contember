# SelectInputWrapperUI

## Overview
SelectInputWrapperUI is a lightweight wrapper placed around SelectInputUI (and its multi-select variations).  
It provides the proper positioning so the input cooperates with PopoverTrigger and other Contember select parts.

## Styling Guidelines
* All examples use Tailwind utility classes.  
* The wrapper itself needs no extra Contember theme class.

## Core Concepts
* Always wrap SelectInputUI with this component.  
* Keep PopoverTrigger inside the wrapper.  
* The wrapper adds no visual styling—only layout logic.  

## Quick Start
```tsx
import { SelectInputWrapperUI, SelectInputUI } from '@contember/ui'

<Popover>
  <SelectInputWrapperUI>
    <PopoverTrigger asChild>
      <SelectInputUI className="w-full">
        Choose…
      </SelectInputUI>
    </PopoverTrigger>
  </SelectInputWrapperUI>
  {/* Popover content here */}
</Popover>
```

## Examples

### Basic select with clear trigger
```tsx
<SelectInputWrapperUI>
  <PopoverTrigger asChild>
    <SelectInputUI id={id ? `${id}-input` : undefined}>
      <SelectPlaceholder>
        <SelectDefaultPlaceholderUI />
      </SelectPlaceholder>

      <SelectEachValue>{children}</SelectEachValue>

      <SelectInputActionsUI>
        <SelectEachValue>
          <SelectItemTrigger>
            <Button size="xs" variant="ghost">
              <XIcon className="w-4 h-4" />
            </Button>
          </SelectItemTrigger>
        </SelectEachValue>
        <ChevronDownIcon className="w-4 h-4 pointer-events-none" />
      </SelectInputActionsUI>
    </SelectInputUI>
  </PopoverTrigger>
</SelectInputWrapperUI>
```
The wrapper keeps the trigger aligned while allowing each selected item to render its own clear button.

### Select inside a form field with label and description
```tsx
<SelectInputWrapperUI>
  <FormContainerUI>
    {label && (
      <FormLabelWrapperUI>
        <FormLabelUI required={required}>{label}</FormLabelUI>
      </FormLabelWrapperUI>
    )}

    <div>
      <PopoverTrigger asChild>
        <SelectInputUI disabled={disabled} className={className}>
          {selectedItem ? (
            getItemLabel(selectedItem)
          ) : placeholder ? (
            <span className="text-gray-400">{placeholder}</span>
          ) : (
            <SelectDefaultPlaceholderUI />
          )}

          <SelectInputActionsUI>
            {isLoading && <Loader size="sm" />}
            {open ? (
              <ChevronUpIcon className="w-4 h-4" />
            ) : (
              <ChevronDownIcon className="w-4 h-4" />
            )}
          </SelectInputActionsUI>
        </SelectInputUI>
      </PopoverTrigger>
    </div>

    {description && (
      <FormDescriptionUI>{description}</FormDescriptionUI>
    )}
  </FormContainerUI>
</SelectInputWrapperUI>
```
This pattern shows the wrapper used together with Contember form elements.  
Labels, descriptions, and loaders sit outside the wrapper; the wrapper focuses on aligning the select input with the popover.

## Best Practices & Gotchas
* Place PopoverTrigger **inside** SelectInputWrapperUI.  
* Do not add margins to the wrapper—style surrounding containers instead.  
* Keep exactly one child inside the wrapper.
