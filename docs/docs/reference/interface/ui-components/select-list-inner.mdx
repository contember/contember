## Overview
SelectListInner renders a scrollable list used by Contember select components.
It composes smaller building blocks such as `SelectOption` and can display an optional filter toolbar.

## Styling Guidelines
All examples use Tailwind.
If you follow the default Contember theme, no extra utility classes are required.

## Core Concepts
- Composition with `SelectOption`, `SelectItemTrigger`, and UI wrappers.
- Optional `filterToolbar` prop injects custom filtering UI above the list.
- Designed to sit inside popovers or dropdowns.
- Renders children without additional layout constraints, so you control item structure.

## Quick Start
```tsx
import { SelectListInner, SelectOption, SelectItemTrigger, SelectDefaultFilter } from '@app/lib/select'

<SelectListInner filterToolbar={<SelectDefaultFilter />}>
	<SelectOption>
		<SelectItemTrigger>
			<div className="px-3 py-2 hover:bg-gray-100">Item&nbsp;1</div>
		</SelectItemTrigger>
	</SelectOption>

	<SelectOption>
		<SelectItemTrigger>
			<div className="px-3 py-2 hover:bg-gray-100">Item&nbsp;2</div>
		</SelectItemTrigger>
	</SelectOption>
</SelectListInner>
```

## Detailed Guide
You can wrap items in your own container to add separators or custom padding.

```tsx
<SelectListInner filterToolbar={<SelectDefaultFilter />}>
	<div className="border-b py-1">
		<SelectOption>
			<SelectItemTrigger>
				<SelectListItemUI>First</SelectListItemUI>
			</SelectItemTrigger>
		</SelectOption>
	</div>

	<SelectOption>
		<SelectItemTrigger>
			<SelectListItemUI>Second</SelectListItemUI>
		</SelectItemTrigger>
	</SelectOption>
</SelectListInner>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| filterToolbar | ReactNode | No | `undefined` | Element shown above the list, typically `<SelectDefaultFilter />`. |


## Sub-Components / Building Blocks
### SelectOption
Wraps a single selectable entity.
Pass it as a direct child of `SelectListInner`.

### SelectItemTrigger
Handles keyboard and pointer interaction for its parent `SelectOption`.

### SelectListItemUI
Themeable UI wrapper for every item.
Use it to keep visual consistency with Contember design system.

## Examples
### Multi-select with inline create dialog
```tsx
<MultiSelect field="tags">
	<SelectListInner filterToolbar={<SelectDefaultFilter />}>
		<SelectOption>
			<SelectItemTrigger>
				<SelectListItemUI>Tag name</SelectListItemUI>
			</SelectItemTrigger>
		</SelectOption>
	</SelectListInner>
</MultiSelect>
```
The list lives inside a `MultiSelect` component.
`filterToolbar` lets users narrow down large datasets before choosing.

### Data-grid relation filter
```tsx
<SelectListInner filterToolbar={<SelectDefaultFilter />}>
	<DataGridRelationFilterSelectItem>
		{children}
	</DataGridRelationFilterSelectItem>
</SelectListInner>
```
`SelectListInner` powers a data-grid filter.
Wrapping items in `DataGridRelationFilterSelectItem` keeps the filter UI consistent.

### Data-view relation filter with include / exclude logic
```tsx
({ children, toggleExclude, toggleInclude, included, excluded, entityName, initialSorting }: {
	children: ReactNode
	toggleInclude: (entityId: EntityId) => void
	toggleExclude: (entityId: EntityId) => void
	included: string[]
	excluded: string[]
	entityName: string
	initialSorting?: DataViewSortingDirections
}) => (
	<DataViewRelationFilterOptions entities={entityName} initialSorting={initialSorting}>
		<SelectListInner filterToolbar={<SelectDefaultFilter />}>
			<DataGridRelationFilterSelectItem
				toggleInclude={toggleInclude}
				toggleExclude={toggleExclude}
				included={included}
				excluded={excluded}
			>
				{children}
			</DataGridRelationFilterSelectItem>
		</SelectListInner>
	</DataViewRelationFilterOptions>
)
```
The component glues together `DataViewRelationFilterOptions` and `DataGridRelationFilterSelectItem`.
`SelectListInner` stays responsible for scrolling, focus handling, and optional filtering UI.

## Best Practices & Gotchas
- Always place `SelectOption` directly under `SelectListInner`.
- Stop event propagation inside removal buttons to keep focus intact.
- Render lightweight items; virtualisation is not built-in.
- Provide unique keys on custom list containers to avoid React warnings.
