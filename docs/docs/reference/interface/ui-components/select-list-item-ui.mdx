# SelectListItemUI

## Overview
SelectListItemUI renders one option inside a Contember `Select` or `MultiSelect` list.  
Wrap the item in `SelectItemTrigger` to commit the value when users click the row.

## Styling Guidelines
* All examples use Tailwind CSS classes.  
* The component adds no styles by itself; apply Tailwind or your Contember theme classes (e.g. `contember-select-item`) as needed.

## Core Concepts
* SelectListItemUI is purely visual; behaviour comes from surrounding triggers.
* Place any content (text, icons, loaders) between its tags.
* Combine it with `SelectItemTrigger` to handle clicks without manual event wiring.
* Works in both single- and multi-select inputs.

## Quick Start
```tsx
import { SelectListItemUI, SelectItemTrigger, SelectOption } from '@contember/react-select'

<SelectOption>
	<SelectItemTrigger>
		<SelectListItemUI className="px-3 py-2 hover:bg-primary-50">
			Rome
		</SelectListItemUI>
	</SelectItemTrigger>
</SelectOption>
```

## Detailed Guide
```tsx
import {
	SelectInputUI,
	SelectListInner,
	SelectOption,
	SelectItemTrigger,
	SelectListItemUI,
	SelectPopoverContent,
} from '@contember/react-select'

<Popover>
	<PopoverTrigger asChild>
		<SelectInputUI className="w-56" />
	</PopoverTrigger>

	<SelectPopoverContent className="p-0">
		<SelectListInner>
			{/* Option 1 */}
			<SelectOption>
				<SelectItemTrigger>
					<SelectListItemUI className="px-3 py-2 cursor-pointer">
						Barcelona
					</SelectListItemUI>
				</SelectItemTrigger>
			</SelectOption>

			{/* Option 2 */}
			<SelectOption>
				<SelectItemTrigger>
					<SelectListItemUI className="px-3 py-2 cursor-pointer">
						Prague
					</SelectListItemUI>
				</SelectItemTrigger>
			</SelectOption>
		</SelectListInner>
	</SelectPopoverContent>
</Popover>
```
`SelectItemTrigger` closes the popover and writes the chosen value. You handle no state yourself.

## Examples
### 1. Show loading spinner inside item
```tsx
<SelectListItemUI>
	<Loader size="md" />
</SelectListItemUI>
```
You can display a loader while the list data downloads.

### 2. Render error message
```tsx
<SelectListItemUI>{error}</SelectListItemUI>
```
Show a full-width error placeholder instead of selectable options.

### 3. Non-interactive “no data” note
```tsx
<SelectListItemUI className="hover:cursor-default hover:bg-transparent">
	{componentLabels.simpleSelect.noData}
</SelectListItemUI>
```
Remove pointer cursor and hover background to signal that the row is not clickable.

### 4. Map dynamic list with custom click handler
```tsx
(item, index) => (
	<SelectListItemUI
		key={getItemValue ? String(getItemValue(item)) : `item-${index}`}
		onClick={() => handleItemClick(item)}
	>
		{getItemLabel(item)}
	</SelectListItemUI>
)
```
When mapping over data, forward the item to an external handler that writes the value and closes the popover.

### 5. Update form field and close popover
```tsx
([value, label]) => (
	<SelectListItemUI
		key={value}
		onClick={() => {
			fieldAccessor.updateValue(value)
			setOpen(false)
		}}
	>
		{label}
	</SelectListItemUI>
)
```
Clicking the item writes `value` to the form field and closes the dropdown.

### 6. Nest inside SelectListInner with filter toolbar
```tsx
<SelectListInner filterToolbar={<SelectDefaultFilter />}>
	<div className="border-b py-1">
		<SelectOption>
			<SelectItemTrigger>
				<SelectListItemUI className="px-3 py-2">
					{children}
				</SelectListItemUI>
			</SelectItemTrigger>
		</SelectOption>
	</div>
</SelectListInner>
```
Combine list items with custom toolbars, dividers, or other UI elements.

## Best Practices & Gotchas
* Always wrap SelectListItemUI in `SelectItemTrigger`; otherwise clicks do nothing.  
* Keep item padding consistent with `SelectInputUI` for visual alignment.  
* Use descriptive keys when mapping over dynamic lists to avoid React warnings.  
* Prevent event bubbling if you render interactive children inside the item.  
* Remove pointer cursor for informational rows so users know they cannot select them.

## Further Reading
* SelectItemTrigger – commits the value.  
* SelectOption – passes context to triggers and list items.  
* SelectInputUI – renders the input box.
