# SelectListItemUI

## Overview
SelectListItemUI renders one option inside a Contember `Select` or `MultiSelect` list.  
You wrap it in `SelectItemTrigger` to commit the value when users click the row.

## Styling Guidelines
* All examples use Tailwind CSS classes.  
* The component adds no styles by itself; apply Tailwind or your Contember theme classes (e.g. `contember-select-item`) as needed.

## Core Concepts
* SelectListItemUI is only visual; behaviour comes from its surrounding triggers.
* You place any custom content (text, icons) between its tags.
* Combine it with `SelectItemTrigger` to handle clicks without manual event wiring.
* It works in both single- and multi-select inputs.

## Quick Start
```tsx
import { SelectListItemUI, SelectItemTrigger, SelectOption } from '@contember/react-select'

<SelectOption>
	<SelectItemTrigger>
		<SelectListItemUI className="px-3 py-2 hover:bg-primary-50">
			Rome
		</SelectListItemUI>
	</SelectItemTrigger>
</SelectOption>
```

## Detailed Guide
```tsx
import {
	SelectInputUI,
	SelectListInner,
	SelectOption,
	SelectItemTrigger,
	SelectListItemUI,
	SelectPopoverContent,
} from '@contember/react-select'

<Popover>
	<PopoverTrigger asChild>
		<SelectInputUI className="w-56" />
	</PopoverTrigger>

	<SelectPopoverContent className="p-0">
		<SelectListInner>
			{/* Option 1 */}
			<SelectOption>
				<SelectItemTrigger>
					<SelectListItemUI className="px-3 py-2 cursor-pointer">
						Barcelona
					</SelectListItemUI>
				</SelectItemTrigger>
			</SelectOption>

			{/* Option 2 */}
			<SelectOption>
				<SelectItemTrigger>
					<SelectListItemUI className="px-3 py-2 cursor-pointer">
						Prague
					</SelectListItemUI>
				</SelectItemTrigger>
			</SelectOption>
		</SelectListInner>
	</SelectPopoverContent>
</Popover>
```
The trigger closes the popover and writes the chosen value. You handle no state yourself.

## Examples
### Update form field and close popover
```tsx
({ value, label, fieldAccessor, setOpen }: Props) => (
	<SelectListItemUI
		className="px-3 py-2 hover:bg-primary-50"
		onClick={() => {
			fieldAccessor.updateValue(value)
			setOpen(false)
		}}
	>
		{label}
	</SelectListItemUI>
)
```
Clicking the item writes the `value` to the form field and closes the dropdown.

### Nest inside SelectListInner with filter toolbar
```tsx
<SelectListInner filterToolbar={<SelectDefaultFilter />}>
	<div className="border-b py-1">
		<SelectOption>
			<SelectItemTrigger>
				<SelectListItemUI className="px-3 py-2">
					{children}
				</SelectListItemUI>
			</SelectItemTrigger>
		</SelectOption>
	</div>
</SelectListInner>
```
You can insert filtering components and dividers between list items.

## Best Practices & Gotchas
* Always wrap SelectListItemUI in `SelectItemTrigger`; otherwise clicks do nothing.  
* Keep item padding consistent with `SelectInputUI` for visual alignment.  
* Use descriptive keys when mapping over dynamic lists to avoid React warnings.  
* Prevent event bubbling if you also render interactive children inside the item.

## Further Reading
* SelectItemTrigger – commits the value.  
* SelectOption – passes context to triggers and list items.  
* SelectInputUI – renders the input box.
