## Overview
`SelectPopoverContent` wraps the dropdown area of Contember select inputs.
It renders the list of options inside a Radix Popover and handles scrolling and positioning.

## Styling Guidelines
* All snippets use Tailwind classes.
* The component itself adds Contember UI theme classes; you usually style only its children (`SelectListItemUI`, etc.).

## Core Concepts
* Radix Popover drives accessibility and positioning.
* `SelectPopoverContent` displays the options list.
* It is always paired with `PopoverTrigger` and `SelectInputUI`.
* You control selection in the parent (`Select`, `MultiSelect`, or custom logic).

## Quick Start
```tsx
import {
	Popover,
	PopoverTrigger,
	SelectInputUI,
	SelectInputActionsUI,
	SelectInputWrapperUI,
	SelectListItemUI,
	SelectPopoverContent,
} from '@contember/ui'

export const SimpleSelect = () => (
	<Popover>
		<SelectInputWrapperUI>
			<PopoverTrigger asChild>
				<SelectInputUI className="w-48 px-3 py-2 border rounded">
					<span className="text-gray-400">Choose…</span>
					<SelectInputActionsUI>
						<ChevronDownIcon className="w-4 h-4" />
					</SelectInputActionsUI>
				</SelectInputUI>
			</PopoverTrigger>
		</SelectInputWrapperUI>

		<SelectPopoverContent className="max-h-60 overflow-auto">
			{['A', 'B', 'C'].map(value => (
				<SelectListItemUI key={value} onClick={() => console.log(value)}>
					{value}
				</SelectListItemUI>
			))}
		</SelectPopoverContent>
	</Popover>
)
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| children | React.ReactNode | No | — | Elements shown in the dropdown. |
| …rest (inherited) | Radix `Popover.ContentProps` | No | — | Any prop supported by Radix Popover.Content. |

## Examples

### Enum field select
```tsx
<SelectPopoverContent>
	{normalizedOptions.map(({ value, label }) => (
		<SelectListItemUI
			key={value}
			onClick={() => {
				fieldAccessor.updateValue(value)
				setOpen(false)
			}}
		>
			{label}
		</SelectListItemUI>
	))}
</SelectPopoverContent>
```
You update the form state and close the popover from inside a list item.

### Data-view backed select
```tsx
<SelectPopoverContent>
	<SelectEntityDataView {...rest}>
		<DefaultSelectDataViewRenderer>{children}</DefaultSelectDataViewRenderer>
	</SelectEntityDataView>
</SelectPopoverContent>
```
`SelectEntityDataView` handles server paging, filtering, and sorting.
`SelectPopoverContent` only provides the scrollable container.

### Searchable select with loading indicator
```tsx
<SelectPopoverContent>
	{searchable && (
		<div className="px-2 py-1.5 pb-4">
			<div className="relative">
				<Search className="absolute left-2.5 top-2 h-4 w-4 text-gray-500" />
				<Input
					placeholder="Search..."
					className="pl-8"
					value={searchQuery}
					onChange={e => setSearchQuery(e.target.value)}
					inputSize="sm"
					autoFocus
				/>
			</div>
		</div>
	)}
	<ScrollArea className="max-h-80">
		{isLoading && (
			<SelectListItemUI>
				<Loader size="md" />
			</SelectListItemUI>
		)}
		{error && <SelectListItemUI>{error}</SelectListItemUI>}
		{!isLoading && filteredItems?.length === 0 && (
			<SelectListItemUI className="hover:cursor-default hover:bg-transparent">
				{componentLabels.simpleSelect.noData}
			</SelectListItemUI>
		)}
		{filteredItems?.map((item, index) => (
			<SelectListItemUI
				key={getItemValue ? String(getItemValue(item)) : `item-${index}`}
				onClick={() => handleItemClick(item)}
			>
				{getItemLabel(item)}
			</SelectListItemUI>
		))}
	</ScrollArea>
</SelectPopoverContent>
```
The search input sits above a scrollable list.
Loading, empty, and error states reuse `SelectListItemUI` for visual consistency.

### Data-grid filter select
```tsx
<Popover>
	<PopoverTrigger asChild>
		<DataGridFilterSelectTriggerUI>{label}</DataGridFilterSelectTriggerUI>
	</PopoverTrigger>
	<SelectPopoverContent>
		<DataViewQueryFilter
			toggleInclude={toggleInclude}
			toggleExclude={toggleExclude}
			included={included}
			excluded={excluded}
			entityName={entityName}
			initialSorting={initialSorting}
		>
			{children}
		</DataViewQueryFilter>
	</SelectPopoverContent>
</Popover>
```
You wrap `DataViewQueryFilter` to offer include/exclude toggles inside the popover.

## Best Practices & Gotchas
* Always place `SelectPopoverContent` inside the same `Popover` that holds the trigger.
* Keep the content width similar to the trigger for visual consistency.
* Limit height with `max-h-* overflow-auto` to maintain usability.
* Close the popover after a successful selection to prevent duplicate clicks.
* Forward unknown props to benefit from Radix features like `side` or `align`.
