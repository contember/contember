# SelectValue

## Overview
SelectValue renders the currently selected option inside a SelectTrigger.  
It can also show a placeholder when no value is chosen.

## Styling Guidelines
* All examples use Tailwind utility classes.  
* No extra Contember theme classes are required.

## Core Concepts
* SelectValue is only meaningful inside a SelectTrigger.  
* You pass children to display custom content, or use the `placeholder` prop.  
* SelectValue does not handle state; the parent Select manages it.

## Quick Start
```tsx
import { Select, SelectTrigger, SelectValue, SelectContent, SelectItem } from '~/lib/ui/select'

export const QuickStart = () => (
  <Select defaultValue="a">
    <SelectTrigger className="w-64">
      <SelectValue placeholder="Choose an option" />
    </SelectTrigger>

    <SelectContent>
      <SelectItem value="a">Option A</SelectItem>
      <SelectItem value="b">Option B</SelectItem>
    </SelectContent>
  </Select>
)
```

## Examples

### Custom render: country picker
```tsx
<SelectValue>
  <span className="flex items-center gap-2">
    <span className="text-base">{selectedCountryDisplay.flag}</span>
    <span>{selectedCountryDisplay.dialCode}</span>
  </span>
</SelectValue>
```
This pattern shows how you can compose arbitrary JSX inside SelectValue to enrich the label.

### Placeholder only
```tsx
<SelectTrigger id="country-select">
  <SelectValue placeholder="Vyberte zemi" />
</SelectTrigger>
```
When no value is selected, the placeholder text appears.

## Best Practices & Gotchas
* Always wrap SelectValue inside SelectTrigger.  
* Keep the content concise; avoid heavy components inside the label.  
* Provide a meaningful placeholder for better accessibility.  
* Style via Tailwind on the surrounding Trigger; avoid inline styles in SelectValue.
