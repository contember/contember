# Select Component Family

## Overview
`Select` renders an accessible dropdown menu.  
You compose it from small sub-components such as `SelectTrigger`, `SelectContent`, and `SelectItem` to control every piece of the UI.

## Core Concepts
* Composition over configuration – build the final UI by nesting sub-components.
* Uncontrolled vs. controlled – pass `value` / `onValueChange` to control the selection.
* Portalled content – `SelectContent` is rendered in a portal so it can break out of overflow/scroll containers.
* Keyboard accessibility – focus management and arrow-key navigation are built-in.

## Quick Start
```tsx
import { Select, SelectTrigger, SelectContent, SelectItem } from '@contember/react-ui/select'

export const FruitPicker = () => (
  <Select>
    <SelectTrigger>Choose</SelectTrigger>
    <SelectContent>
      <SelectItem value="apple">Apple</SelectItem>
      <SelectItem value="banana">Banana</SelectItem>
    </SelectContent>
  </Select>
)
```

## Detailed Guide
The component tree always follows the same pattern:

1. `Select` – parent context  
2. `SelectTrigger` – button that opens the menu  
3. `SelectContent` – portal with menu items  
4. `SelectItem` – one option (optionally wrapped in `SelectGroup`)  
5. `SelectValue` – placeholder or current value  
6. `SelectLabel` – textual label inside a group

Example with two groups:

```tsx
<Select>
  <SelectTrigger>Choose country</SelectTrigger>

  <SelectContent>
    <SelectGroup>
      <SelectItem value="apple">Apple</SelectItem>
      <SelectItem value="banana">Banana</SelectItem>
    </SelectGroup>

    <SelectGroup>
      <SelectItem value="cucumber">Cucumber</SelectItem>
      <SelectItem value="paprika">Paprika</SelectItem>
    </SelectGroup>
  </SelectContent>
</Select>
```

## Best Practices & Gotchas
* Always wrap items in `SelectContent`; they will not work elsewhere.
* Keep `value` stable (string/number) to avoid unnecessary re-renders.
* Add an explicit `placeholder` in `SelectValue` to improve accessibility.
* Style `SelectContent` with Tailwind `max-h` and `overflow-auto` when you expect many items.
* Do not nest multiple `Select` components – they each need their own context.
