## Styling Guidelines
* All code snippets use Tailwind CSS.
* No Contember-specific theme classes are required for `SheetContent`.

## Core Concepts
* `SheetContent` renders the visible panel inside a `Sheet`.
* You control placement with the `side` prop (`left`, `right`, `top`, `bottom`).
* Pass regular DOM props (`className`, `style`, etc.) for sizing and colors.
* Combine `SheetTrigger`, `Sheet`, and `SheetContent` for a complete drawer.

## Quick Start
```tsx
import { Sheet, SheetTrigger, SheetContent } from '~/lib/ui/sheet'

export function Demo() {
	return (
		<Sheet>
			<SheetTrigger className="px-4 py-2 bg-blue-600 text-white rounded">
				Open panel
			</SheetTrigger>

			<SheetContent side="right" className="w-64 p-6 bg-white shadow-lg">
				<h2 className="text-xl font-semibold mb-4">Hello</h2>
				<p className="text-sm text-gray-700">I slide in from the right.</p>
			</SheetContent>
		</Sheet>
	)
}
```

## Detailed Guide
### Choosing a side
```tsx
<SheetContent side="left" className="w-56 p-4 bg-gray-50" />
```
* Use `side` to match your layout needs.
* Omit `side` to fall back to the library default (usually `"right"`).

### Rendering custom elements with `asChild`
```tsx
<SheetContent asChild side="right">
	<aside className="w-80 p-6 bg-white">…</aside>
</SheetContent>
```
* `asChild` lets you keep semantic tags like `<aside>` while preserving sheet behavior.

## Props Reference
| Name | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| side | `"left" \| "right" \| "top" \| "bottom"` | No | `"right"` | Drawer placement. |
| asChild | `boolean` | No | `false` | Renders passed child element instead of wrapping `div`. |
| className | `string` | No | `""` | Tailwind utility classes. |
| style | `React.CSSProperties` | No | `{}` | Inline styles for dynamic sizing. |
| …HTMLDivElement props | — | No | — | Standard DOM attributes. |

## Examples
### Mobile sidebar toggle
```tsx
export function RightSidebar({ open, setOpen, children }: PropsWithChildren<{ open: boolean; setOpen: (v: boolean) => void }>) {
	const isMobile = useIsMobile(1024)

	if (isMobile) {
		return (
			<Sheet open={open} onOpenChange={setOpen}>
				<SheetContent side="right" className="w-72 p-4 pt-12">
					<button onClick={() => setOpen(false)} className="absolute right-4 top-4 lg:hidden">
						Close
					</button>
					{children}
				</SheetContent>
			</Sheet>
		)
	}

	return <aside>{children}</aside>
}
```
You render a temporary drawer on small screens and fall back to a static sidebar on desktops.

### Lightweight info panel
```tsx
<Sheet>
	<SheetTrigger className="text-sm text-gray-600">Details</SheetTrigger>
	<SheetContent className="bg-white w-60 p-5">
		<article className="prose">
			<h3>Order #123</h3>
			<p>Status: Shipped</p>
		</article>
	</SheetContent>
</Sheet>
```
Show additional context without navigating away.

## Best Practices & Gotchas
* Always wrap `SheetContent` in a `Sheet` provider.
* Avoid fixed widths; use responsive classes like `w-[--sidebar-width]`.
* Combine `asChild` with semantic tags (`<aside>`, `<nav>`).
* Remember to set an accessible close control inside the panel.
* Keep interaction flows consistent—slide in from the same side across screens.
