# SheetTrigger

## Overview
SheetTrigger wraps an interactive element and connects it to a Sheet component.
When users activate the wrapped element, the corresponding Sheet opens or closes.

## Styling Guidelines
* All snippets use Tailwind CSS.
* SheetTrigger itself adds no styles; style the wrapped element with Tailwind.
* No special Contember theme classes are required.

## Core Concepts
* SheetTrigger must be a child of a parent `Sheet`.
* It accepts any single React element and forwards events.
* You can style the wrapped element freely with Tailwind.

## Quick Start
```tsx
import { Sheet, SheetContent, SheetTrigger } from '~/lib/ui/sheet'

export const Demo = () => (
  <Sheet>
    <SheetTrigger>
      <button className="px-4 py-2 bg-blue-600 text-white rounded">
        Open panel
      </button>
    </SheetTrigger>

    <SheetContent side="right" className="p-6">
      <h2 className="text-lg font-semibold">Details</h2>
      <p className="mt-2 text-sm text-gray-600">Here goes the sheet content.</p>
    </SheetContent>
  </Sheet>
)
```

## Examples

### Display course progress information
```tsx
<SheetTrigger>
  <Button size="sm" className="flex items-center gap-1">
    <CourseProgressNumber courseProgressState={courseProgressState} />
    <InfoIcon size={14} />
  </Button>
</SheetTrigger>
```
The trigger wraps a styled `Button`.
When users click the button, the related sheet shows detailed course progress.

## Best Practices & Gotchas
* Always place SheetTrigger inside a `Sheet` component.
* Wrap exactly one interactive element (button, link, icon).
* Combine with accessible roles (`button` or `aria-haspopup`) when creating custom elements.
* Avoid putting focusable elements outside the trigger wrapper; they wonâ€™t open the sheet.
