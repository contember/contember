## Overview
`SideDimensions` renders content for a single dimension inside a flexible side panel.  
It wraps your fields with `DimensionRenderer` and a `HasOne` relationship, so you can edit translated or versioned data in isolation.

## Styling Guidelines
* All examples use Tailwind utility classes.  
* No Contember‑specific theme classes are required for `SideDimensions`.

## Core Concepts
* Dimension context – switch data by dimension (`locale`, `stage`, etc.).
* `as` variable – expose the selected dimension value to descendant components.
* Has‑one filtering – limit rendered entity by the `field` filter.
* Composition – place any form fields or layout components as children.

## Quick Start
```tsx
import { SideDimensions } from '~/lib/dimensions'

<SideDimensions
  dimension="locale"
  as="currentLocale"
  field="locales(locale.code = $currentLocale)"
>
  <h2 className="mb-4 font-semibold">
    <Variable name="currentLocale" />
  </h2>
  <InputField field="title" className="w-full" />
</SideDimensions>
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| dimension | `string` | Yes | – | Name of the dimension to render. |
| as | `string` | Yes | – | Variable name that stores the selected dimension value. |
| field | `string \| SugarableHasOneRelation \| SugarableHasOneRelation[]` | Yes | – | Field or filter used to locate the dimension entity. |
| children | `React.ReactNode` | Yes | – | Elements rendered inside the dimension context. |

## Examples

### Full translation editor with locale switcher
```tsx
<>
  <Slots.Title>
    <Title icon={<LanguagesIcon />}>Dimensions</Title>
  </Slots.Title>

  <Binding>
    <div>
      <DimensionsSwitcher
        options="DimensionsLocale"
        slugField="code"
        dimension="locale"
        isMulti
      >
        <Field field="label" />
      </DimensionsSwitcher>
    </div>
  </Binding>

  <Binding>
    <Slots.Actions><PersistButton /></Slots.Actions>
    <EntitySubTree entity="DimensionsItem(unique = unique)">
      <SideDimensions
        dimension="locale"
        as="currentLocale"
        field="locales(locale.code = $currentLocale)"
      >
        <Card>
          <CardHeader>
            <CardTitle><Variable name="currentLocale" /></CardTitle>
          </CardHeader>
          <CardContent>
            <InputField field="title" />
            <TextareaField field="content" />
          </CardContent>
        </Card>
      </SideDimensions>
    </EntitySubTree>
  </Binding>
</>
```
Explanation  
* `DimensionsSwitcher` lets users pick one or more locales.  
* `SideDimensions` renders the form for the selected locale only.  
* `currentLocale` is available for headings or conditional logic.  
* The layout remains isolated inside a card.

### Environment middleware with inline heading
```tsx
<EnvironmentMiddleware create={it => it.withDimensions({ locale: ['cs', 'en'] })}>
  <SideDimensions
    dimension="locale"
    field="locales(locale = $currentLocale)"
    as="currentLocale"
  >
    <h2 className="text-2xl mb-4">
      <Variable name="currentLocale" />
    </h2>
    <ArticleLocaleForm />
  </SideDimensions>
</EnvironmentMiddleware>

<DefaultRepeater field="notes" orderBy="createdAt" initialEntityCount={1}>
  <ArticleNoteForm />
</DefaultRepeater>

<Slots.Sidebar>
  <InputField field="internalName" required />
  <CheckboxField field="locked" />
  <InputField field="publishedAt" />
  <MultiSelectField field="tags" createNewForm={<ArticleTagForm />}>
    <Field field="name" />
  </MultiSelectField>
  <SelectField field="author" createNewForm={<ArticleAuthorForm />}>
    <Field field="name" />
  </SelectField>
</Slots.Sidebar>
```
Explanation  
* The environment middleware pre‑loads both locales.  
* `SideDimensions` switches the locale form without extra UI.  
* Unrelated forms (`notes`, sidebar inputs) coexist with the dimension‑aware part.

## Best Practices & Gotchas
* Always set both `dimension` and `as`; without `as` descendants cannot read the selected value.
* Keep the `field` filter as specific as possible to avoid accidental entity creation.
* Combine with `DimensionsSwitcher` to let users change the dimension.
* Wrap heavy content in cards to preserve side panel spacing.
