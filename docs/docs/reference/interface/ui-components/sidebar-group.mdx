# SidebarGroup

## Overview
SidebarGroup wraps a set of sidebar elements and keeps them visually together.  
Use it to separate logical blocks inside `SidebarContent` and `SidebarFooter`.

## Styling Guidelines
* All examples use Tailwind CSS.  
* SidebarGroup relies on Contember’s sidebar theme classes (`text-sidebar-…`, `bg-sidebar-…`) applied globally by the layout.

## Core Concepts
* Group related links, actions, or slots in one container.  
* Place SidebarGroup only inside `SidebarContent` or `SidebarFooter`.  
* Nest `SidebarGroupContent` for extra padding or borders.  
* Combine multiple groups to create clear visual sections.

## Quick Start
```tsx
import { Sidebar, SidebarContent, SidebarGroup } from '~/lib/ui/sidebar'

export function SimpleSidebar() {
	return (
		<Sidebar className="w-64 bg-sidebar-bg">
			<SidebarContent className="p-4 space-y-4">
				<SidebarGroup className="space-y-2">
					<a className="block text-sm font-medium" href="#">Dashboard</a>
					<a className="block text-sm font-medium" href="#">Reports</a>
				</SidebarGroup>
			</SidebarContent>
		</Sidebar>
	)
}
```

## Sub-Components / Building Blocks
### SidebarGroupContent
Wrap the actual content of a group when you need inner padding or borders.

```tsx
import { SidebarGroup, SidebarGroupContent } from '~/lib/ui/sidebar'

<SidebarGroup>
	<SidebarGroupContent className="p-4 border-t border-sidebar-border">
		{/* group items here */}
	</SidebarGroupContent>
</SidebarGroup>
```

## Examples
### Navigation and Quick Actions
```tsx
<SidebarContent>
	<SidebarGroup className="[&_svg]:text-sidebar-accent">
		<SlotTargets.Navigation />
	</SidebarGroup>

	<SidebarGroup>
		<SlotTargets.NavigationQuickActions />
	</SidebarGroup>
</SidebarContent>
```
This pattern keeps main navigation and quick actions visually separated while inheriting sidebar accent colors.

### Account and Workspace Section
```tsx
<SidebarFooter className="space-y-3">
	<SidebarGroup>
		<SlotTargets.Account />
	</SidebarGroup>

	<SidebarGroup>
		<SlotTargets.WorkSpace />
	</SidebarGroup>
</SidebarFooter>
```
Placing groups in the footer lets you cluster user-specific widgets away from global navigation.

## Best Practices & Gotchas
* Use one group per logical section; avoid over-nesting.  
* Apply spacing on the group, not individual items, for consistent layout.  
* Inherit icon colors with `[&_svg]:text-sidebar-accent-…` rather than styling each icon.  
* Keep groups empty only temporarily; remove them in production builds.
