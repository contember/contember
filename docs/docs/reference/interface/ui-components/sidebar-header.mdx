# SidebarHeader

## Overview
SidebarHeader displays the top section of a Contember sidebar.
You place any React nodes inside it—logo, close button, or custom heading.

## Styling Guidelines
* All examples use Tailwind utility classes.
* The Contember sidebar theme expects text colors defined by the `sidebar-*` palette.

## Core Concepts
* SidebarHeader must be nested inside `<Sidebar>`.
* It renders its `children` unchanged.
* You can combine it with SidebarMobileTrigger for responsive closing.
* Tailwind classes control padding, background, and positioning.

## Quick Start
```tsx
import { Sidebar, SidebarHeader } from '~/lib/ui/sidebar'

export default function Page() {
	return (
		<Sidebar>
			<SidebarHeader className="p-4 bg-sidebar-surface">
				<h1 className="text-lg font-semibold">My App</h1>
			</SidebarHeader>
		</Sidebar>
	)
}
```

## Detailed Guide
```tsx
import {
	Sidebar,
	SidebarHeader,
	SidebarContent,
	SidebarFooter,
	SidebarGroup,
	SidebarGroupContent,
} from '~/lib/ui/sidebar'

function AppLayout() {
	return (
		<Sidebar>
			<SidebarHeader className="p-4 relative">
				<img src="/logo.svg" alt="Logo" className="h-6" />
				{/* The trigger is optional but recommended on mobile */}
				<SidebarMobileTrigger className="absolute right-4 top-4" />
			</SidebarHeader>

			<SidebarContent>
				<SidebarGroup>
					<SidebarGroupContent>
						Navigation items go here
					</SidebarGroupContent>
				</SidebarGroup>
			</SidebarContent>

			<SidebarFooter>
				<p className="text-xs text-sidebar-muted">© 2024</p>
			</SidebarFooter>
		</Sidebar>
	)
}
```
The header uses `relative` positioning so the mobile trigger can be absolutely positioned within its bounds.

## Examples

### Logo with Close Button
```tsx
<SidebarHeader className="p-4 relative">
	<div>
		<Link to="index">
			<img src="/logo.svg" className="h-6" />
		</Link>
	</div>
	<SidebarMobileTrigger icon={<XIcon />} className="absolute right-4 top-4" />
</SidebarHeader>
```
Placing the trigger inside the same container ensures layout consistency on all breakpoints.

### Minimal Header in Playground Layout
```tsx
<SidebarHeader>
	<h1>Header</h1>
</SidebarHeader>
```
This basic variant is used in the Contember playground for quick prototyping.

## Best Practices & Gotchas
* Always keep SidebarHeader the first child of Sidebar.
* Add `className="p-4"` to align with default Contember paddings.
* Position interactive elements (like close icons) with `relative` + `absolute`.
* Do not place heavy logic inside the header; keep it presentational.
