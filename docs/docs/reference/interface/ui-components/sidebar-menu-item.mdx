## Styling Guidelines
* All examples use Tailwind CSS classes.
* SidebarMenuItem relies on Contember sidebar theme tokens such as `text-sidebar-accent-corporate` found in parent elements.

## Core Concepts
* SidebarMenuItem acts as a list item inside `SidebarMenu` or `SidebarMenuSub`.
* You wrap interactive content with `SidebarMenuButton`.
* Combine multiple items to build hierarchical, collapsible sidebars.
* The component itself is presentational; it forwards props to the underlying `<li>` element.

## Sub-Components / Building Blocks
### SidebarMenuButton
Wrap links or buttons. Use the `asChild` prop to pass your own anchor or link element.

### SidebarMenuAction
Render an optional action (e.g., badge or shortcut) next to the main button.

### SidebarMenuSub / SidebarMenuSubItem
Create a nested menu level when you need indented items.

## Best Practices & Gotchas
* Always place content inside `SidebarMenuButton`; otherwise focus and hover styles might break.
* Use `asChild` when integrating with `react-router`‚Äôs `<Link>` to avoid extra DOM wrappers.
* Keep SVG icons at a fixed size (e.g., `className="h-5 w-5"`) for visual alignment.
* Avoid mixing arbitrary `<li>` elements with `SidebarMenuItem`; it breaks keyboard navigation.
