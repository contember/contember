## Core Concepts
* Skeleton components communicate loading states.
* Sidebar skeletons keep layout stable while data loads.
* You render `SidebarMenuSkeleton` until real navigation items arrive.

## Quick Start

	<aside className="w-64 border-r border-gray-200">

## Examples

### Show skeleton until permissions load
```tsx
import { useProjectRoles } from '@contember/react-client'
import { SidebarMenuSkeleton } from '@contember/admin'
import { SidebarMenu } from './SidebarMenu'

export const SecureSidebar = () => {
	const { roles, isLoading } = useProjectRoles()

	if (isLoading) {
		return <SidebarMenuSkeleton />
	}
	return <SidebarMenu roles={roles} />
}
```
The sidebar stays visually consistent while `useProjectRoles` fetches role-based navigation.

## Best Practices & Gotchas
* Display the skeleton for at least 500 ms to avoid flicker.
* Keep the sidebar width identical between skeleton and final menu.
* Do not overlay content; reserve space with a fixed width container.
