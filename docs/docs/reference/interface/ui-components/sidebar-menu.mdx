# SidebarMenu

## Overview
SidebarMenu renders a vertical list of navigation or action items inside a `Sidebar`.
It works together with `SidebarMenuItem`, `SidebarMenuButton`, and optional sub-menus to build hierarchical sidebars.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* You can freely apply Contember theme classes such as `text-sidebar-accent-corporate` to harmonize with the default design system.

## Core Concepts
* SidebarMenu is just a container; it delegates interaction to its children.
* Combine `SidebarMenuItem` and `SidebarMenuButton` to create clickable rows.
* Place SidebarMenu inside `SidebarFooter`, `SidebarContent`, or any other sidebar section.
* Icons are commonly rendered before text and inherit color from the parent item.
* You can nest `SidebarMenuSub` inside an item for collapsible sub-navigation.

## Quick Start
```tsx
import {
  Sidebar,
  SidebarFooter,
  SidebarMenu,
  SidebarMenuItem,
  SidebarMenuButton,
} from '~/lib/ui/sidebar'

export default function BasicSidebar() {
  return (
    <Sidebar className="h-screen w-64 bg-gray-900 text-white">
      <SidebarFooter className="p-4">
        <SidebarMenu>
          <SidebarMenuItem>
            <SidebarMenuButton asChild>
              <a className="flex gap-2 hover:text-sidebar-accent-corporate">
                Dashboard
              </a>
            </SidebarMenuButton>
          </SidebarMenuItem>

          <SidebarMenuItem>
            <SidebarMenuButton asChild>
              <a className="flex gap-2 hover:text-sidebar-accent-corporate">
                Support
              </a>
            </SidebarMenuButton>
          </SidebarMenuItem>
        </SidebarMenu>
      </SidebarFooter>
    </Sidebar>
  )
}
```

## Sub-Components / Building Blocks

### SidebarMenuItem
Wraps a single row within the menu. Use one instance per link or action.

### SidebarMenuButton
Handles focus, hover and active states. Set `asChild` to pass the click target element.

### SidebarMenuSub
Optional collapsible container for nested items.

### SidebarMenuSubButton
Trigger that toggles a `SidebarMenuSub`.

### SidebarMenuSubItem
Child item inside a sub-menu.

### SidebarMenuAction
Utility component for contextual actions such as toggling or closing the sidebar.

### useSidebar (Hook)
Returns sidebar state and helpers like `toggle()` for responsive layouts.

## Examples

### 1. Support & Settings in a Footer
```tsx
<SidebarFooter className="p-4 border-t border-gray-700">
  <SidebarMenu>
    <SidebarMenuItem>
      <SidebarMenuButton asChild>
        <a className="flex gap-2">
          <LifeBuoyIcon className="w-4 h-4" />
          Support
        </a>
      </SidebarMenuButton>
    </SidebarMenuItem>

    <HasCompanyRole role="admin">
      <SidebarMenuItem>
        <SidebarMenuButton asChild>
          <Link to="company/settings" className="flex gap-2">
            <CogIcon className="w-4 h-4" />
            Company settings
          </Link>
        </SidebarMenuButton>
      </SidebarMenuItem>
    </HasCompanyRole>
  </SidebarMenu>
</SidebarFooter>
```
The footer groups two entries. `HasCompanyRole` hides the settings link when the user lacks permissions.

### 2. Persisting Open Items
```tsx
const [state, setState] = useStoredState<{ openedItems: string[] }>(
  'session',
  ['navigation', 'sidebar'],
  v => ({ openedItems: [], ...v }),
)

<MenuContext.Provider value={{ level: 0 }}>
  <OpenedItemsContext.Provider value={{ menuCollapsibleState: state, setMenuCollapsibleState: setState }}>
    <SidebarGroupLabel>Navigation</SidebarGroupLabel>
    <SidebarMenu>{children}</SidebarMenu>
  </OpenedItemsContext.Provider>
</MenuContext.Provider>
```
The app stores the list of opened sub-menus in `sessionStorage` to restore them after reload.

## Best Practices & Gotchas
* Keep text labels short; long strings break the fixed sidebar width.
* Always wrap anchors with `SidebarMenuButton asChild` to preserve focus styling.
* Use Tailwindâ€™s `hover:` and `aria-expanded` selectors to customize interactive states.
* Avoid putting heavy business logic in menu components; compute permissions outside and render conditionally.
* When nesting sub-menus, increment padding to show hierarchy clearly.
* Remember that icons inherit text color; apply only theme text classes, not background classes, to them.
