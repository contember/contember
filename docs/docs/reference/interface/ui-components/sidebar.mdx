## Styling Guidelines
All examples use Tailwind CSS utility classes.  
Some designs rely on Contember theme tokens such as `text-sidebar-accent-corporate` for brand-level coloring.

## Core Concepts
- Composition: build the sidebar by nesting header, content, and footer primitives.  
- Grouping: wrap related links in `SidebarGroup` to keep navigation readable.  
- Action rows: render quick actions inside `SidebarMenu` via `SidebarMenuItem` + `SidebarMenuButton`.  
- Context: call `useSidebar` to toggle the component in mobile layouts.

## Quick Start
```tsx
import {
	Sidebar,
	SidebarHeader,
	SidebarContent,
	SidebarFooter,
	SidebarGroup,
	SidebarMenu,
	SidebarMenuItem,
	SidebarMenuButton,
} from '~/lib/ui/sidebar'

export function BasicSidebar() {
	return (
		<Sidebar>
			<SidebarHeader className="p-4 font-bold">My App</SidebarHeader>

			<SidebarContent>
				<SidebarGroup>
					<nav className="flex flex-col gap-2">
						<a className="text-sm hover:underline" href="#">Dashboard</a>
						<a className="text-sm hover:underline" href="#">Reports</a>
					</nav>
				</SidebarGroup>
			</SidebarContent>

			<SidebarFooter className="p-4">
				<SidebarMenu>
					<SidebarMenuItem>
						<SidebarMenuButton asChild>
							<a href="#">Help</a>
						</SidebarMenuButton>
					</SidebarMenuItem>
				</SidebarMenu>
			</SidebarFooter>
		</Sidebar>
	)
}
```

## Sub-Components / Building Blocks
### SidebarHeader
Header area for logos, titles, or mobile triggers.

### SidebarContent
Scrollable middle section that holds navigation groups.

### SidebarFooter
Sticky bottom area for menus, account widgets, or status indicators.

### SidebarGroup
Logical wrapper that spaces its children consistently.

### SidebarMenu
Unordered list styled for action items.

### SidebarMenuItem
Single row inside a `SidebarMenu`.

### SidebarMenuButton
Clickable button; use `asChild` to forward props to an anchor or Link.

### useSidebar
React hook that exposes open/close state and imperative helpers.

## Examples
### Production-style sidebar with account widgets
```tsx
export function AppSidebar() {
	const env = useEnvironment()
	return (
		<Sidebar>
			<SidebarHeader className="p-4 relative">
				<div>
					<SlotTargets.Logo />
				</div>
				<SidebarMobileTrigger icon={<XIcon />} className="absolute right-4 top-4" />
			</SidebarHeader>

			<SidebarContent>
				<SidebarGroup className="[&_svg]:text-sidebar-accent-corporate">
					<SlotTargets.Navigation />
				</SidebarGroup>
				<SidebarGroup>
					<SlotTargets.NavigationQuickActions />
				</SidebarGroup>
			</SidebarContent>

			<SidebarFooter>
				<SidebarMenu>
					<SidebarMenuItem>
						<SidebarMenuButton asChild>
							<a href="#">
								<LifeBuoyIcon />
								{componentLabels.navigation.support}
							</a>
						</SidebarMenuButton>
					</SidebarMenuItem>

					{getIsCompanySelected(env) && (
						<HasCompanyRole role={hasAnyRole([COMPANY_ADMIN_ROLE, EDU_MANAGER_ROLE, 'admin'])}>
							<SidebarMenuItem>
								<SidebarMenuButton asChild>
									<Link to="company/settings">
										<a>
											<CogIcon />
											{componentLabels.navigation.companySettings}
										</a>
									</Link>
								</SidebarMenuButton>
							</SidebarMenuItem>
						</HasCompanyRole>
					)}
				</SidebarMenu>

				<div className="py-0">
					<SlotTargets.Account />
				</div>
				<div className="py-0">
					<SlotTargets.WorkSpace />
				</div>
				<div className="py-0">
					<SlotTargets.UserProfile />
				</div>
			</SidebarFooter>
		</Sidebar>
	)
}
```
The snippet illustrates:  
- Role-based menu rendering.  
- SVG icon theming with a single utility on `SidebarGroup`.  
- Account and workspace widgets inside the footer.

## Best Practices & Gotchas
- Keep navigation depth shallow; prefer multiple groups over deep nesting.  
- Always wrap quick links in `SidebarMenuButton` for consistent sizing and hover states.  
- Reuse `className="[&_svg]:text-sidebar-accent-corporate"` to tint all icons at once.  
- Call `useSidebar` in the layout root to control mobile visibility.
