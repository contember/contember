# `Skeleton`

## Overview
`Skeleton` renders a lightweight placeholder while data loads.  
You use it to reduce perceived waiting time and avoid layout shifts.

## Styling Guidelines
* All snippets rely on Tailwind CSS.  
* No Contember-specific theme classes are required.

## Core Concepts
* Show a skeleton while the real content fetches.
* Replace the skeleton with children once data arrives.
* Style through Tailwind utility classes or the `className` prop.

## Quick Start
```tsx
import { Skeleton } from '~/lib/ui/skeleton'

export default function Page() {
	return (
		<Skeleton className="h-4 w-32 rounded-md bg-gray-200 dark:bg-gray-700" />
	)
}
```

## Examples

### Wrapping a custom placeholder
```tsx
<Skeleton className="w-full bg-transparent">
	<CompanySwitcherPlaceholder label="Loading companies…" />
</Skeleton>
```
This pattern keeps the layout stable. Once the company list loads, replace the skeleton with the real switcher.

### Simple bar placeholder
```tsx
<Skeleton className="h-6 w-48 rounded bg-gray-200 animate-pulse" />
```
A single bar works well for titles or labels.

## Best Practices & Gotchas
* Match the skeleton’s size to the final content to prevent jank.
* Combine `animate-pulse` or `animate-[custom]` for subtle motion.
* Remove skeletons immediately after data resolves to avoid flicker.
* Keep color contrast low; skeletons should not draw user attention.
* Avoid complex shapes—simplicity aids performance.
