# SortableMultiSelectInput

## Overview
_None_

## Styling Guidelines
* All examples use Tailwind utility classes.  
* No Contember theme-specific classes are required.

## Core Concepts
* Manage a has-many relation through a multi-select UI.
* Drag items to reorder them (`sortableBy`).
* Fetch available options via `queryField` or local `options`.
* Optionally create new related entities with `createNewForm`.
* Persist initial order with `initialSorting`.

## Quick Start
```tsx
import { SortableMultiSelectInput } from '@contember/react-controls'

<FormHasManyRelationScope field="articles">
	<SortableMultiSelectInput
		field="articles"
		placeholder="Select articles"
		className="w-full max-w-md"
	/>
</FormHasManyRelationScope>
```

## Best Practices & Gotchas
* Always pass `sortableBy` to persist ordering in the database.
* Provide `options` only for small, static lists; otherwise use `queryField`.
* Wrap the input in `FormHasManyRelationScope` to avoid unexpected query nesting.
* Keep `placeholder` short; long text harms accessibility.
