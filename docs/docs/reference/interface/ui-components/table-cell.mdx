## Overview
TableCell renders a single `<td>` element inside Contember’s composable `Table` component family.  
You use it to display a field, input, or any custom content inside data-driven tables.

## Styling Guidelines
* All examples use Tailwind CSS.
* TableCell adds no styles on its own.  
  Apply Tailwind utility classes or rely on parent `Table` theme classes such as `table-auto` or `w-full` as needed.

## Core Concepts
* TableCell only works inside `TableRow`.
* You pass arbitrary React nodes as children—usually `Field`, `FormattedField`, or an input component.
* Formatting or ACL logic belongs to your child components, not to TableCell itself.
* Styling is purely presentational; no data logic happens here.

## Quick Start
```tsx
import { Table, TableBody, TableCell, TableHeader, TableHead, TableRow } from '~/lib/ui/table'

export const MinimalTable = () => (
	<Table className="table-auto w-full">
		<TableHeader>
			<TableRow>
				<TableHead className="text-left">Title</TableHead>
				<TableHead className="text-left">Category</TableHead>
			</TableRow>
		</TableHeader>

		<TableBody>
			<TableRow>
				<TableCell className="py-2">Example title</TableCell>
				<TableCell className="py-2">Example category</TableCell>
			</TableRow>
		</TableBody>
	</Table>
)
```

## Examples

### Display formatted fields
```tsx
<TableCell>
	<FormattedField field="ageFrom" />
</TableCell>
```
`FormattedField` handles number formatting; TableCell only provides layout.

### Boolean with custom formatter
```tsx
<TableCell>
	<Field field="canRead" format={formatBoolean} />
</TableCell>
```
`formatBoolean` turns database booleans into ✅ / ❌ strings for the UI.

### Editable value with ACL awareness
```tsx
<TableCell>
	<AclAwareInputField field="primaryValue" />
</TableCell>
```
The input respects Contember access control. TableCell remains presentation-only.

## Best Practices & Gotchas
* Keep business logic out of TableCell; place it in child components.
* Use Tailwind spacing utilities (`py-2`, `px-4`) to control cell padding.
* Wrap long text with `break-all` or `truncate` in the cell className.
* Remember to internationalize content inside your `Field` or `FormattedField` components.
* TableCell does not forward refs; manage focus inside child inputs directly.
