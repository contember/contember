# TableHead

## Overview
TableHead renders a single header cell inside a Contember `Table`.  
Use it within `TableHeader > TableRow` to create accessible and consistently styled table headings.

## Styling Guidelines
* All code examples use Tailwind classes.  
* TableHead picks up Contember’s default table theme—no extra classes are required unless you override them.

## Core Concepts
* Compose tables with `Table`, `TableHeader`, `TableRow`, and `TableHead`.
* Place one `TableHead` for each column.
* Wrap dynamic labels (e.g., `entityLabels.*`) directly inside the component.

## Quick Start
```tsx
import { Table, TableHeader, TableRow, TableHead } from '~/lib/ui/table'

export const MinimalTable = () => (
	<Table className="w-full text-left">
		<TableHeader>
			<TableRow>
				<TableHead className="py-2">Name</TableHead>
				<TableHead className="py-2">Status</TableHead>
			</TableRow>
		</TableHeader>
	</Table>
)
```

## Examples

### Dynamic labels from a localisation file
```tsx
import { entityLabels } from '~/locales'
import { Table, TableHeader, TableRow, TableHead } from '~/lib/ui/table'

<TableHeader>
	<TableRow>
		<TableHead>{entityLabels.HealthRiskPeriod.fields.ageFrom}</TableHead>
		<TableHead>{entityLabels.HealthRiskPeriod.fields.ageTo}</TableHead>
		<TableHead>{entityLabels.HealthRiskPeriod.fields.intervalMonths}</TableHead>
	</TableRow>
</TableHeader>
```
This pattern keeps heading text in a single localisation source and avoids hard-coding strings.

### Combining ACL-aware inputs in body rows
```tsx
<Table>
	<TableHeader>
		<TableRow>
			<TableHead>canRead</TableHead>
			<TableHead>canEdit</TableHead>
			<TableHead>Input</TableHead>
		</TableRow>
	</TableHeader>

	{/* TableBody omitted for brevity */}
</Table>
```
The header communicates permissions for each column. The body can then render `AclAwareInputField` components under the matching headings.

## Best Practices & Gotchas
* Keep heading text concise; long labels wrap awkwardly on small screens.
* Align numeric columns right by adding `className="text-right"` to each `TableHead`.
* Always match the number of `TableHead` components with `TableCell` components in body rows.
* Avoid interactive elements (buttons, inputs) inside `TableHead`; reserve it for labels only.
