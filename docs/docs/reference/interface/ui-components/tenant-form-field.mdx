# TenantFormField

## Overview
`TenantFormField` renders a single HTML `<input>` bound to a Contember tenant form context.  
It forwards all standard `<input>` attributes while adding Contember–specific binding via the `form` and `field` props.

## Styling Guidelines
* All snippets use Tailwind CSS utility classes.  
* No special Contember theme classes are required—style it as any standard input.

## Core Concepts
* Contember tenant form context supplies current values and validation state.
* `field` selects the key inside `form.values` that this input edits.
* `messages` defines human-readable validation error texts.
* The component accepts every regular `<input>` attribute, so you can reuse familiar HTML semantics.

## Quick Start
```tsx
import { TenantFormField } from '@contember/react-client-tenant'

export function EmailField({ form }: { form: any }) {
	return (
		<TenantFormField
			form={form}
			field="email"
			type="email"
			messages={{ required: 'Email is required' }}
			placeholder="you@example.com"
			className="block w-full rounded border-gray-300 p-2"
		/>
	)
}
```

## Props Reference
| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| form | CtxValue | Yes | — | Form context returned from Contember tenant form hooks. |
| field | keyof CtxValue["values"] & string | Yes | — | Name of the value inside the form state. |
| type | React.HTMLInputTypeAttribute | Yes | — | Standard HTML `input` type. |
| messages | Record\<string, string\> | Yes | — | Map of validation error codes to user messages. |
| …HTML `<input>` attributes | Same as `React.InputHTMLAttributes` | No | — | All other standard input props are forwarded. |

## Examples

### File upload field
```tsx
<TenantFormField
	form={form}
	field="avatar"
	type="file"
	accept="image/*"
	messages={{ required: 'Avatar is required', filetype: 'Only images are allowed' }}
	className="block w-full cursor-pointer rounded border p-2"
/>
```
The `accept` attribute restricts selection to images. Validation messages rely on keys coming from your validator.

### Number stepper
```tsx
<TenantFormField
	form={form}
	field="quantity"
	type="number"
	min={1}
	max={10}
	step={1}
	messages={{ range: 'Choose between 1 and 10' }}
	className="w-24 rounded border text-center"
/>
```
`min`, `max`, and `step` are plain HTML props, so the browser enforces basic constraints.

## Best Practices & Gotchas
* Always pass the same `form` instance to every `TenantFormField` to keep context consistent.
* Prefer descriptive `messages` keys that match your validation rules.
* Combine native attributes (`required`, `minLength`, etc.) with server-side validation for better UX.
* Remember to set `type="password"` to hide sensitive input in browsers.
