## Overview
`TextareaField` renders a multi-line text input that integrates with Contember form scopes.  
You use it inside `FormContainer`, `FormFieldScope`, or any compatible form context.

It provides labels, descriptions, required validation, auto-resize via `TextareaAutosize`, and forwards additional DOM props through `inputProps`.

## Styling Guidelines
* All code samples use Tailwind utility classes.  
* `TextareaField` itself adds no visual styles; rely on your Tailwind setup or Contember theme classes like `con-input`, `con-label`, and `con-error`.

## Core Concepts
* Form context is mandatory—wrap the field in `FormContainer` or equivalent.
* The `field` prop binds the component to an entity attribute.
* Accessibility comes from `label` and `description`.
* Auto-resize works out-of-the-box; override with `inputProps={{ minRows: n }}`.
* Validation respects the `required` prop and server errors.

## Quick Start
```tsx
import { FormContainer, TextareaField } from '~/lib/form'

export default () => (
  <FormContainer>
    <TextareaField
      field="bio"
      label="Biography"
      description="Tell us about yourself"
      required
      inputProps={{ placeholder: 'Write something...', className: 'con-input w-full min-h-[120px]' }}
    />
  </FormContainer>
)
```

## Examples
### Auto-resized comment box
```tsx
<TextareaField
  field="comment"
  label="Comment"
  inputProps={{ minRows: 2, placeholder: 'Leave your feedback...', className: 'w-full con-input' }}
/>
```
This creates a comment box that grows with content thanks to `TextareaAutosize`.

### Edit-in-place pattern
```tsx
<ClickToEdit
  view={entity => entity.getField('bio').value}
  edit={<TextareaField field="bio" label={undefined} />}
/>
```
`ClickToEdit` toggles between read and edit modes. Passing `label={undefined}` hides the label in compact UIs.

## Best Practices & Gotchas
* Always provide `label` for accessibility unless the design omits it intentionally.  
* Combine `minRows` and `maxRows` in `inputProps` to control resize limits.  
* Wrap the field in `FormLocales` to localize rich text.  
* Server validation errors flow automatically; you rarely need custom error handling.  
* Avoid heavy CSS on `textarea`; let Tailwind or Contember theme styles handle appearance.
