# ToastContent

## Overview
ToastContent wraps any React nodes you want to show inside a toast notification.
Use it with `useShowToast` and `Toaster` to display success, error, or info messages across your app.

## Styling Guidelines
* All examples use Tailwind CSS classes.
* ToastContent renders plain HTML. You do not need Contember-specific theme classes.

## Core Concepts
* ToastContent is presentational; state lives in `useShowToast`.
* Children define the body of the toast.
* Optional `title` adds a bold heading.
* Display styles come from the surrounding Toaster implementation.

## Quick Start
```tsx
import { ToastContent, Toaster, useShowToast } from '@app/lib/toast'

export default function App() {
  const showToast = useShowToast()

  return (
    <>
      <button
        className="px-3 py-2 bg-blue-600 text-white rounded"
        onClick={() =>
          showToast(<ToastContent>Password changed</ToastContent>, { type: 'success' })
        }
      >
        Change password
      </button>

      {/* Mount once, usually at the root */}
      <Toaster />
    </>
  )
}
```

## Sub-Components / Building Blocks
### useShowToast
Hook that returns a `showToast` function. Pass `<ToastContent />` and options to display notifications.

## Examples
### Show a simple success toast
```tsx
showToast(<ToastContent>Password changed</ToastContent>, { type: 'success' })
```
This confirms a successful operation.

### Include dynamic data from a mutation result
```tsx
args => {
  showToast(
    <ToastContent>Invitation sent to {args.result.person?.email}</ToastContent>,
    { type: 'success' }
  )
  memberListController.current?.refresh()
}
```
You pass variables straight into JSX to personalize the message.

### Add a title and custom JSX
```tsx
showToast(
  <ToastContent title="API key created">
    <Input value={apiKeyToken} readOnly className="w-full border p-2" />
  </ToastContent>,
  { type: 'success' }
)
```
Titles improve scanability. Embed form elements for copy-to-clipboard workflows.

## Best Practices & Gotchas
* Mount one `<Toaster />` at the app root.
* Keep toast messages short; aim for two lines max.
* Pass stable keys to dynamic lists inside ToastContent.
* Avoid long-running JSX logic inside the toast callback. Compute data beforehand.
* Do not style ToastContent directly; style the surrounding toast container if needed.
