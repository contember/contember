# ToastDescription

## Overview
`ToastDescription` renders the body text of a toast notification.  
Place it under a title element to give users extra context.

## Styling Guidelines
* All examples use Tailwind CSS classes.  
* No Contember-specific theme classes are required.

## Core Concepts
* Wrap descriptive text in `ToastDescription` for consistent spacing and typography.  
* Combine it with a title element for clear hierarchy.  
* Keep descriptions short; one or two brief sentences work best.

## Quick Start
```tsx
import { ToastDescription } from "@contember/ui"

export const SimpleToast = () => (
	<div className="flex flex-col gap-1 rounded bg-gray-800 p-4 text-white shadow-lg">
		<strong className="text-sm font-semibold">Upload complete</strong>
		<ToastDescription>
			Your files have been uploaded successfully.
		</ToastDescription>
	</div>
)
```

## Examples

### Success toast with icon
```tsx
import { CheckCircleIcon } from "@heroicons/react/24/solid"
import { ToastDescription } from "@contember/ui"

export const SuccessToast = () => (
	<div className="flex items-start gap-3 rounded bg-green-600 p-4 text-white shadow-md">
		<CheckCircleIcon className="h-5 w-5 flex-none" />
		<div className="flex flex-col">
			<strong className="text-sm font-medium">Saved</strong>
			<ToastDescription>
				Profile information updated.
			</ToastDescription>
		</div>
	</div>
)
```
This toast pairs an icon with descriptive text to reinforce the positive outcome.

### Error toast with dismiss button
```tsx
import { XMarkIcon } from "@heroicons/react/24/solid"
import { ToastDescription } from "@contember/ui"

export const ErrorToast = () => (
	<div className="relative flex flex-col gap-1 rounded bg-red-700 p-4 text-white shadow-md">
		<button className="absolute right-2 top-2 rounded p-1 hover:bg-red-600">
			<XMarkIcon className="h-4 w-4" />
		</button>

		<strong className="text-sm font-semibold">Upload failed</strong>
		<ToastDescription>
			We could not reach the server. Try again later.
		</ToastDescription>
	</div>
)
```
The dismiss button lets users close the message manually.

## Best Practices & Gotchas
* Keep description concise; long text overwhelms the toast.  
* Use a readable color contrast against the toast background.  
* Avoid interactive elements inside `ToastDescription`. Put buttons outside.  
* Use ARIA live regions on the parent toast container for accessibility.
