## Overview
`Todo` marks unfinished UI work during development.
It renders your message in a highlighted banner only when `import.meta.env.DEV` is `true`.

Use it to keep TODO notes visible without shipping them to production.

## Styling Guidelines
* All examples use Tailwind classes.
* `Todo` does not require Contember theme classes, but you can add any Tailwind utilities.

## Core Concepts
* Development-only rendering controlled by `import.meta.env.DEV`.
* Simple children-only API.
* Visual highlighting through Tailwind or your own classes.

## Quick Start
```tsx
import { Todo } from '@app/lib/dev'

export const Dashboard = () => (
	<Todo className="mb-4 rounded bg-yellow-100 p-2 text-yellow-800">
		Implement authentication logic
	</Todo>
)
```

## Detailed Guide
When `import.meta.env.DEV` is `false`, `Todo` returns `null` and renders nothing.
This keeps production bundles clean.

```tsx
import { Todo } from '@app/lib/dev'

export const ProfilePage = () => (
	<>
		<h1 className="text-2xl font-bold">Profile</h1>

		<Todo className="mt-2 rounded bg-red-100 p-3 text-red-800">
			Load user data from API
		</Todo>
	</>
)
```

## Best Practices & Gotchas
* Wrap only short, actionable messages.
* Remove or resolve each TODO before release.
* Style with Tailwind to match your design system.
* Do not rely on `Todo` for runtime feature flags; use proper environment checks instead.
