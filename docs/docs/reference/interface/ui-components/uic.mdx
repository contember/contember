## Overview
`uic` wraps any React element or component and injects a utility-class variance system.  
You configure variants once and receive a fully-typed component that handles default variants, data attributes, and composition helpers.

## Styling Guidelines
All examples use Tailwind. The `uic` utility itself does not require Contember theme classes.

## Core Concepts
- Variant definition through a `cva`-style `variants` object.  
- `defaultVariants` apply when you omit a variant prop.  
- Optional mapping of variant values to `data-*` attributes for easier selectors.  
- `passVariantProps` controls which variant props remain on the DOM element.  
- Composition helpers:  
  - `asChild` forwards styling to another component.  
  - `wrapInner` and `wrapOuter` let you nest extra markup.

## Quick Start
```tsx
import { uic } from '~/lib/utils'

const Button = uic('button', {
  baseClass: 'px-4 py-2 font-bold',
  variants: {
    intent: {
      primary: 'bg-blue-500 text-white',
      secondary: 'bg-gray-500 text-white',
    },
  },
  defaultVariants: {
    intent: 'primary',
  },
  displayName: 'Button',
})

export default function App() {
  return <Button intent="secondary">Click me</Button>
}
```

```tsx
const Button = uic('button', {
  baseClass: 'px-4 py-2 font-bold',
  variants: {

      primary: 'bg-blue-500 text-white',

    intent: 'primary',
  },
  displayName: 'Button',
})

<Button intent="secondary">Click me</Button>
```
This example shows how a single `intent` variant toggles Tailwind classes.  
Because `defaultVariants.intent` is `"primary"`, omitting the prop yields a blue button; passing `"secondary"` turns it gray.

## Best Practices & Gotchas
- Keep `baseClass` minimal; place variant-specific styles inside `variants`.  
- Always set `displayName` for clearer React DevTools output.  
- Use `passVariantProps: false` to avoid leaking unknown props to the DOM.  
- Combine `data-*` attributes with Tailwindâ€™s arbitrary selectors for complex UIs.
