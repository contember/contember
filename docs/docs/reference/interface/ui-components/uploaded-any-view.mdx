## Styling Guidelines
* All code samples use Tailwind utility classes.
* The component itself adds no Contember-specific theme classes, so you may style freely.

## Core Concepts
* Bind the file record’s URL via `urlField` – this prop is mandatory.
* Optional field props (`fileNameField`, `lastModifiedField`, `fileSizeField`, `fileTypeField`) let you display extra metadata.
* Pass `baseField` when the file entity lives on a related record.
* Render custom UI (e.g. buttons) through `actions` and `edit`.
* Disable deletion with `noDestroy` or supply a custom `DestroyAction` component.

## Quick Start
```tsx
import { UploadedAnyView } from '@contember/react-components/upload'

export const FilePreview = () => (
	<div className="p-4 border rounded-md">
		<UploadedAnyView
			urlField="url"
			className="max-w-full"
		/>
	</div>
)
```

## Best Practices & Gotchas
* Always supply `urlField`; nothing renders without it.
* Prefer relation objects over plain strings when files live outside the current entity.
* Wrap large previews in a container with `overflow-auto` to avoid layout shifts.
* Use `noDestroy` in read-only contexts to avoid accidental deletions.
* Provide a custom `DestroyAction` when you need permission checks or confirmation dialogs.
