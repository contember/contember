# UploadedAudioView

## Overview
UploadedAudioView renders a single audio file stored in Contember.
You can preview the track, display metadata, and plug in custom edit or destroy actions.

## Styling Guidelines
* All examples use Tailwind CSS.
* The component adds no classes. Wrap it with your own Tailwind containers or Contember theme utilities as needed.

## Core Concepts
* Audio metadata lives in Contember entity fields.
* Map entity fields to props like `urlField="url"`.
* Use `actions`, `edit`, or `DestroyAction` to extend the default UI.
* Toggle the built-in destroy button with `noDestroy`.

## Quick Start
```tsx
import { UploadedAudioView } from '@contember/react-components'

export const SongPreview = () => (
  <div className="p-4 bg-white rounded shadow">
    <UploadedAudioView urlField="url" />
  </div>
)
```

## Examples

### 1. Custom Actions
Add a download button while keeping the default destroy action.

```tsx
<UploadedAudioView
  urlField="url"
  actions={
    <a
      href={record.url}
      download
      className="text-blue-600 hover:underline"
    >
      Download
    </a>
  }
/>
```

The link appears next to the built-in controls. Tailwind styles keep visual consistency.

### 2. Replace Destroy Action
Wrap the destroy trigger with your own component.

```tsx
const DangerButton = ({ children }: { children: React.ReactElement }) => (
  <span className="text-red-600 hover:text-red-800">{children}</span>
)

<UploadedAudioView
  urlField="url"
  DestroyAction={DangerButton}
/>
```

You changed only the visuals; Contember still handles deletion logic.

## Best Practices & Gotchas
* Always pass `urlField`; the component throws if missing.
* Wrap the player in a fixed-width container to prevent stretching.
* Use `noDestroy` when the user lacks delete permission.
* Keep custom actions small; they share horizontal space.
