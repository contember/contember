# UploadedVideoView

## Overview
`UploadedVideoView` renders a preview of a video already stored in Contember Storage.  
You point the component to entity fields such as the file URL, file name, or dimensions and it outputs a ready-to-use video player plus optional metadata and action buttons.

## Styling Guidelines
* All snippets use Tailwind CSS.  
* The component itself ships **unstyled**. Add your own Tailwind classes on wrappers or via `actions` / `edit` render props.  
* No Contember theme classes are required.

## Core Concepts
* Map entity fields to props (e.g. `urlField`, `fileNameField`).  
* Combine with form helpers like `FileRepeaterFieldInner` for multiple uploads.  
* Supply custom UI through `actions`, `edit`, or a custom `DestroyAction`.  
* Use `noDestroy` to hide the default “delete” action when provided by the parent.

## Quick Start
```tsx
import { UploadedVideoView } from '@contember/react-admin'

export const VideoPreview = () => (
  <div className="max-w-md rounded border p-4">
    <UploadedVideoView
      urlField="url"
      fileNameField="fileName"
      widthField="width"
      heightField="height"
    />
  </div>
)
```

## Sub-Components / Building Blocks
### DestroyAction
Optional override that wraps the default “delete” icon or button. Provide a component that receives the preview element as `children` and handles the removal logic.

## Examples

### 1. File Repeater with Remove Trigger
```tsx
import {
  FileRepeaterFieldInner,
  UploadedVideoView,
  RepeaterRemoveItemTrigger,
} from '@contember/react-admin'

export const VideoRepeaterField = props => (
  <FileRepeaterFieldInner {...props} fileType="video/*">
    {props.children ?? (
      <UploadedVideoView
        {...props}
        DestroyAction={RepeaterRemoveItemTrigger}
        urlField="url"
        fileNameField="fileName"
      />
    )}
  </FileRepeaterFieldInner>
)
```
This pattern creates a repeater for many videos and uses `RepeaterRemoveItemTrigger` to remove an item from the list.

### 2. Upload Field with Disconnect Trigger
```tsx
import {
  UploadFieldInner,
  UploadedVideoView,
  DisconnectEntityTrigger,
} from '@contember/react-admin'

export const SingleVideoField = props => (
  <UploadFieldInner {...props} fileType="video/*">
    {props.children ?? (
      <UploadedVideoView
        {...props}
        DestroyAction={DisconnectEntityTrigger}
        urlField="url"
        fileNameField="fileName"
      />
    )}
  </UploadFieldInner>
)
```
`DisconnectEntityTrigger` detaches the file entity rather than deleting it. Use this when you only want to break the relation.

### 3. Custom Item UI in Upload Field
```tsx
import {
  UploadFieldInner,
  UploaderItemUI,
  UploadedVideoView,
  DisconnectEntityTrigger,
} from '@contember/react-admin'

export const CustomItemVideoField = props => (
  <UploadFieldInner {...props} fileType="video/*">
    <UploaderItemUI className="border rounded p-4 flex gap-4">
      {props.children ?? (
        <UploadedVideoView
          {...props}
          DestroyAction={DisconnectEntityTrigger}
          urlField="url"
          fileNameField="fileName"
        />
      )}
    </UploaderItemUI>
  </UploadFieldInner>

```
`UploaderItemUI` lets you inject your own layout around the preview while still leveraging built-in upload logic.

### 4. Video Repeater with Drag & Drop
```tsx
import {
  UploaderDropzone,
  UploaderRepeaterItemsWrapperUI,
  UploaderRepeaterItemUI,
  UploadedVideoView,
  UploaderRepeaterHandleUI,
  UploaderRepeaterDragOverlayUI,
} from '@contember/react-admin'

export const VideoRepeater = () => (
  <UploaderDropzone
    fileType="video/*"
    className="border-dashed border-2 rounded p-6 text-center"
  >
    <UploaderRepeaterItemsWrapperUI className="space-y-4">
      <UploaderRepeaterItemUI className="relative">
        <UploaderRepeaterHandleUI className="absolute -left-6 top-1/2 -translate-y-1/2 cursor-move" />
        <UploadedVideoView
          urlField="url"
          fileNameField="fileName"
          DestroyAction={UploaderRepeaterDragOverlayUI}
        />
      </UploaderRepeaterItemUI>
    </UploaderRepeaterItemsWrapperUI>
  </UploaderDropzone>
)
```
The snippet shows how to:
* Allow multi-file uploads.  
* Reorder items with drag handles.  
* Use a custom `DestroyAction` that integrates with the Repeater overlay.

## Best Practices & Gotchas
* Always supply `urlField`; without it the component renders nothing.  
* Wrap the view in a container that sets width; the video element uses 100 % width by default.  
* When you pass relation objects instead of strings, keep the `field` property short and precise.  
* Use `noDestroy` if the parent already provides a delete UI to avoid duplicated buttons.  
* Restrict uploads by MIME type on the server; `UploadedVideoView` only displays the file.  
* Prefer `DisconnectEntityTrigger` over deletion when you must keep the file entity around.
