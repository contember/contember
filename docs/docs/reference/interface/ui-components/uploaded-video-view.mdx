# UploadedVideoView

## Overview
`UploadedVideoView` renders a preview of a video already stored in Contember Storage.
You point the component to entity fields such as the file URL, file name, or dimensions and it outputs a ready-to-use video player plus optional metadata and action buttons.

## Styling Guidelines
* All snippets use Tailwind CSS.
* The component itself ships **unstyled**. Add your own Tailwind classes on wrappers or via `actions` / `edit` render props.
* No Contember theme classes are required.

## Core Concepts
* Map entity fields to props (e.g. `urlField`, `fileNameField`).
* Combine with form helpers like `FileRepeaterFieldInner` for multiple uploads.
* Supply custom UI through `actions`, `edit`, or a custom `DestroyAction`.
* Use `noDestroy` to hide the default “delete” action when provided by the parent.

## Quick Start
```tsx
import { UploadedVideoView } from '@contember/react-admin'

export const VideoPreview = () => (
  <div className="max-w-md rounded border p-4">
    <UploadedVideoView
      urlField="url"
      fileNameField="fileName"
      widthField="width"
      heightField="height"
    />
  </div>
)
```

## Sub-Components / Building Blocks
### DestroyAction
Optional override that wraps the default “delete” icon or button. Provide a component that receives the preview element as `children` and handles the removal logic.

## Best Practices & Gotchas
* Always supply `urlField`; without it the component renders nothing.
* Wrap the view in a container that sets width; the video element uses 100 % width by default.
* When you pass relation objects instead of strings, keep the `field` property short and precise.
* Use `noDestroy` if the parent already provides a delete UI to avoid duplicated buttons.
* Restrict uploads by MIME type on the server; `UploadedVideoView` only displays the file.
* Prefer `DisconnectEntityTrigger` over deletion when you must keep the file entity around.
