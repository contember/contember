# UploaderDropzoneWrapperUI

## Overview
UploaderDropzoneWrapperUI groups your uploader’s interactive area and any fallback UI into a single, styled wrapper. It switches between an active dropzone and an inactive placeholder based on your own logic, letting you keep the layout consistent across states.

Use it with UploaderDropzoneArea, UploaderDropzoneAreaUI, and UploaderInactiveDropzoneUI to build a fully featured drag-and-drop experience.

## Styling Guidelines
* All snippets below use Tailwind CSS classes.
* No special Contember theme classes are required; feel free to apply your own design tokens.

## Core Concepts
* Wrapper only – it adds no logic; you control which child gets rendered.
* Render prop pattern – place UploaderDropzoneArea or UploaderInactiveDropzoneUI as children.
* Composability – nest it inside UploaderDropzoneRoot to gain full uploader context.

## Quick Start
```tsx
import {
	UploaderDropzoneRoot,
	UploaderDropzoneWrapperUI,
	UploaderDropzoneArea,
	UploaderDropzoneAreaUI,
	UploaderInactiveDropzoneUI,
} from '@contember/react-uploader-ui'

export const AvatarUploader = () => (
	<UploaderDropzoneRoot>
		<UploaderDropzoneWrapperUI>
			<UploaderDropzoneArea>
				<UploaderDropzoneAreaUI className="flex flex-col items-center gap-2 p-6 border-2 border-dashed rounded-md">
					<span className="text-sm font-semibold">Drag & drop an image</span>
				</UploaderDropzoneAreaUI>
			</UploaderDropzoneArea>
		</UploaderDropzoneWrapperUI>
	</UploaderDropzoneRoot>
)
```

## Examples

### Switching to inactive UI while uploading
```tsx
({ inactiveOnUpload }: { inactiveOnUpload?: boolean }) => {
	const filesInProgress = useUploaderStateFiles({ state: ['uploading', 'initial', 'finalizing'] })
	const showLoader = inactiveOnUpload && filesInProgress.length > 0

	return (
		<UploaderDropzoneRoot>
			<UploaderDropzoneWrapperUI>
				{showLoader ? (
					<UploaderInactiveDropzoneUI />
				) : (
					<UploaderDropzoneArea>
						<UploaderDropzoneAreaUI className="flex flex-col items-center gap-2 p-6 border-2 border-dashed rounded-md">
							<span className="text-sm font-semibold">Drop files here</span>
							<span className="text-xs">or</span>
							<button className="px-3 py-1 text-xs border rounded">Browse</button>
						</UploaderDropzoneAreaUI>
					</UploaderDropzoneArea>
				)}
			</UploaderDropzoneWrapperUI>
		</UploaderDropzoneRoot>
	)
}
```
The component stays agnostic of upload state. You decide when to render UploaderInactiveDropzoneUI.

## Best Practices & Gotchas
* Always wrap the component inside UploaderDropzoneRoot to get proper context.
* Place only one child node (active or inactive) inside UploaderDropzoneWrapperUI.
* Handle uploader state externally; the wrapper does not monitor uploads.
* Keep the wrapper’s padding and border consistent to avoid layout shifts.
