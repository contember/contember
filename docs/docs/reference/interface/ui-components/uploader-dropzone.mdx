## Overview
UploaderDropzone renders an area where users can drag-and-drop files.  
It usually lives inside `MultiUploader` or similar upload wrappers and reacts to upload state.

## Styling Guidelines
* All examples in this documentation use Tailwind CSS utility classes.  
* No Contember-specific theme classes are required for this component.

## Core Concepts
* UploaderDropzone shows a visual target for dropping files.  
* It works best inside `MultiUploader`, which provides the actual upload logic.  
* `inactiveOnUpload` hides or disables the dropzone while a file is being uploaded.  
* `dropzonePlaceholder` lets you replace the default text with any React node.

## Quick Start
```tsx
import { UploaderDropzone } from '~/lib/upload'

export const AvatarUploader = () => (
	<div className="max-w-sm border border-dashed border-gray-300 p-4">
		<UploaderDropzone dropzonePlaceholder="Drop your avatar here" />
	</div>
)
```

## Examples

### Example&nbsp;1 – Conditional rendering with button placeholder
```tsx
entity => {
	entity = baseField ? entity.getEntity({ field: baseField }) : entity
	if (entity.getField('file.url').value === null) {
		return (
			<UploaderDropzone
				inactiveOnUpload
				dropzonePlaceholder={
					<Button variant="secondary" size="sm" className="w-full min-h-10">
						{dict.uploader.dropFiles}
					</Button>
				}
			/>
		)
	}
	return (
		<div className="w-full">
			{/* Alternative UI once file URL exists */}
		</div>
	)
}
```
You render the dropzone only when no file URL exists. The secondary button provides a consistent call-to-action.

### Example&nbsp;2 – Basic dropzone inside a complex form
```tsx
<FormFieldStateProvider>
	<FormContainer description={description} label={label}>
		<Repeater initialEntityCount={0}>
			<UploaderRepeaterItemsWrapperUI>
				<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
					<UploaderHasFile>
						<UploaderProgress />
					</UploaderHasFile>
					<UploaderDropzone dropzonePlaceholder={dropzonePlaceholder} />
				</MultiUploader>
				{/* …sortable repeater items… */}
			</UploaderRepeaterItemsWrapperUI>
		</Repeater>
	</FormContainer>
</FormFieldStateProvider>
```
The dropzone appears only when `MultiUploader` reports no file. It inherits sizing from its wrapper, so Tailwind borders from the parent remain visible.

### Example&nbsp;3 – Slim integration inside `MultiUploader`
```tsx
<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
	<UploaderHasFile>
		<UploaderProgress />
	</UploaderHasFile>

	{/* Visible element until a file exists */}
	<UploaderDropzone dropzonePlaceholder={dropzonePlaceholder} />
</MultiUploader>
```
Use this pattern when you need nothing more than a progress bar and a dropzone.

## Best Practices & Gotchas
* Place the dropzone inside `MultiUploader` for automatic state handling.  
* Always size the parent container; the dropzone stretches to 100%.  
* Use `inactiveOnUpload` to avoid user confusion during long uploads.  
* Replace placeholder text with a button or illustration to match your design.

## Further Reading
* `MultiUploader` – Orchestrates uploads and exposes state.  
* `UploaderHasFile` – Conditionally renders content once a file exists.  
* `UploaderProgress` – Shows real-time upload progress.
