## Overview
UploaderDropzone renders a drag-and-drop area for file uploads.
Use it inside an uploader context so it can react to upload state.

It can show custom placeholder content and auto-hide while files upload.

## Styling Guidelines
* All examples use Tailwind CSS utility classes.
* No Contember-specific theme classes are required.

## Core Concepts
* The dropzone only displays when its uploader context is active.
* `inactiveOnUpload` disables the zone during active uploads.
* `dropzonePlaceholder` replaces default placeholder content.
* You decide size through the parent container’s styles.

## Quick Start
```tsx
import { UploaderDropzone } from '~/lib/upload'

export const AvatarUploader = () => (
	<div className="max-w-sm border border-dashed border-gray-300 p-4">
		<UploaderDropzone dropzonePlaceholder="Drop your avatar here" />
	</div>
)
```

## Props Reference
Name | Type | Required | Default | Description
---- | ---- | -------- | ------- | -----------
dropzonePlaceholder | React.ReactNode | No | `undefined` | Custom content rendered inside the dropzone.
inactiveOnUpload | boolean | No | `false` | When **true**, the dropzone hides or disables during an active upload.

## Examples

### Example&nbsp;1 – Basic usage with custom placeholder
```tsx
<UploaderDropzone
	inactiveOnUpload
	dropzonePlaceholder={<div>Drop files here or click to upload</div>}
/>
```
The dropzone displays a simple text placeholder. It hides while uploads are active.

### Example&nbsp;2 – Conditional rendering with button placeholder
```tsx
entity => {
	entity = baseField ? entity.getEntity({ field: baseField }) : entity
	if (entity.getField('file.url').value === null) {
		return (
			<UploaderDropzone
				inactiveOnUpload
				dropzonePlaceholder={
					<Button variant="secondary" size="sm" className="w-full min-h-10">
						{dict.uploader.dropFiles}
					</Button>
				}
			/>
		)
	}
	return <div className="w-full">{/* Alternative UI once file URL exists */}</div>
}
```
You render the dropzone only when no file URL exists. A styled button invites users to add files.

### Example&nbsp;3 – Dropzone inside a complex form
```tsx
<FormFieldStateProvider>
	<FormContainer description={description} label={label}>
		<Repeater initialEntityCount={0}>
			<UploaderRepeaterItemsWrapperUI>
				<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
					<UploaderHasFile>
						<UploaderProgress />
					</UploaderHasFile>
					<UploaderDropzone dropzonePlaceholder={dropzonePlaceholder} />
				</MultiUploader>
				{/* …sortable repeater items… */}
			</UploaderRepeaterItemsWrapperUI>
		</Repeater>
	</FormContainer>
</FormFieldStateProvider>
```
The dropzone appears only when `MultiUploader` reports no files. The parent container provides borders and spacing.

### Example&nbsp;4 – Slim integration inside `MultiUploader`
```tsx
<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
	<UploaderHasFile>
		<UploaderProgress />
	</UploaderHasFile>

	{/* Visible element until a file exists */}
	<UploaderDropzone dropzonePlaceholder={dropzonePlaceholder} />
</MultiUploader>
```
Use this pattern when you need only a progress bar and a dropzone.

## Best Practices & Gotchas
* Place the dropzone inside `MultiUploader` for automatic state handling.
* Size the parent container; the dropzone stretches to 100 % width and height.
* Use `inactiveOnUpload` to prevent double uploads.
* Replace placeholder text with visuals that match your design system.

## Further Reading
* `MultiUploader` – Orchestrates uploads and exposes state.
* `UploaderHasFile` – Shows or hides children based on file presence.
* `UploaderProgress` – Displays real-time upload progress.
