# UploaderFileProgressUI

## Overview
UploaderFileProgressUI shows a single file’s upload status.  
You pass the `File` object, optional progress percentage, and any extra UI like buttons or info.

## Styling Guidelines
* All examples use TailwindCSS.  
* The component itself adds no styles—you supply classes via `actions` or `info` nodes.

## Core Concepts
* Track upload progress per file rather than globally.  
* Compose extra actions (abort, dismiss) as children via the `actions` prop.  
* Keep additional details (bytes, icons) in the `info` prop for total layout control.  
* Progress is a plain number `0–100`; calculation happens in your code.

## Quick Start
```tsx
import { UploaderFileProgressUI } from '@contember/react-uploader'

function Example({ file }: { file: File }) {
	return (
		<UploaderFileProgressUI
			file={file}
			progress={0}
			actions={<button className="text-red-600 text-xs">Abort</button>}
			info={<span className="text-gray-400 text-xs">{file.name}</span>}
		/>
	)
}
```

## Examples
### 1. Abortable Upload
```tsx
<UploaderFileProgressUI
	file={it.file.file}
	progress={(it.progress.progress ?? 0) * 100}
	actions={<AbortButton state={it} />}
	info={
		<div className="text-gray-400 text-xs">
			{formatBytes(it.progress.uploadedBytes)} / {formatBytes(it.progress.totalBytes)}
		</div>
	}
/>
```
The Abort button comes from `@contember/react-uploader` utilities. You keep UX consistent while letting users cancel stalled uploads.

### 2. Success Toast
```tsx
<UploaderFileProgressUI
	file={it.file.file}
	progress={100}
	actions={<DismissButton state={it} />}
	info={
		<div className="flex gap-1 items-center text-green-600">
			<CheckIcon className="w-3 h-3" />
			<span className="text-xs">Done ({formatBytes(it.file.file.size)})</span>
		</div>
	}
/>
```
Combine with a toast to notify users when the upload finishes.

### 3. Finalizing Spinner
```tsx
<UploaderFileProgressUI
	file={it.file.file}
	progress={100}
	actions={<AbortButton state={it} />}
	info={
		<div className="text-gray-400 text-xs inline-flex gap-1 items-center">
			<Loader2Icon className="animate-spin h-3 w-3" />
			{formatBytes(it.file.file.size)}
		</div>
	}
/>
```
Show a spinner while the backend performs post-upload processing.

## Best Practices & Gotchas
* Always supply the raw `File` so the component can render file name and type.  
* Compute progress (`0–100`) yourself; the component stays presentation-only.  
* Debounce progress updates for smoother UI.  
* Keep `actions` compact; crowded layouts break on small screens.  
* Remember to unmount the component after success to free memory.
