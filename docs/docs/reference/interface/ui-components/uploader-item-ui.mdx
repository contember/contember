## UploaderItemUI

### Overview
`UploaderItemUI` wraps one logical upload slot and decides what to render inside.
It commonly toggles between a `UploaderDropzone` and a preview or input tied to the uploaded file.

### Styling Guidelines
* All examples use Tailwind CSS utility classes.
* `UploaderItemUI` injects no Contember-specific theme classes, so you style it freely.

### Core Concepts
* Encapsulates a single file’s UI state.
* Renders only its children—no own visuals.
* Works inside `Uploader`, `UploadFieldInner`, or custom wrappers.
* You decide when to show a drop-zone, preview, or even a text input.

### Quick Start
```tsx
import { Uploader, UploaderItemUI, UploaderDropzone, UploadedImageView } from '~/lib/upload'

export const ImageUploadField = () => (
  <Uploader>
    <UploaderItemUI>
      <UploaderDropzone className="w-full h-32 flex items-center justify-center border-2 border-dashed rounded text-gray-500" />
      <UploadedImageView className="mt-4" />
    </UploaderItemUI>
  </Uploader>
)
```

### Examples
#### 1. Custom image field with destroy action
```tsx
props => (
  <UploadFieldInner {...props} fileType={createImageFileType(props)}>
    <UploaderItemUI>
      {props.children ?? (
        <UploadedImageView
          {...props}
          DestroyAction={props.noDestroy ? undefined : DisconnectEntityTrigger}
        />
      )}
    </UploaderItemUI>
  </UploadFieldInner>
)
```
This wrapper chooses between a user-supplied child or the default preview.
`DestroyAction` lets users remove the file.

#### 2. Conditional drop-zone vs. preview
```tsx
<FormContainer label="Profile picture">
  <Uploader baseField="image">
    <UploaderItemUI>
      <EntityView
        render={entity =>
          entity.getField('url').value === null
            ? (
              <UploaderDropzone
                inactiveOnUpload
                dropzonePlaceholder="Drag image…"
              />
            )
            : <UploadedImageView />
        }
      />
    </UploaderItemUI>
  </Uploader>
</FormContainer>
```
`EntityView` inspects the `url` field and decides which child to render.

#### 3. Corporate-style file upload with inline URL editing
```tsx
<UploaderItemUI className="bg-transparent border-0 shadow-none [&_.rounded]:bg-transparent [&_.rounded]:shadow-none [&_.rounded]:border-0 [&_.h-40]:h-10 [&_.h-40]:w-10">
  <EntityView
    render={entity => {
      entity = baseField ? entity.getEntity({ field: baseField }) : entity
      if (entity.getField('file.url').value === null) {
        return (
          <UploaderDropzone
            inactiveOnUpload
            dropzonePlaceholder={
              <Button variant="secondary" size="sm" className="w-full min-h-10">
                {dict.uploader.dropFiles}
              </Button>
            }
          />
        )
      }
      return (
        <div className="w-full">
          {readonly ? (
            <Field field={`${baseField}.file.url`} />
          ) : (
            <InputField
              field={`${baseField}.file.url`}
              label={
                <div className="flex items-center gap-2">
                  {entityLabels.File.fields.url}
                  <HasOne field={`${baseField}.file`}>
                    <DisconnectEntityTrigger>
                      <Button variant="link" size="sm" className="text-red-500">
                        <FileXIcon size={18} />
                      </Button>
                    </DisconnectEntityTrigger>
                  </HasOne>
                </div>
              }
            />
          )}
        </div>
      )
    }}
  />
</UploaderItemUI>
```
The company’s design system demands a transparent background and zero shadow.
Utility selectors like `&[_.rounded]` reset nested component styling.
A `DisconnectEntityTrigger` provides a red “delete” button next to the URL input.

### Best Practices & Gotchas
* Place exactly one logical file inside each `UploaderItemUI`.
* Always render a placeholder (drop-zone or preview) to avoid empty UI.
* Use stateful helpers (`EntityView`, `UploaderHasFile`) to switch visuals.
* Keep layout consistent by wrapping `UploaderItemUI` in a flex or grid parent.
* Do not forward refs through `UploaderItemUI`—forward through its children instead.
