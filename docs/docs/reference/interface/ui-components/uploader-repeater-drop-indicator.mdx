# UploaderRepeaterDropIndicator

## Overview
UploaderRepeaterDropIndicator displays a visual cue showing where users can drop an item inside a repeater.  
Pass the `position` prop to decide whether the cue appears before or after the target item.

## Styling Guidelines
All examples use Tailwind utility classes.  
No additional Contember theme classes are required.

## Core Concepts
- The component renders only the drop-zone indicator, not the draggable logic itself.  
- You must set `position` to either `"before"` or `"after"`.  
- Combine this component with your drag-and-drop handlers or a library such as `@dnd-kit/core`.  
- Render the indicator conditionally during drag operations to reduce visual noise.

## Quick Start
```tsx
import { UploaderRepeaterDropIndicator } from '../upload/repeater'

export const SimpleIndicator = () => (
	<UploaderRepeaterDropIndicator
		position="before"
		className="h-2 bg-blue-500/30 rounded-md"
	/>
)
```

### Real-World: Learning Platform File Reordering (Example 1)

This snippet shows how to place drop indicators before and after each sortable item inside a repeater.

```tsx
({ baseField, label, description, children, fileType, dropzonePlaceholder, ...props }) => {
	const defaultUploader = useS3Client()
	const [fileTypeStable] = useState(fileType)
	const fileTypeWithUploader = useMemo(
		() => ({ ...fileTypeStable, uploader: fileTypeStable?.uploader ?? defaultUploader }),
		[defaultUploader, fileTypeStable],
	)

	return (
		<FormFieldStateProvider>
			<FormContainer description={description} label={label}>
				<Repeater {...props} initialEntityCount={0}>
					<UploaderRepeaterItemsWrapperUI>
						<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
							<UploaderHasFile>
								<UploaderProgress />
							</UploaderHasFile>
							<UploaderDropzone dropzonePlaceholder={dropzonePlaceholder} />
						</MultiUploader>

						<RepeaterSortable>
							<RepeaterSortableEachItem>
								<div className="flex">
									<UploaderRepeaterDropIndicator position={'before'} />
									<RepeaterSortableItemNode>
										<UploaderRepeaterItemUI>
											<RepeaterSortableItemActivator>
												<UploaderRepeaterHandleUI />
											</RepeaterSortableItemActivator>
											{children}
										</UploaderRepeaterItemUI>
									</RepeaterSortableItemNode>
									<UploaderRepeaterDropIndicator position={'after'} />
								</div>
							</RepeaterSortableEachItem>

							<RepeaterSortableDragOverlay>
								<UploaderRepeaterDragOverlayUI>{children}</UploaderRepeaterDragOverlayUI>
							</RepeaterSortableDragOverlay>
						</RepeaterSortable>
					</UploaderRepeaterItemsWrapperUI>
				</Repeater>
			</FormContainer>
		</FormFieldStateProvider>
	)
}
```

The component pair (`before` and `after`) makes the insertion point obvious while dragging. The surrounding `RepeaterSortable` components handle the actual drag-and-drop mechanics.

## Best Practices & Gotchas
- Render the indicator only while dragging to avoid clutter.  
- Match the indicator’s width to its parent container for perfect alignment.  
- Use subtle, semi-transparent colors (`bg-color/opacity`) so the indicator does not overpower the UI.  
- Unmount the indicator immediately after the drop to prevent stray layout shifts.  
- Keep the indicator’s height small (2–4 px) so it reads as a guideline, not a divider.
