## Overview
`UploaderRepeaterItemUI` visually wraps a single item inside a file-upload repeater.  
It provides consistent spacing and borders, making each file preview look like a draggable card.

## Styling Guidelines
* All examples use Tailwind classes.  
* The component assumes Contember’s default theme but adds no theme-specific classes on its own.

## Core Concepts
* You wrap every uploaded file preview in `UploaderRepeaterItemUI`.
* The component works best with `RepeaterSortableItemActivator` for drag-and-drop.
* Pair it with `UploaderRepeaterHandleUI` to show a handle icon users can grab.
* Surround multiple items with `UploaderRepeaterItemsWrapperUI` for proper grid or flex layout.

## Quick Start
```tsx
import {
	UploaderRepeaterItemUI,
	UploaderRepeaterHandleUI,
} from '../upload'

export const FileItem = ({ children }) => (
	<UploaderRepeaterItemUI className="flex items-center gap-3 p-2 border rounded-md bg-white">
		<RepeaterSortableItemActivator>
			<UploaderRepeaterHandleUI />
		</RepeaterSortableItemActivator>

		{/* Put your <UploadedImageView /> or other preview here */}
		{children}
	</UploaderRepeaterItemUI>
)
```

## Sub-Components / Building Blocks
### UploaderRepeaterHandleUI
Draggable handle you place inside each `UploaderRepeaterItemUI`.

### RepeaterSortableItemActivator
Wrapper that enables drag-and-drop for the current item.

### UploaderRepeaterDragOverlayUI
Custom overlay shown while the item is being dragged.

### UploaderRepeaterItemsWrapperUI
Flex or grid container that holds a list of `UploaderRepeaterItemUI` components.

## Best Practices & Gotchas
* Always place the drag handle inside `RepeaterSortableItemActivator`.
* Keep padding consistent; otherwise drag overlay sizes may jump.
* Do not nest interactive elements with their own drag logic inside the item—it interferes with sorting.
* Combine with `UploaderHasFile` and `UploaderProgress` to show upload state beside the preview.
