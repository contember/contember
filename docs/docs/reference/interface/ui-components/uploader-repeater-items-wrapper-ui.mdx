# UploaderRepeaterItemsWrapperUI

## Overview
`UploaderRepeaterItemsWrapperUI` groups uploader controls and repeater items into a single visual block.  
You typically wrap a `MultiUploader` and a `RepeaterSortable` tree inside it so the uploader and each uploaded file share one layout context.

## Styling Guidelines
* All examples use Tailwind CSS classes.  
* No Contember-specific theme classes are required.

## Core Concepts
* Wrap uploader controls and sortable items in one container.  
* Maintain consistent spacing and borders around the uploader area.  
* Delegate all drag-and-drop logic to nested components (`RepeaterSortable*`, `UploaderRepeater*`).  
* The wrapper itself holds **no props**; its purpose is layout only.

## Quick Start
```tsx
import {
	UploaderRepeaterItemsWrapperUI,
	MultiUploader,
	UploaderDropzone,
	RepeaterSortable,
	RepeaterSortableEachItem,
	RepeaterSortableItemNode,
	UploaderRepeaterItemUI,
} from '../upload'

export const ImagesField = () => (
	<UploaderRepeaterItemsWrapperUI>
		<MultiUploader baseField="images">
			<UploaderDropzone dropzonePlaceholder="Drag & drop images" />
		</MultiUploader>

		<RepeaterSortable>
			<RepeaterSortableEachItem>
				<RepeaterSortableItemNode>
					<UploaderRepeaterItemUI className="flex gap-2 p-3 border rounded" />
				</RepeaterSortableItemNode>
			</RepeaterSortableEachItem>
		</RepeaterSortable>
	</UploaderRepeaterItemsWrapperUI>
)
```

## Examples

### 1. Uploader with Drag Handle and Drop Indicators
```tsx
<UploaderRepeaterItemsWrapperUI>
	<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
		<UploaderDropzone dropzonePlaceholder="Upload files" />
	</MultiUploader>

	<RepeaterSortable>
		<RepeaterSortableEachItem>
			<div className="flex">
				<UploaderRepeaterDropIndicator position="before" />
				<RepeaterSortableItemNode>
					<UploaderRepeaterItemUI className="flex items-center gap-2 p-2 border">
						<RepeaterSortableItemActivator>
							<UploaderRepeaterHandleUI />
						</RepeaterSortableItemActivator>
						{children}
					</UploaderRepeaterItemUI>
				</RepeaterSortableItemNode>
				<UploaderRepeaterDropIndicator position="after" />
			</div>
		</RepeaterSortableEachItem>

		<RepeaterSortableDragOverlay>
			<UploaderRepeaterDragOverlayUI>{children}</UploaderRepeaterDragOverlayUI>
		</RepeaterSortableDragOverlay>
	</RepeaterSortable>
</UploaderRepeaterItemsWrapperUI>
```
Use this pattern when you need sortable uploaded files with visual drop indicators.

### 2. Conditional Uploader Rendering
```tsx
<UploaderRepeaterItemsWrapperUI>
	{withoutUploader !== true ? (
		<MultiUploader baseField={baseField} fileType={fileTypeWithUploader}>
			<UploaderDropzone dropzonePlaceholder="Add files" />
		</MultiUploader>
	) : null}

	{/* RepeaterSortable tree identical to example above */}
</UploaderRepeaterItemsWrapperUI>
```
Render the wrapper even when the uploader is hidden. The layout of existing items stays stable.

## Best Practices & Gotchas
* Always place `MultiUploader` first so users drop files at the top.  
* Keep nested elements inside one flex container for predictable spacing.  
* The wrapper itself renders a plain `div`; add Tailwind classes on children, not the wrapper.  
* Do not attach event handlers directly to `UploaderRepeaterItemsWrapperUI`. Use nested components instead.  
* Remember to wrap the whole block in a `Repeater` when working inside Contember forms.
