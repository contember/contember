# UploaderRepeaterItemsWrapperUI

## Overview
`UploaderRepeaterItemsWrapperUI` groups uploader controls and repeater items into a single visual block.  
Wrap `MultiUploader` and a `RepeaterSortable` tree inside it so the uploader and each uploaded file share one layout context.

## Styling Guidelines
* All examples use Tailwind CSS classes.  
* No Contember-specific theme classes are required.

## Core Concepts
* Wrap uploader controls and sortable items in one container.  
* Maintain consistent spacing and borders around the uploader area.  
* Delegate drag-and-drop logic to nested components (`RepeaterSortable*`, `UploaderRepeater*`).  
* The wrapper holds **no props**; it only manages layout.

## Quick Start
```tsx
import {
	UploaderRepeaterItemsWrapperUI,
	MultiUploader,
	UploaderDropzone,
	RepeaterSortable,
	RepeaterSortableEachItem,
	RepeaterSortableItemNode,
	UploaderRepeaterItemUI,
} from '../upload'

export const ImagesField = () => (
	<UploaderRepeaterItemsWrapperUI>
		<MultiUploader baseField="images">
			<UploaderDropzone dropzonePlaceholder="Drag & drop images" />
		</MultiUploader>

		<RepeaterSortable>
			<RepeaterSortableEachItem>
				<RepeaterSortableItemNode>
					<UploaderRepeaterItemUI className="flex gap-2 p-3 border rounded" />
				</RepeaterSortableItemNode>
			</RepeaterSortableEachItem>
		</RepeaterSortable>
	</UploaderRepeaterItemsWrapperUI>
)
```

## Best Practices & Gotchas
* Place `MultiUploader` first so users drop files at the top.  
* Keep nested elements inside one `flex` container for predictable spacing.  
* The wrapper renders a plain `div`; add Tailwind classes to children, not the wrapper.  
* Avoid attaching event handlers directly to `UploaderRepeaterItemsWrapperUI`.  
* Wrap the whole block in a `Repeater` when working inside Contember forms.
