## Overview
`VideoField` lets users upload and preview video files in one step.
It wraps Contember’s uploader with a video tag, progress indicator, and optional metadata extraction.

## Styling Guidelines
* All snippets use Tailwind for layout and spacing.
* VideoField renders its own markup and applies Contember theme classes such as `ct-input`, `ct-label`, and `ct-upload-preview`. Override them with Tailwind utilities when needed.

## Core Concepts
* Uploading is handled by Contember’s upload client.
* You must map `urlField` (and optional metadata fields) to your entity tree.
* Validation can reject files via `accept`, `acceptFile`, or `noDestroy`.
* Preview uses the browser `<video>` element; no third-party player needed.

## Quick Start
```tsx
import { VideoField } from '~/lib/form'

export default () => (
  <div className="max-w-md space-y-4">
    <VideoField
      label="Demo Video"
      urlField="video.url"
    />
  </div>
)
```

## Detailed Guide
### Collecting video metadata
```tsx
<VideoField
  label="Lecture recording"
  baseField="video"
  urlField="url"
  durationField="duration"
  fileNameField="fileName"
  fileSizeField="fileSize"
  fileTypeField="fileType"
  lastModifiedField="lastModified"
  accept={{ 'video/*': ['.mp4', '.webm', '.ogg'] }}
/>
```
This variant stores common metadata so you can later show duration or file size in listings.

`FormLocales` renders one `VideoField` per language so each locale can have its own recording.

## Best Practices & Gotchas
* Always define `accept` to avoid user confusion on unsupported formats.
* Provide `noDestroy` only when legal or auditing rules forbid file deletion.
* Store `durationField` when you plan to show video length in lists.
* Keep previews under 300 px wide to prevent large layout shifts.
* Use `extractors` for custom metadata like video codec.
