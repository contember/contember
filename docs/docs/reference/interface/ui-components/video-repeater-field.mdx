# VideoRepeaterField

## Overview
VideoRepeaterField lets you manage an ordered collection of videos inside an Entity context.
It handles uploads, previews, drag-and-drop sorting, and progress tracking for each video file.

## Styling Guidelines
* All code samples use Tailwind classes.
* No special Contember theme classes are required; the component inherits your form styles.

## Core Concepts
* Runs only inside `<EntitySubTree />` or `<EntityListSubTree />`.
* Treats each item as a nested entity for metadata fields.
* Saves the video URL (or path) into the `urlField`.
* Maintains the item order via `orderBy` or `sortableBy`.
* Supports optional metadata fields (duration, size, type, etc.).

## Quick Start
```tsx
// tailwind: simple vertical stack
<div className="space-y-4">
  <VideoRepeaterField
    label="Course Videos"
    field="courses"
    urlField="video.url"
    orderBy="createdAt"
  />
</div>
```

## Examples

### Basic list ordered by creation time
```tsx
<VideoRepeaterField
  label="Course Videos"
  field="courses"
  urlField="video.url"
  orderBy="createdAt"
/>
```
The component lists existing videos by `createdAt` and lets users add more. Drag-and-drop is disabled because no `sortableBy` field is set.

### Sortable list with rich metadata
```tsx
<VideoRepeaterField
  field="courses"
  baseField="video"
  urlField="url"
  sortableBy="order"
  durationField="duration"
  fileNameField="fileName"
  fileSizeField="fileSize"
  fileTypeField="fileType"
  lastModifiedField="lastModified"
  label="Course Videos"
/>
```
This setup saves all video metadata under the nested `video` object and allows reordering by the integer `order` field.

## Best Practices & Gotchas
* Always place the component inside an Entity context; otherwise it throws runtime errors.
* Use `sortableBy` only on integer fields that default to sequential numbers.
* Keep `urlField`, `durationField`, and other metadata in the same entity to avoid query complexity.
* Limit max file size on the server; the component does not validate size.
* Provide a fallback poster image for videos if the backend cannot generate thumbnails.
