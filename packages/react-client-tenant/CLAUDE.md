# react-client-tenant

React hooks and headless components over the Contember **Tenant GraphQL API** (identity, authentication, projects, memberships, API keys, project secrets). This is the data layer that `react-identity` and `react-ui-lib-tenant` build on.

It is built on top of `@contember/graphql-client-tenant` — a fully typed client **generated from the engine's tenant SDL at build time** (`packages/graphql-client-tenant`, regenerated by its `pre-build`; the committed `src/generated` is just stubs). That generated client tracks the API by construction, so it is never the gap layer — real gaps live here (hand-written hooks/components) and in the UI package.

## GraphQL plumbing

- `useTenantApi(options?)` — returns an executor that runs a graphql-ts-client `Fetcher` against the tenant client (`useTenantGraphQlClient()` from `@contember/react-client`). Builds the query string from the fetcher DSL; resolves the auth token (`apiToken` / `LoginToken`).
- `useTenantMutation` / `createTenantMutation(fetcher)` — wrap a mutation fetcher and normalize the result into `TenantMutationResponse` (`{ ok: true, result } | { ok: false, error, developerMessage }`).
- `useTenantQueryLoader(fetcher, variables)` — drives a query hook through `loading → success/error → refreshing` states and exposes `refresh()`.

## Hooks

- **Queries** (`hooks/queries/`): `useMeQuery`, `useProjectMembersQuery`, `useProjectMembershipsQuery`, `useProjectRolesDefinitionQuery`, `usePersonsQuery`, `useProjectApiKeysQuery`, `useGlobalApiKeysQuery`, `useProjectSecretsQuery`.
- **Mutations** (`hooks/mutations/`, grouped by domain): `apiKey`, `auth`, `memberships`, `invite`, `otp`, `passwordReset`, `passwordless`, `emailVerification`, `profile`, `idp`.

Each hook defines a fragment with the `TenantApi.x$` / `x$$` fetchers, derives its result type via `ModelType<typeof fragment>`, and passes query arguments with `ParameterRef.of(...)`.

## Components

- **Form providers** (`components/forms/`): `TenantForm` plus per-flow wrappers — `LoginForm`, `InviteForm`, `CreateApiKeyForm`, `CreateGlobalApiKeyForm`, `UpdateProjectMemberForm`, `ChangeMyPasswordForm`, `PasswordReset*`, `VerifyEmailForm`, `PasswordlessSignIn*`, `Otp*`, etc. Each exposes a `useXForm()` hook (a typed view of `useForm`) consumed by the field components in `react-ui-lib-tenant`. Form state is `FormContextValue<Values, ErrorCode>` (`values` / `errors` / `state` / `setValue`).
- **Identity** (`components/`, `contexts.ts`): `IdentityProvider`, `IdentityState`, and the `useIdentity` / `useIdentityMethods` / `useIdentityState` contexts; `useFetchIdentity`, `useLogout`.
- **Triggers** (`components/triggers/`): `DisableOtpTrigger`, `LogoutTrigger`, `RemoveProjectMemberTrigger`, `TenantActionTrigger`.
- **IDP** (`components/idp/`): `IDP`, `IDPInitTrigger`, `IDPState` + `useIDPState`.

## Public API surface

`react-client-tenant.api.md` (in `build/api`) is checked by the `api-exporter` CI job. After adding/removing exports, regenerate it with `bun run ae:update` and commit the diff.

See [tenant API reference](https://docs.contember.com/reference/engine/tenant/overview) for the underlying operations.
